# SPEC 12 — 테스트 전략(TDD)·품질 게이트 (SP-TEST)

> **ID 대역**: `SP-TEST`. **상위 근거**: CONTEXT_BRIEF §10(TDD 규칙), FRD 전 문서(FR 커버리지), SP-ARCH §9(테스트 계층·도구·릴리스 게이트).
> **소유 범위(횡단)**: 프로젝트 전역 TDD 전략(red-green-refactor), 계층별 테스트·러너·디렉토리·명명 규약, 실행 명령·집계 러너, **각 SPEC 영역 테스트 명세의 종합·매핑 표**, 커버리지 목표·우선순위, 로컬 릴리스/품질 게이트, TASK 완료 기준(Definition of Done).
> **비소유(인용만)**: 개별 케이스 정의는 각 SP-* 문서가 소유한다(SP-DB-16·SP-SEED-11·SP-API-14·SP-ENGINE-17·SP-FE-12·SP-GEN-12·SP-ADS-10·SP-POL-10·SP-DS-11). 본 문서는 **중복 정의 없이 종합·매핑·게이트**만 확정한다.

---

## 0. 요약·범위 경계

loupit은 **무빌드 정적 프론트 + 슬림 읽기 전용 FastAPI + MySQL 참조 DB + 빌드타임 정적 생성기**로 구성된다(SP-ARCH-6). 로그인/회원/프로파일러/서버 측 사용자 쓰기는 영구 제외이므로(FR-01, INV-4) **인증·세션·쓰기 경로 테스트는 존재하지 않고**, 대신 그 부재(무쓰기·무인증·무전송)를 **회귀 테스트로 강제**한다. 계산은 100% 클라이언트이므로(§2-8) 핵심 검증 대상은 **순수 계산 모듈(`calc.js`)**, **읽기 전용 API 4종 계약**, **시드·백필 데이터 무결성**, **정적 생성물**, **렌더 이스케이프·광고 게이팅·디자인 토큰 대비**다.

**핵심 원칙(브리프 §10):** 모든 코드는 TDD(red-green-refactor). 새 모듈/공개 함수/라우트는 테스트 우선(또는 동시) 작성. **TASK 완료 기준 = 구현 + 테스트 통과 둘 다.** 한쪽만이면 "진행 중".

**CI 없음(로컬 전용).** 호스팅형 CI 파이프라인(GitHub Actions 등)을 두지 않는다. 테스트는 (a) 개발자 로컬, (b) 배포 호스트의 릴리스 스크립트(`infra/deploy/release.sh` 4단계, SP-ARCH-9)에서 실행되는 **로컬 게이트**로만 강제한다. 각 SP-* 문서가 "CI 게이트"로 부른 것은 본 문서에서 **로컬 릴리스 게이트**로 통일해 읽는다(§8). 격리 MySQL(`loupit_test`)은 배포/개발 호스트에서 별도 스키마 또는 로컬 docker `mysql:8.0` 컨테이너로 프로비저닝한다(원격 CI 인프라 아님).

## [SP-TEST-1] TDD 전역 원칙·TASK 완료 기준(Definition of Done)

**근거**: 브리프 §10, FRD 01(FR-08 추적성).

### 1.1 red-green-refactor 사이클(전 계층 공통)

| 단계 | 행위 | 게이트 |
| --- | --- | --- |
| **RED** | 구현 전 대상 공개 함수/라우트/DDL/생성물의 **실패하는 테스트**를 먼저 작성. 기대 입력→출력을 고정(경계·0나눗셈·오류 경로 포함). | 테스트가 실제로 실패(미구현) 확인 |
| **GREEN** | 테스트를 통과하는 **최소 구현**. | 대상 스위트 green |
| **REFACTOR** | 동작 불변 유지하며 정리(중복 제거·명명·순수성). | 스위트 재실행 green 유지 |

- **동시 작성 허용**: 순수 함수·DDL 등은 테스트를 구현과 동시에 작성해도 되나, **머지 시점엔 반드시 대응 테스트가 존재하고 green**이어야 한다(테스트 없는 스펙/코드는 미완).
- **회귀 우선**: 버그 수정은 먼저 그 버그를 재현하는 실패 테스트를 추가한 뒤 고친다(재발 방지).

### 1.2 TASK 완료 기준(DoD) — 전 TASK 공통 체크리스트

한 TASK는 아래를 **모두** 만족해야 "완료"다. 하나라도 미충족이면 "진행 중".

1. 구현이 대상 SP-* 항목의 계약(시그니처·타입·수식·DDL·엔드포인트·생성물)을 충족한다.
2. 해당 영역 테스트 스위트(§6 매핑의 담당 스위트)가 **green**이며, 새/변경 코드 경로에 **대응 케이스가 추가**되었다.
3. 변경이 다른 영역 계약을 건드리면 그 영역 스위트도 green(예: 스키마·시드 변경 PR = SP-DB-16 + SP-SEED-11 동시 통과 — SP-SEED §11 실행 주석).
4. §7 **Tier 0(회귀 게이트) 케이스**가 하나도 깨지지 않는다.
5. 범위 불변식(로그인/프로파일러/서버 쓰기 부재, 클라 계산, 96개 등록, DEC-2 밴드)을 위반하지 않는다(§9 INV 표).

## [SP-TEST-2] 테스트 계층·도구·러너 매트릭스

**근거**: SP-ARCH §9.1(계층·도구), SP-ARCH-7(버전 pin).

| 계층 | 대상 | 러너·도구(버전 pin) | 서버·DB 필요 | 소유 SPEC |
| --- | --- | --- | :---: | --- |
| **API 계약** | 라우트·응답·헤더·상태코드·오류 | `pytest 8.3.x` + `httpx 0.28.x`(`ASGITransport`) + `pytest-asyncio 0.24.x` | 러닝 서버·DB **불필요**(monkeypatch 캔드 주입) | SP-API·SP-ARCH |
| **DB 스키마·데이터 계약** | DDL·제약·시드·백필·불변식 | `pytest 8.3.x` + `pymysql`(동기) + **MySQL 8.0**(`loupit_test`) | 격리 MySQL **필요** | SP-DB·SP-SEED |
| **계산 순수모듈** | `web/assets/js/calc.js` | `node --test`(node ≥18) + `node:assert/strict` | 브라우저·네트워크 **불필요** | SP-ENGINE(≡SP-CALC) |
| **프론트 순수로직** | `dom/store/inputs/search/route` 순수부 | `node --test` + 최소 스텁(in-memory `localStorage`) | 불필요(수동 통합은 별도) | SP-FE·SP-ADS |
| **정적 생성물** | `web/dist/**` HTML·sitemap·정책 | `pytest 8.3.x` + fake 번들(DB 없이) | 불필요(선택 실 시드 스모크) | SP-GEN·SP-POL |
| **디자인 토큰·대비** | `styles.css` `:root`·`@font-face` | `node --test` + WCAG 대비 유틸(정적 CSS 파싱) | 불필요 | SP-DS(≡SP-DSN) |
| **통합 스모크** | 시드→생성→서빙 end-to-end | `bash`+`curl`(로컬 스택) | 로컬 전체 스택 | SP-ARCH·SP-INFRA |
| **설정 문법** | `infra/nginx/loupit.conf` | `nginx -t` | 배포 호스트 | SP-INFRA |
| **수동 QA/브라우저** | DOM 통합·시각 polish·접근성 | 체크리스트(육안·DevTools·axe/Lighthouse) | 브라우저 | SP-FE·SP-ADS·SP-DS |

- **무빌드 유지**: 프론트 테스트는 **브라우저가 로드하는 것과 동일 ES module 파일을 node가 import**한다(번들러·트랜스파일 없음). node ESM 해석을 위해 `web/package.json`에 `{"type":"module","private":true}`를 둔다(**node 테스트 전용**, 브라우저 무영향 — 브라우저는 `<script type="module">`로 로드).
- **DB 미의존 우선**: API 계약 테스트는 `database.fetch_all/fetch_one`·`build_reference_bundle`를 monkeypatch로 대체한 **캔드 행 주입**을 기본으로 하여 MySQL 없이 돌린다(SP-API-14). 실 MySQL 통합은 선택 스모크.

## [SP-TEST-4] 실행 명령·집계 러너

**근거**: 각 SP-* "실행" 주석, SP-ARCH-9(릴리스 4단계).

### 4.1 스위트별 개별 실행

| 스위트 | 명령 | 전제 |
| --- | --- | --- |
| 백엔드 전체(API+스키마+시드) | `python -m pytest server/tests/ -v` | `loupit_test` MySQL 기동(스키마·시드 픽스처는 conftest 적용). API 계약만 돌릴 땐 `-k "TS or TH or TR or TSE or TC or TM or TN or TCORS or TE"`(DB 불요) |
| API 계약만(DB 없이) | `python -m pytest server/tests/ -k "surface or health or reference or search or company_detail" -v` | 없음(monkeypatch 캔드) |
| 생성물+정책 | `python -m pytest generator/tests/ -v` | 없음(fake 번들). 실 시드 스모크는 `--bundle-json <snapshot>` |
| 계산 엔진 | `node --test web/assets/js/calc.test.js` | node ≥18 |
| 프론트 순수+광고+계산+토큰(집계) | `node --test web/` | node ≥18(`web/package.json` type:module) |
| 통합 스모크 | `bash infra/deploy/smoke.sh` | 로컬 전체 스택 기동 |
| Nginx 문법 | `nginx -t -c infra/nginx/loupit.conf` | nginx |

> `node --test web/`는 `web/` 하위의 모든 `*.test.js`(계산·FE 모듈·광고·토큰·메타)를 재귀 발견해 실행한다(`contrast.mjs`는 테스트 명명이 아니므로 import 전용, 직접 실행 안 됨).

### 4.2 집계 러너 `infra/deploy/run_tests.sh`(로컬 전용, CI 없음)

릴리스 게이트(SP-ARCH-9 4단계)와 개발자 사전 검증이 **동일 스크립트**를 호출한다. 어느 단계든 실패 시 non-zero exit로 배포를 차단한다.

```bash
#!/usr/bin/env bash
# infra/deploy/run_tests.sh — 전 계층 테스트 집계(로컬 전용). 실패 시 배포 차단.
set -euo pipefail
ROOT="$(cd "$(dirname "$0")/../.." && pwd)"; cd "$ROOT"

echo "[1/5] 백엔드(API·스키마·시드) — pytest (+ loupit_test MySQL)"
python -m pytest server/tests/ -q

echo "[2/5] 정적 생성물·정책 — pytest (fake 번들)"
python -m pytest generator/tests/ -q

echo "[3/5] 프론트 순수모듈·계산엔진·광고·디자인토큰 — node:test"
node --test web/

echo "[4/5] 테스트 하네스 자체 검증(메타) — 이미 [3]에 포함(web/test/harness.test.js)"

echo "[5/5] Nginx 설정 문법(배포 호스트에서만)"
if command -v nginx >/dev/null; then nginx -t -c "$ROOT/infra/nginx/loupit.conf"; else echo "  nginx 미설치 — 스킵(개발 로컬)"; fi

echo "ALL GREEN — 릴리스 게이트 통과"
```

- 통합 스모크(`smoke.sh`, T7·T8)는 서빙 스택 기동이 필요하므로 릴리스 파이프라인 **6~7단계**(서비스 재시작·Nginx reload 이후)에서 별도 호출한다(SP-ARCH-9).

## [SP-TEST-5] 픽스처·테스트 데이터·격리 전략

**근거**: SP-API-14.1, SP-DB-16, SP-SEED-11, SP-GEN-12.1, SP-FE-12, SP-ENGINE-17.

| 전략 | 대상 스위트 | 방법 |
| --- | --- | --- |
| **캔드 행 monkeypatch** | SP-API | `database.fetch_all/fetch_one`·`build_reference_bundle`를 async 스텁으로 대체(회사 2개·유형 6종 축약). `ASGITransport`로 러닝 서버 없이 앱 호출. 캐시는 `TTLCache` 주입, 풀 lifespan은 no-op. |
