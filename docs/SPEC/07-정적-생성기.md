# SPEC 07 — 정적 페이지 생성기 (빌드타임 SEO 산출물)

**영역 ID 대역**: `SP-GEN`
**저장 경로**: `docs/SPEC/07-정적-생성기.md`
**목적**: loupit 빌드타임 파이썬 정적 생성기(C2, `generator/`)의 구현 계약을 개발자가 추측 없이 그대로 만들 수 있는 수준으로 확정한다. (1) 데이터 소스(참조 번들 단일 소스, SP-ARCH-4) → 회사 상세 ~96개 HTML(`/company/{slug}`)·인기 조합 페이지(`/vs/{a}-{b}`)·정책 4종·404·`sitemap.xml`·`robots.txt` 산출 파이프라인, (2) slug/조합 경로 규칙과 충돌 검증, (3) Jinja2 템플릿 구조·이스케이프·필터·포맷 헬퍼, (4) 각 페이지의 SEO `<head>`(고유 title·meta description·OpenGraph·JSON-LD Organization·자기참조 canonical)와 JS 없는 본문(NFR12·NFR24), 콘텐츠 위주 광고 슬롯 자리, (5) 증분/전체 재생성·매니페스트, (6) 릴리스 플로우(해시·gzip·원자적 스왑), (7) 생성물 검증 테스트(개수·필수 태그·링크 404·slug 유일성)를 소유한다.

**상위 추적**: CONTEXT_BRIEF §2-6(광고 배치)·§2-7(SEO 정적 페이지)·§5(등록 96개)·§8(빌드타임 파이썬 생성 스크립트)·§10(TDD), FRD `07-회사페이지-정적생성.md`(**FR-50~FR-59**), FRD `08-인기조합-정적생성.md`(**FR-60~FR-65**), PRD `07-비기능-요구사항.md`(**NFR1·NFR2·NFR5·NFR6~NFR12·NFR13·NFR15·NFR19·NFR21·NFR22·NFR24·NFR29**), SPEC `01-개요와-아키텍처.md`(**SP-ARCH-1·3·4·6·7·9·10** — C2 컴포넌트·빌드타임 흐름·번들 단일 소스·디렉토리·버전·릴리스 파이프라인·불변식), SPEC `04-백엔드-API.md`(**SP-API-6·7** — `build_reference_bundle`·번들 필드맵), FLOW `06-회사상세-인기조합.md`(SCR-50~56), WIREFRAME `06-회사상세-인기조합.md`(G/B/C/V/M/W/L 구획·광고 슬롯 `content_mid`/`content_bottom`). 하위 TASK 문서는 `SP-GEN-N` ID를 인용한다.

**범위 경계**: 본 문서는 **빌드타임 정적 생성기와 그 산출물(HTML·sitemap·robots)·릴리스**만 소유한다.
- 참조 번들 빌더 함수(`server/services/reference.py::build_reference_bundle`)의 원시 SQL·조립 계약은 **SP-API-7**이 소유하고 본 문서는 이를 **소비**한다(단일 소스, SP-ARCH-4). DB DDL은 SP-DB, 96개 시드 재이식은 SP-SEED가 소유한다.
- 광고/제휴 슬롯의 **컴포넌트·게이팅·클라이언트 주입 로직**은 SP-ADS(F6)가 소유하고, 본 문서는 정적 페이지 내 **빈 슬롯 호스트 자리와 SP-ADS-9 배선 마크업(body page-type·동의 배너·진입 스크립트) 방출**만 규정한다(SP-GEN-8, 2026-07-19 개정 — 라벨·예약·client id는 전부 SP-ADS 런타임 소유).
- 정책 4종 페이지의 **본문 문안**은 SP-POL(F7)이 소유하고, 본 문서는 그 문안을 base 템플릿에 감싸 SEO·canonical·sitemap 엔트리를 부여하는 **렌더·병합**만 소유한다.
- 비교 툴 SPA 셸·프리필 **소비**(REF 해석·슬롯 반영)는 SP-FE가 소유하고, 본 문서는 CTA가 발신하는 **프리필 URL 파라미터의 형태**만 규정한다(FR-57·FR-62).
- 랜딩(`/`)·`styles.css`·디자인 실값은 SP-FE/SP-DSN 소유이며 본 문서는 랜딩 URL의 sitemap 기여만 담당한다.
- Nginx location·TLS·systemd·`release.sh` 오케스트레이션은 SP-INFRA가 소유하고, 본 문서는 생성기 CLI(`generator/build.py`)와 그 출력 계약, 릴리스 산출물(gzip·해시·매니페스트) 규격을 규정한다.
- 로그인/회원/프로파일러/서버측 사용자 쓰기는 영구 제외이며 생성·표시·CTA 어디에도 등장하지 않는다(FR-01, NFR16).

**전역 불변식(SP-ARCH-10 상속)**: INV-3(정적 경로는 앱·DB 미경유, 정적 HTML은 JS 없이 본문 가독) · INV-6(회사 등록 = 복지 실데이터 ~96개(≈95, SP-SEED-1), 회사페이지 프리셋 폴백 없음) · INV-2(번들 최상위 3키·프로파일러 키 부재). 밴드 계수(DEC-2)는 클라이언트 계산(SP-CALC) 소유이며 정적 페이지는 배지(공식/추정/만료)를 **표시**만 한다(FR-54).

## SP-GEN-1 — 생성기 개요·모듈 구조·데이터 소스·CLI

### SP-GEN-1.2 데이터 소스 — 번들 단일 소스 (`bundle.py`)

런타임 API와 **동일 함수** `build_reference_bundle`을 호출한다. 시그니처는 async(SP-API-7)이므로 빌드타임은 `asyncio.run`으로 구동한다. 테스트는 dict를 직접 주입한다.

```python
# generator/bundle.py
import asyncio, aiomysql
from server.config import get_settings
from server.services.reference import build_reference_bundle   # SP-API-7 단일 소스

async def _load_async() -> dict:
    s = get_settings()
    conn = await aiomysql.connect(
        host=s.db_host, port=s.db_port, user=s.db_user, password=s.db_password,
        db=s.db_name, charset="utf8mb4", cursorclass=aiomysql.DictCursor,
    )
    try:
        return await build_reference_bundle(conn)     # 최상위 3키(INV-2)
    finally:
        conn.close()

def load_bundle() -> dict:
    """빌드타임 진입. 반환 dict = {company_types[], benefit_presets{}, companies[{…,aliases[],benefits[]}]}"""
    return asyncio.run(_load_async())
```

- 반환 dict 구조·필드명은 SP-API-6/7 계약과 **1:1**(예: `comp_eng_nm`·`comp_nm`·`industry_nm`·`logo_nm`·`work_style_val`·`aliases[]`·`benefits[]`, 각 benefit `benefit_nm`·`benefit_amt`·`benefit_ctgr_cd`·`badge_cd`·`amt_source`·`qual_yn`·`qual_desc_ctnt`·`note_ctnt`·`verified_dtm`·`expires_dtm`·`badge_src_cd`·`badge_src_url_ctnt`·`sort_order_no`). 생성기는 이를 **재정의하지 않고 소비**한다.
- 등록 회사는 모두 `benefits[]` ≥1(실데이터, INV-6). 프리셋 폴백 없음.
- `--bundle-json <path>` 옵션으로 사전 덤프된 번들 JSON을 읽어 DB 없이 렌더 가능(CI·오프라인·재현). 없으면 `load_bundle()`로 DB 조회.

### SP-GEN-1.4 엔트리 CLI (`build.py`)

```python
# generator/build.py  (SP-INFRA release.sh가 호출: python -m generator.build --out web/dist)
import argparse, sys
from generator.config import CFG
from generator.bundle import load_bundle, load_bundle_json
from generator.context import build_context
from generator.render import make_env
from generator.pages import company, combo, policy, sitemap
from generator.release import stage_and_swap, write_manifest

def run(out_dir: str, bundle: dict, *, incremental: bool, only: list[str] | None,
        lastmod: str, gzip: bool) -> int:
    env = make_env()
    ctx = build_context(bundle)                       # 인덱스·검증(slug 충돌→예외, SP-GEN-3)
    pages = []
    pages += company.render_all(env, ctx)             # 회사 ~95 (SP-GEN-5·6)
    pages += combo.render_all(env, ctx, CFG)          # 조합 N (SP-GEN-7)
    pages += policy.render_all(env, ctx)              # 정책 4 + 404 (SP-POL 문안)
    if only:                                          # 개발용 필터
        pages = [p for p in pages if any(p.path.startswith(o) for o in only)]
    site_urls = [p.url for p in pages if p.in_sitemap] + \
                [CFG.site_origin + p for p in CFG.extra_sitemap_paths]
    pages.append(sitemap.render_sitemap(env, site_urls, lastmod))   # sitemap.xml (SP-GEN-9)
    pages.append(sitemap.render_robots())                           # robots.txt
    manifest = stage_and_swap(out_dir, pages, incremental=incremental, gzip=gzip)  # SP-GEN-11
    write_manifest(out_dir, manifest)
    return 0

def main(argv=None) -> int:
    ap = argparse.ArgumentParser("loupit static generator")
    ap.add_argument("--out", default=CFG.out_dir)
    ap.add_argument("--bundle-json")                  # DB 없이 렌더(선택)
    ap.add_argument("--incremental", action="store_true")   # 변경분만 기록(SP-GEN-10)
    ap.add_argument("--only", nargs="*")              # 경로 접두 필터(dev)
    ap.add_argument("--lastmod")                      # 기본 = 오늘(UTC date)
    ap.add_argument("--no-gzip", dest="gzip", action="store_false")
    a = ap.parse_args(argv)
    bundle = load_bundle_json(a.bundle_json) if a.bundle_json else load_bundle()
    lastmod = a.lastmod or _today_iso()
    return run(a.out, bundle, incremental=a.incremental, only=a.only, lastmod=lastmod, gzip=a.gzip)

if __name__ == "__main__":
    sys.exit(main())
```

- 종료 코드: 0 = 성공(검증 통과·스왑 완료), 비0 = 렌더/검증/스왑 실패(이전 산출물 유지, SP-ARCH-9). 검증 실패 예외는 `BuildError`(SP-GEN-3·12).
- **추적**: FR-50, SP-ARCH-3·4·6·9, NFR1, §8.

## SP-GEN-2 — 출력 디렉토리·경로 규약·Page 모델

**근거**: FR-50·FR-51·FR-60, SP-ARCH-6(레이아웃)·SP-ARCH-2(Nginx location).

### SP-GEN-2.1 출력 트리 (`web/dist/`)

```
web/dist/
├─ company/{slug}.html          # 회사 상세 ~95 (SCR-06/50)           canonical /company/{slug}
├─ vs/{first}-{second}.html     # 인기 조합 N (SCR-07/53)             canonical /vs/{first}-{second}
├─ privacy.html terms.html disclaimer.html ads.html   # 정책 4종(SCR-08)
├─ 404.html                     # 오류 안내(SCR-09)
├─ sitemap.xml                  # 전 URL(회사+조합+정책+랜딩) NFR9
├─ robots.txt                   # Sitemap: 라인 포함 NFR10
├─ *.gz                         # 위 .html/.xml의 사전압축(gzip_static, SP-GEN-11)
└─ .manifest.json               # path→{sha256,bytes,gz} (증분·릴리스 검증, gitignore)
```

- **경로↔URL 매핑**(SP-ARCH-6 정본, `.html` 평면 레이아웃): 파일 `company/{slug}.html` → URL `/company/{slug}`. Nginx는 `try_files $uri $uri.html $uri/ =404`로 확장자 없는 canonical URL을 파일에 매핑한다(location 상세는 SP-INFRA). FRD-51의 `company/{slug}/index.html` 표현과 **동일 canonical URL을 산출**하며, 본 SPEC은 SP-ARCH-6의 평면 `.html`을 채택한다.
- `web/dist/`는 gitignore(빌드 산출물). 배포 시 생성(SP-ARCH-6·SP-GEN-11).

### SP-GEN-2.2 Page 모델 (`context.py`)

모든 렌더 함수는 아래 `Page`를 반환한다. 릴리스(SP-GEN-11)·sitemap(SP-GEN-9)·검증(SP-GEN-12)이 공통 소비.

```python
# generator/context.py
from dataclasses import dataclass

@dataclass(frozen=True)
class Page:
    path: str            # dist 상대 경로 (예: "company/samsung-elec.html")
    url: str             # 절대 canonical URL (예: "https://jobcho.wiki/company/samsung-elec")
    html: str            # 완성 HTML 문자열 (또는 sitemap/robots는 xml/txt)
    title: str           # <title> 텍스트(중복 검증용, FR-59)
    description: str      # meta description(중복 검증용)
    in_sitemap: bool = True    # 404 등은 False
    content_type: str = "text/html; charset=utf-8"
```

### SP-GEN-2.3 컨텍스트 인덱스 (`build_context`)

## SP-GEN-3 — slug 규칙·조합 경로 규칙·충돌 검증

**근거**: FR-51(회사 slug), FR-60 R2·R3(조합 경로), FR-59(충돌=빌드 실패), NFR11.

### SP-GEN-3.1 회사 slug (`slug_of`) — FR-51

```python
# generator/slug.py
import re
class BuildError(Exception): ...

def slug_of(comp_eng_nm: str) -> str:
    s = comp_eng_nm.strip().lower()
    s = re.sub(r"[^a-z0-9]+", "-", s)   # 비영숫자(공백·_·기호) → 하이픈
    s = re.sub(r"-{2,}", "-", s)        # 연속 하이픈 축약
    s = s.strip("-")                    # 양끝 하이픈 제거
    if not s:
        raise BuildError(f"empty slug for COMP_ENG_NM={comp_eng_nm!r}")
    if not re.fullmatch(r"[a-z0-9]+(-[a-z0-9]+)*", s):
        raise BuildError(f"invalid slug {s!r}")
    return s
```

- 예: `samsung_elec`→`samsung-elec`, `sk_hynix`→`sk-hynix`, `cj`→`cj`, `S-Oil` 원본 eng가 `soil`이면 `soil`. slug 소스는 `COMP_ENG_NM`(unique, D1.2)이며 한글 정식명은 URL에 넣지 않는다.
- 재빌드 간 동일 `COMP_ENG_NM` → 동일 slug(안정 URL, 색인 유지). `COMP_ENG_NM` 변경은 URL 변경이므로 데이터 변경 시 주의(FR-51 R2).

### SP-GEN-3.2 slug 유일성 검증 (`validate_slugs`) — FR-59

```python
def validate_slugs(companies: list[dict]) -> dict[str, str]:
    slugs = {}
    seen = {}
    for c in companies:
        s = slug_of(c["comp_eng_nm"])
        if s in seen and seen[s] != c["comp_eng_nm"]:
            raise BuildError(f"slug collision {s!r}: {seen[s]} vs {c['comp_eng_nm']}")
        seen[s] = c["comp_eng_nm"]; slugs[c["comp_eng_nm"]] = s
    assert len({slugs[c['comp_eng_nm']] for c in companies}) == len(companies)  # 충돌 0
    return slugs
```

### SP-GEN-3.3 조합 경로 (`combo_slug`) — FR-60 R2·R3

두 회사 slug를 **사전식(정렬) 순서**로 배치해 단일 canonical 경로를 만든다. 역순 쌍(B vs A)은 별도 파일을 만들지 않고 동일 canonical로 귀속된다(NFR11).

```python
def combo_slug(eng_a: str, eng_b: str, slugs: dict[str, str]) -> tuple[str, str, str]:
    sa, sb = slugs[eng_a], slugs[eng_b]
    first, second = sorted([sa, sb])            # 사전식 정규화(canonical)
    return f"{first}-{second}", first, second   # 경로 세그먼트, 정렬 첫/둘째
```

- 경로 `vs/{first}-{second}.html`, URL `/vs/{first}-{second}`. 정렬로 canonical 1개 고정(FR-60 R2).
- **세그먼트 모호성(FR-60 R3)**: 회사 slug 내부 하이픈으로 인해 서로 다른 두 조합이 동일 `first-second` 문자열을 만들 위험이 있다(예: (`a-b`,`c`) vs (`a`,`b-c`) → `a-b-c`). 본 SPEC은 **전 조합 경로 유일성**을 빌드타임에 강제한다(아래 `validate_combo_paths`). 충돌 시 빌드 실패(데이터/목록 교정). 비교 툴 진입은 경로 파싱이 아니라 **명시적 파라미터**(`a=`,`b=`, FR-62 R2)로 회사를 지시하므로, 경로 문자열의 역파싱은 라우팅·프리필에 불필요하다(정적 파일은 정확한 파일명 매칭으로 서빙).

```python
def validate_combo_paths(pairs: list[tuple[str, str]], slugs) -> dict[str, tuple[str, str]]:
    """pairs=[(eng_a,eng_b)] → {combo_path: (eng_first,eng_second)}. 경로 충돌→BuildError."""
    out = {}
    for a, b in pairs:
        path, _, _ = combo_slug(a, b, slugs)
        if path in out and set(out[path]) != {a, b}:
            raise BuildError(f"combo path collision {path!r}: {out[path]} vs {(a, b)}")
        out[path] = (a, b)
    return out
```

- **추적**: FR-51·FR-59·FR-60(R2·R3)·NFR11.

### SP-GEN-4.3 포맷·파생 헬퍼 (`format.py`)

```python
# generator/format.py
import json
from datetime import datetime, date

def krw_manwon(amt) -> str:                 # 만원 정수 → "만/억" 한국어 (FR-04)
    if amt is None: return ""
    amt = int(amt); eok, man = divmod(amt, 10000)
    if eok and man:  return f"{eok}억 {man:,}만원"
    if eok:          return f"{eok}억원"
    return f"{man:,}만원"                    # 1200 → "1,200만원"

def badge_state(benefit: dict, now: datetime) -> dict:   # FR-54·FR-05 3파생
    """returns {'code': official|stale|est, 'label': str}"""
    exp = benefit.get("expires_dtm")
    if exp and _to_dt(exp) < now:
        return {"code": "stale", "label": "만료·재확인 필요"}
    if benefit.get("badge_cd") == "official":
        return {"code": "official", "label": "공식 확인"}
    return {"code": "est", "label": "추정"}

def jsonld_dumps(obj) -> str:               # <script> 안전 JSON (XSS·breakout 차단, NFR21)
    s = json.dumps(obj, ensure_ascii=False, separators=(",", ":"))
    return s.replace("<", "\\u003c").replace(">", "\\u003e").replace("&", "\\u0026")

def iso_date(v) -> str:
    if v is None: return ""
    return v[:10] if isinstance(v, str) else v.isoformat()[:10]

WS_LABELS = {"remote": "재택근무", "flex": "유연근무", "unlimitedPTO": "무제한 휴가",
             "refreshLeave": "리프레시 휴가", "overtime": "야근 있음(고지)"}
def work_style_label(key: str) -> str:
    return WS_LABELS.get(key, key)
```

- `jsonld_dumps` 결과는 **템플릿에서 `| safe`**로 `<script type="application/ld+json">` 내부에 삽입한다(이미 `<`/`>`/`&`를 유니코드 이스케이프해 script breakout·HTML 파싱 오염 불가). autoescape가 JSON을 훼손하지 않도록 하는 유일한 예외 경로다.
- `badge_state`는 FR-54 3파생(공식/만료/추정)만 산출한다. **밴드 계수(DEC-2)는 계산하지 않는다**(SP-CALC 소유). 정량 금액이 `amt_source='estimated'`면 라벨 옆에 "추정 금액" 보조 표기를 붙일 수 있으나(선택), 신뢰도 배지 자체는 3파생을 따른다.

### SP-GEN-5.3 복지표 그룹핑 (`_group_benefits`) — FR-53·FR-54

```python
CATEGORY_ORDER = ["compensation","flexibility","work_env","time_off",
                  "health","family","growth","leisure","perks"]      # D1.7 9종
CATEGORY_LABEL = {"compensation":"보상","flexibility":"유연성","work_env":"근무환경",
                  "time_off":"휴가","health":"건강","family":"가족","growth":"성장",
                  "leisure":"여가","perks":"복리후생"}

def _group_benefits(benefits, now):
    buckets = {k: [] for k in CATEGORY_ORDER}
    for b in benefits:
        item = {
            "name": b["benefit_nm"],
            "amount": krw_manwon(b["benefit_amt"]) if not b["qual_yn"] else "",  # FR-04
            "qual": b["qual_yn"], "qual_desc": b.get("qual_desc_ctnt"),
            "note": b.get("note_ctnt"),
            "badge": badge_state(b, now),                       # 공식/추정/만료 (FR-54)
            "src_cd": b.get("badge_src_cd"),
            "src_url": _safe_http(b.get("badge_src_url_ctnt")), # http/https만(FR-54 R3)
            "verified": iso_date(b.get("verified_dtm")),
            "expires": iso_date(b.get("expires_dtm")),
            "sort": b.get("sort_order_no") or 0,
        }
        buckets.setdefault(b["benefit_ctgr_cd"], []).append(item)   # 알 수 없는 카테고리도 방어적 수용
    for k in buckets:
        buckets[k].sort(key=lambda x: x["sort"])                # SORT_ORDER_NO 우선
    return [(k, CATEGORY_LABEL[k], buckets[k]) for k in CATEGORY_ORDER if buckets[k]]  # 비지 않은 카테고리만

def _safe_http(url):     # 출처 URL 스킴 제한 (FR-54 R3, NFR21)
    if not url: return None
    return url if url.startswith(("http://", "https://")) else None
```

- 등록 회사는 복지 ≥1(INV-6)이므로 **비지 않은 그룹이 ≥1** 존재(검증 SP-GEN-12 GC-복지). 프리셋 폴백 없음(FR-53).

### SP-GEN-5.4 "이 회사로 비교하기" CTA — FR-57

```python
def _related(eng_first, eng_second, ctx, all_pairs):
    a_first, a_second = ctx.by_eng[eng_first], ctx.by_eng[eng_second]
    company_links = [                                   # L1: 양사 회사 상세(항상 존재)
        (a_first["comp_nm"],  f"/company/{ctx.slugs[eng_first]}"),
        (a_second["comp_nm"], f"/company/{ctx.slugs[eng_second]}"),
    ]
    related = []                                        # L2: 관련 조합(생성된 것만)
    for (x, y) in all_pairs:
        if {x, y} == {eng_first, eng_second}: continue
        share = {x, y} & {eng_first, eng_second}        # 공유 회사
        same_ind = ctx.by_eng[x]["industry_nm"] and \
                   ctx.by_eng[x]["industry_nm"] == ctx.by_eng[eng_first]["industry_nm"]
        if share or same_ind:
            p, _, _ = combo_slug(x, y, ctx.slugs)
            related.append((f"{ctx.by_eng[x]['comp_nm']} vs {ctx.by_eng[y]['comp_nm']}", f"/vs/{p}"))
    return company_links, related[:6]                   # 관련 없으면 L2 미표시(FR-63 R1)
```

## SP-GEN-11 — 릴리스 플로우 (해시·gzip·원자적 스왑)

**근거**: SP-ARCH-9(릴리스 파이프라인 3~4단계), §8(gzip), NFR2·NFR5(성능). 실패 시 이전 산출물 유지(무중단 지향).

### SP-GEN-11.1 절차 (`release.py::stage_and_swap`)

```
1. 스테이징: {out}.next/ 를 비우고 전 Page.html/xml/txt 기록(증분이면 변경분 + 이전본 복사)
2. gzip 사전압축: 각 .html/.xml/.txt → .gz (gzip level 9). 원본보다 크면 .gz 생략(gzip_static)
3. 해시 매니페스트: 파일별 sha256·bytes·gz 기록 → {out}.next/.manifest.json
4. 검증 게이트: run_generated_checks({out}.next)  (SP-GEN-12, 실패→예외·스왑 중단)
5. 원자적 스왑: os.replace({out} → {out}.prev) ; os.replace({out}.next → {out})   # 디렉토리 rename
   (실패 시 {out} 유지, {out}.prev 롤백 가능)
```

```python
def stage_and_swap(out, pages, *, incremental, gzip) -> dict:
    nxt = out + ".next"; _reset_dir(nxt)
    prev = _load_manifest(out) if incremental else {}
    manifest = {}
    for p in pages:
        target = os.path.join(nxt, p.path); _ensure_parent(target)
        data = p.html.encode("utf-8")
        digest = hashlib.sha256(data).hexdigest()
        if incremental and prev.get(p.path, {}).get("sha256") == digest and os.path.exists(os.path.join(out, p.path)):
            _copy(os.path.join(out, p.path), target)         # 변경 없음 → 이전본 복사
        else:
            _write_bytes(target, data)
        gz = _gzip_if_smaller(target, data) if gzip and _is_compressible(p.path) else None
        manifest[p.path] = {"sha256": digest, "bytes": len(data), "gz": gz}
    run_generated_checks(nxt, pages)                          # SP-GEN-12 게이트
    _atomic_swap(out, nxt)                                    # os.replace 2단
    return manifest
```

### SP-GEN-11.2 규격

| 항목 | 규칙 | 근거 |
