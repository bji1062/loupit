# 06. 프론트엔드 구조 (SPA) — SP-FE

**영역 ID 대역**: `SP-FE`
**저장 경로**: `docs/SPEC/06-프론트엔드-구조.md`
**목적**: loupit 비교 툴 **무빌드 바닐라 JS SPA**(C1 `web/`)의 구조를 개발자가 추측 없이 그대로 만들 수 있는 수준으로 확정한다. (1) 파일·모듈 구성과 로드 순서, (2) HTML 셸과 점진적 향상(비-JS 본문), (3) 화면 라우팅(`go(screenId)`·해시/History·`popstate`), (4) 전역 클라이언트 상태 모델(`matched`/`benS`/`wsState`/`curPri`… — 프로파일러 상태 없음), (5) 부팅 시 `reference/all` 로드·캐시, (6) 읽기 전용 API 클라이언트(`apiFetch`, 인증 헤더 0), (7) 검색·선택·정규화(`normalizeCompany`)·프리셋 채움 오케스트레이션, (8) 렌더링 안전 규약(이스케이프·`textContent`)·**엔진 호출 조립(`assembleCompareState`)·리포트 DOM 렌더**(판정카드·카테고리 델타·배지/밴드 — 엔진 반환 구조체 소비, 구 SP-RPT 대역 흡수), (9) "최근 비교" localStorage(봉투 `v:1`·최대 10·FIFO·손상 폐기), (10) URL 파라미터 프리필, (11) 테스트 명세를 소유한다.

**상위 추적**: CONTEXT_BRIEF `§8`(기술 스택·무빌드·옵션1 디자인 토큰)·`§2`(무로그인·무쓰기·클라이언트 계산)·`§6`(읽기 전용 API), FRD `03-검색-회사선택.md`(**FR-10~FR-17**), FRD `04-비교입력.md`(**FR-20~FR-25**), FRD `06-리포트-표현.md`(**FR-40·FR-41·FR-42·FR-43·FR-44·FR-45** 리포트 구성·배지/밴드·재계산·localStorage·렌더 안전), FRD `01-개요와-FR규칙.md`(**FR-02·FR-03·FR-07** 부팅·상태 모델·로컬 지속성, **§4 용어**, **§5 전역 상태**), FRD `02-데이터-계약.md`(**FR-D4·FR-D6·FR-D7·FR-D8** 회사 객체·검색 투영·`normalizeCompany`·`value_source`), FLOW `03·04·05`(SCR-20~43·FLOW-20~43), WIREFRAME `03·04·05`, SPEC `01-개요와-아키텍처.md`(**SP-ARCH-5·6·10** 런타임 흐름·디렉토리·불변식), SPEC `04-백엔드-API.md`(**SP-API-9·10·11·12·13** 엔드포인트 계약·캐시 헤더).

**범위 경계**: 본 문서는 **SPA 셸·부팅·라우팅·상태·검색/입력 오케스트레이션·API 클라이언트·로컬 저장 배관**을 소유한다. **순수 계산**(실효연봉·시간가치·밴드·vdCard 판정 수치)은 `calc.js`(SP-CALC/SP-ENGINE)가, **리포트 렌더(판정카드·카테고리 델타·배지/밴드 표시 마크업)·"최근 비교" 요약 콘텐츠·저장 UI**는 **본 문서(SP-FE-9.3·9.4)**가 엔진 반환 구조체를 소비해 소유하고(구 'SP-RPT' 대역은 별도 문서 없이 본 문서가 흡수 소유 — SP-TEST(12) 결정과 정합), **광고 슬롯·게이팅**은 SP-ADS(09)가, **색/타이포/간격 토큰과 스타일**은 SP-DSN(11 `styles.css`)이 소유한다. 본 문서는 이들을 **호출·마운트하는 지점**까지 명세하고 그 내부 로직을 재정의하지 않는다. 정적 회사/조합/정책 페이지(F4·F5·F7)는 SP-GEN(05)이 소유하며, 본 문서는 그 페이지의 CTA가 넘긴 **URL 파라미터를 소비**(프리필, SP-FE-11)하는 쪽만 다룬다. 로그인/회원/프로파일러/서버측 사용자 쓰기는 영구 제외이며 어떤 모듈·상태·화면에도 없다(FR-01·FR-03 §5.1).

**전역 불변식(SP-ARCH-10 상속)**: INV-4(비교 계산·사용자 상태는 클라이언트/localStorage에만, 런타임 사용자 데이터 서버 전송 0) · INV-2(REF 최상위 3키·프로파일러 키 부재·`Cache-Control: public, max-age=3600`) · NFR21(데이터 보간 `innerHTML` 금지, 이스케이프/`textContent`).

## SP-FE-1 — 파일 구성·모듈 그래프

**구현**: FR-02·FR-03 / SP-ARCH-6 상속. 무빌드(번들러·트랜스파일 0). ES2020+ 네이티브 모듈(`<script type="module">`)로 로드한다.

### SP-FE-1.2 의존 방향(비순환)

```
        app.js (엔트리·상태 소유·엔진 호출 조립)
         │  imports
   ┌─────┼───────┬────────┬────────┬────────┬───────┬───────┐
   ▼     ▼       ▼        ▼        ▼        ▼       ▼       ▼
 boot   search  inputs  store   report   ads    calc    (dom)
   │     │       │                 │            (순수)
   └─►api└─►api  └─►dom           └─►dom
```

- **규칙**: (1) `calc.js`는 아무 것도 import하지 않는다(순수, `fetch`/`localStorage`/DOM 미참조 — SP-CALC INV-4, node:test 대상). (2) `dom.js`는 브라우저 표준 API만 사용(다른 앱 모듈 import 0)해 단위 테스트 가능. (3) 상태(`App.state`)는 `app.js`가 **단일 소유**하고 다른 모듈은 함수 인자 또는 `App` 네임스페이스로 접근한다(전역 난립 금지). (4) 순환 import 금지. (5) `report.js`는 **이미 계산된** `Report`(SP-ENGINE-2.2)만 렌더하므로 `calc.js`를 import하지 않고 `dom.js`만 쓴다(표현·계산 분리). 엔진 호출(`compare`/`calc`)은 `app.js`가 `assembleCompareState`(SP-FE-9.3)로 App.state를 엔진 입력으로 조립해 수행한다.
- **로드**: 셸이 `<script type="module" src="/assets/js/app.js" defer>` 하나만 로드하고, `app.js`가 나머지를 `import`한다(HTTP/2 다중 요청; 무빌드이므로 개별 파일 서빙). 폰트·CSS는 `<head>`에서 우선 로드.
- **추적**: FR-02·FR-03, SP-ARCH-5·6, INV-4.

## SP-FE-3 — 화면 라우팅(`go(screenId)`·해시/History)

**구현**: FR-02·FR-03 / FLOW-20·30·40·41 / SCR-03→04→05.

비교 툴은 **단일 문서 내 4뷰 상태 머신**이다. 서버 라우트를 추가하지 않고(모든 `/compare*`는 셸) **해시 라우팅**을 정본으로 하며, History `pushState`로 뒤로가기·앞으로가기를 지원한다.

동일한 셸·동일한 `app.js`를 랜딩(`web/index.html`)과 비교 툴(`web/compare/index.html`)이 **공유**한다. 따라서 본 절의 모든 규칙은 `/`와 `/compare` 양쪽에 동일하게 적용되며, 검증도 양쪽에서 수행한다.

### SP-FE-3.1 뷰 정의

| screenId | 뷰 컨테이너 | 대응 SCR | 진입 조건 |
| --- | --- | --- | --- |
| `search` | `#view-search` | SCR-03 | 기본(부팅 후) / 슬롯 재구성 |
| `input` | `#view-input` | SCR-04 | 슬롯 a·b 구성 후(또는 프리필/직접입력) |
| `report` | `#view-report` | SCR-05 | "비교하기" 실행 성공 / 부팅 시 `#report` + 최근 레코드 복원 성공(SP-FE-10.3 L-8) |
| `company` | `#view-company` | — | GNB 검색 제출(`showCompanyPage(term)`)로만 진입. **해시 딥링크로는 진입 불가**(렌더 재료인 검색어가 없다) |

### SP-FE-3.2 라우팅 API·전이

```js
// app.js
const SCREENS = ['search', 'input', 'report', 'company'];

function go(screenId, { push = true } = {}) {
  if (!SCREENS.includes(screenId)) screenId = 'search';   // 방어: 미지 뷰 → 검색
  App.state.ui.screen = screenId;
  for (const s of SCREENS) {
    document.getElementById('view-' + s).hidden = (s !== screenId);
  }
  if (push) history.pushState({ screen: screenId }, '', '#' + screenId);  // 해시 + History 상태
  window.scrollTo(0, 0);
  focusFirstHeading(screenId);                             // 접근성: 뷰 전환 시 포커스 이동(NFR14)
}

// 해시가 요구한 뷰 + 현재 사실 → 실제로 들어갈 뷰(규칙 3의 술어). 순수 함수 — 브라우저 무의존.
// restore:true는 "복원을 시도하라"는 지시일 뿐이며, 이동(go)은 언제나 호출부가 소유한다.
function resolveBootScreen({ want, hasSlotState, hasReport, recentCount }) {
  const fallback = hasSlotState ? 'input' : 'search';
  if (want === 'search') return { screen: 'search', restore: false };
  if (want === 'report') {
    if (hasReport)        return { screen: 'report', restore: false };  // 이미 렌더됨(popstate)
    if (hasSlotState)     return { screen: 'input',  restore: false };  // 프리필 > 자동 복원(규칙 5)
    if (recentCount > 0)  return { screen: 'search', restore: true };   // 복원 성공 시 report
    return { screen: 'search', restore: false };                        // 복원 재료 없음 → 강등
  }
  return { screen: fallback, restore: false };  // 'input'(슬롯 없음)·'company'·null·미지
}

// 규칙 (3)의 '상태' 술어 — 부팅·popstate 공통.
function hasSlotState(state) {
  return !!(state.matched.a || state.matched.b || state.chosenType.a || state.chosenType.b
    || state.inputMode.a === 'direct' || state.inputMode.b === 'direct');
}

window.addEventListener('popstate', (e) => {
  // go()가 만든 항목은 뷰 DOM이 살아 있으므로 신뢰한다(세션 내 뒤로/앞으로 상태 보존).
  if (e.state && e.state.screen) { go(e.state.screen, { push: false }); return; }
  // 무상태 항목(네이티브 앵커·크로스도큐먼트 복원) → 부팅과 동일 강등. 자동 복원은 하지 않는다.
  const d = resolveBootScreen({ want: parseHash(), hasSlotState: hasSlotState(App.state),
    hasReport: hasRenderedReport(), recentCount: 0 });
  go(d.screen, { push: false });                          // 뒤로/앞으로 → 재푸시 없이 표시만
});

function parseHash() {                                     // '#input' → 'input'
  const h = location.hash.replace(/^#/, '');
  return SCREENS.includes(h) ? h : null;
}
```

### SP-FE-3.3 전이 규칙

| 전이 | 트리거 | 가드 |
| --- | --- | --- |
| `search → input` | 슬롯 지정/직접입력/프리필 완료 | 최소 한 슬롯이 구성(회사 선택 또는 직접입력 진입). 미충족 시 안내 후 잔류 |
| `input → report` | "비교하기" | 연봉 등 필수값은 F3가 부분 미산출로 흡수(리포트 무효화 금지, FR-40 2a). 전이 자체는 항상 허용 |
| `report → input` | "입력 수정" | 상태 보존, 값 유지 |
| `report → search` | "슬롯 교체" | 해당 슬롯 재검색(FR-15) |
| `* → search` | 미지 해시/`popstate` 유실 | 방어 기본값 |
| `#report → search` | 부팅/무상태 `popstate`, 슬롯·레코드 없음 | 빈 `#report-body` 노출 금지(딥링크 강등) |

| `#input → search` | 부팅/무상태 `popstate`, 슬롯 없음 | 막다른 입력 뷰(빈 슬롯) 노출 금지 |
| `#company → search\|input` | 항상 | 검색어 없이 진입 불가 — 백지(sr-only h2뿐) 노출 금지 |

- **규칙**: (1) 뷰 전환은 DOM `hidden` 토글이며 재마운트하지 않는다(입력 상태 보존). (2) 해시는 `#search`/`#input`/`#report`만 유효하며, `#company`는 SCREENS 원소이지만 **딥링크 진입 대상이 아니다**(규칙 3에 의해 항상 강등). 그 외는 `search`로 정규화. (3) 부팅 시 `parseHash()`가 지정한 뷰로 초기 진입하되, **상태가 없으면 강등**한다 — 판정은 `resolveBootScreen()`이 단독 소유하며 뷰별 '상태'의 정의는 아래와 같다. (4) 프리필 진입(SP-FE-11)은 `go('input')` 또는 `go('search')`로 합류한다. (5) **프리필 > 자동 복원**: 슬롯이 이미 차 있으면 `#report`라도 최근 레코드를 복원하지 않는다(사용자가 방금 지정한 쌍이 우선).

| 뷰 | '상태 있음'의 정의 | 없을 때 |
| --- | --- | --- |
| `report` | `#report-body`에 렌더된 리포트가 있음, 또는 복원 가능한 최근 레코드가 있음(`recent.list().length > 0`) | `search` |
| `input` | `hasSlotState()` — 회사 슬롯 · `chosenType` · `inputMode==='direct'` 중 하나 | `search` |
| `search` | 해당 없음(항상 진입 가능) | — |
| `company` | 원리적으로 없음(렌더 재료인 검색어가 URL에 없다) | `search`(슬롯 있으면 `input`) |

- **비고**: 규칙 (3)은 본 명세의 최초판부터 존재했으나 SP-FE-3.2·SP-FE-5.1의 의사코드가 조건을 누락해 구현이 이를 어겼다(HANDOFF-2026-07-19 §B-1, 2026-07-19 수정). 술어를 순수 함수로 명시한 것은 같은 누락이 재발하지 않게 하기 위함이다.

## SP-FE-5 — 부팅 시퀀스·`reference/all` 로드·캐시

### SP-FE-5.1 부팅 순서(`boot()`)


- **B-6 부팅 뷰 결정**: `go`는 부팅 경로에서 **정확히 1회** 호출된다. 자동 복원 헬퍼에는 no-op `go`를 주입해 이동 권한을 `boot()`이 독점하며(이중 `go`·부팅 중 `pushState` 방지), 주입 시 `deps`를 변형하지 않고 새 객체를 만든다 — `recentCtx.onRestore`가 클로저로 붙잡은 `deps`가 오염되면 "불러오기" 버튼이 영구히 이동하지 않는다.
- **B-7 자동 복원은 새 비교가 아니다**: `restoreLatestComparison`은 (a) `deps.runReport`(비교 로그 전송 래퍼)가 아니라 로그 없는 경로를 쓰고, (b) `runReport`에 `save:false`를 넘겨 이미 저장된 레코드를 재저장하지 않는다. (a)를 어기면 새로고침마다 익명 쌍 로그가 재전송되어 `GET /comparisons/trending`의 "실시간 비교 TOP 10" 집계가 **실제 비교 없이 부풀고**(FRD 11 §개정 2026-07-14 — 로그는 사용자가 수행한 비교를 뜻한다), (b)를 어기면 `id`·`savedAt`이 새로 발급되어 새로고침만으로 "최근 비교" 목록의 순서·식별자가 요동친다. *(주: INV-1은 API **라우트 표면** 불변식이며 본 규칙과 별개다 — 본 규칙은 클라이언트 전송 시점을 규정한다.)*

## SP-FE-6 — 읽기 전용 API 클라이언트(`apiFetch`, 인증 헤더 0)

**구현**: FR-02·F1·F8 / SP-API-9·10·11·12·13 / NFR16·NFR20.

`api.js`는 GET 4종 중 3종(health 제외)을 소비하는 얇은 클라이언트다. **인증 헤더·쿠키·자격증명을 전송하지 않으며**, 쓰기 메서드를 노출하지 않는다.

### SP-FE-6.1 `apiFetch` 계약

```js
// api.js
const API_BASE = '/api/v1';                       // 동일 오리진(SP-ARCH-2), CORS 불필요
const DEFAULT_TIMEOUT = 8000;                      // ms

export async function apiFetch(path, { signal, timeout = DEFAULT_TIMEOUT } = {}) {
  const ctrl = new AbortController();
  const timer = setTimeout(() => ctrl.abort(), timeout);
  if (signal) signal.addEventListener('abort', () => ctrl.abort());   // 외부 취소(경합 폐기) 연결
  try {
    const res = await fetch(API_BASE + path, {
      method: 'GET',                              // GET 전용 — 쓰기 메서드 없음(NFR20)
      headers: { 'Accept': 'application/json' },  // Authorization·Cookie 미부착(NFR16)
      credentials: 'omit',                        // 자격증명 미전송
      signal: ctrl.signal,
    });
    if (!res.ok) throw new ApiError(res.status, path);   // 4xx/5xx → 구조화 오류
    return await res.json();
  } finally {
    clearTimeout(timer);
  }
}

export class ApiError extends Error {
  constructor(status, path) { super('API ' + status + ' ' + path); this.status = status; this.path = path; }
}

export const getReference    = (opt)        => apiFetch('/reference/all', opt);
export const searchCompanies = (q, opt)     => apiFetch('/companies/search?q=' + encodeURIComponent(q), opt);
export const getCompany      = (id, opt)    => apiFetch('/companies/' + encodeURIComponent(id), opt);
```

### SP-FE-6.2 엔드포인트 소비 표

| 클라이언트 함수 | 엔드포인트 | 소비처(FR) | 캐시/오류 처리 |
| --- | --- | --- | --- |
| `getReference` | `GET /reference/all` | 부팅(FR-02) | 브라우저 캐시 1h. 실패→FR-E1 |
| `searchCompanies` | `GET /companies/search?q=` | 검색(FR-10·11) | `no-store`. 실패(≠200/네트워크)→오류 상태(FR-13). `[]`→무결과 |
| `getCompany` | `GET /companies/{id}` | 선택 상세·프리필(FR-14·16) | REF 인라인 우선; 필요 시만 호출. 404→미선택 유지(FR-D11) |

### SP-FE-7.1 디바운스·경합(search.js)

```js
const DEBOUNCE_MS = 300;                                 // 레거시 doSearch 이식값(FR-10)
```

### SP-FE-7.2 후보 정렬(결정적, FR-12 R3)

```js
function sortCandidates(items, q) {                      // 서버 순서에 의존하지 않음(FR-12)
  const nq = q.toLowerCase();
  return items.slice().sort((x, y) =>
      rank(x, nq) - rank(y, nq)                          // 1 완전일치 <2 정식명 접두 <3 그외
   || x.comp_nm.length - y.comp_nm.length                // 4 길이 오름차순
   || x.comp_nm.localeCompare(y.comp_nm, 'ko'));         // 5 가나다(ko)
}
function rank(it, nq) {
  const n = it.comp_nm.toLowerCase();
  if (n === nq) return 0;
  if (n.startsWith(nq)) return 1;
  return 2;
}
function dedupById(items) {                              // comp_id 방어적 dedup(FR-11 R3)
  const seen = new Set(); const out = [];
  for (const it of items) if (!seen.has(it.comp_id)) { seen.add(it.comp_id); out.push(it); }
  return out;
```

**구현**: FR-45 / NFR21. **데이터를 보간한 `innerHTML` 직접 삽입 금지**. 정적 구조 템플릿만 `innerHTML` 허용하고, 모든 데이터 값은 이스케이프 또는 `textContent`로 삽입한다.

### SP-FE-9.1 `dom.js` 유틸

```js
// dom.js
export function escapeHtml(s) {                          // & < > " ' 치환
  return String(s ?? '').replace(/[&<>"']/g, c => (
    { '&':'&amp;', '<':'&lt;', '>':'&gt;', '"':'&quot;', "'":'&#39;' }[c]));
}

// 안전 DOM 빌더: 텍스트는 textContent, 속성은 setAttribute (자동 이스케이프)
export function el(tag, opts = {}, ...children) {
  const node = document.createElement(tag);
  for (const [k, v] of Object.entries(opts)) {
    if (k === 'text') node.textContent = v;              // ★데이터는 항상 textContent
    else if (k === 'html') throw new Error('el(): raw html 금지');   // 데이터 보간 innerHTML 차단
    else if (k === 'class') node.className = v;
    else if (v != null) node.setAttribute(k, v);
  }
  for (const c of children) node.append(c);
  return node;
}

// 태그드 템플릿: 정적 부분은 그대로, 보간값은 자동 escapeHtml → innerHTML에 안전 사용
export function h(strings, ...values) {
  return strings.reduce((acc, s, i) => acc + s + (i < values.length ? escapeHtml(values[i]) : ''), '');
}

export function safeUrl(u) {                             // 스킴 화이트리스트(http/https만)
  try { const url = new URL(u, location.origin);
        return (url.protocol === 'http:' || url.protocol === 'https:') ? url.href : null; }
  catch { return null; }
}
```

### SP-FE-9.3 엔진 호출·상태 조립(`assembleCompareState` — App.state → `CompareState`)

**구현**: FR-40·FR-42 / SP-ENGINE-1·2(엔진 입력 계약). 엔진(`calc.js`, SP-CALC/SP-ENGINE)은 **UI 상태 형태를 모른다** — 순수 입력 타입 `CompareState`(SP-ENGINE-2)만 받는다. 본 절은 `App.state`(SP-FE-4)를 그 입력으로 **변환하는 유일한 어댑터**를 확정한다. 상태 단일 소유자인 `app.js`가 소유하며, `report` 진입·재계산(FR-42) 시 `compare(assembleCompareState(App.state))`로 호출한다. 이 어댑터가 없으면 두 문서가 **같은 데이터에 서로 다른 필드명·형태**를 써(엔진 `salStr`↔프론트 `salS`, `com`↔`cmtS`, `companyTypes`↔`REF.company_types`, `PriKey`↔한국어 `curPri`) 매핑 불일치가 구현 공백으로 남는다.

```js
    companyTypes: state.REF.company_types,               // 브랜드 축용 참조 주입
  };
}

// 리포트 진입·재계산(FR-42): 조립 → 계산 → 렌더
```

## SP-FE-10 — localStorage(`store.js`)·"최근 비교" 봉투

**구현**: FR-07·FR-43·FR-44 / FRD 06 스키마. **저장소 배관·봉투·무결성**은 본 문서가 소유하고, "최근 비교" **요약 콘텐츠·저장/불러오기 UI**의 렌더는 리포트 렌더(**SP-FE-9.4**)가, `RecentComparison`의 `input`/`result` **필드 구성**은 FR-43(FRD 06)이 규정한다(경계).

### SP-FE-10.1 저수준 래퍼(`store`)

```js
// store.js
export const store = {
  get(key) {
    try { const v = localStorage.getItem(key); return v == null ? null : JSON.parse(v); }
    catch { return null; }                               // 접근 거부·파싱 실패 → null(FR-44)
  },
  set(key, value) {
    try { localStorage.setItem(key, JSON.stringify(value)); return true; }
    catch { return false; }                              // QuotaExceeded·비활성 → 흡수(FR-44)
  },
  remove(key) { try { localStorage.removeItem(key); } catch {} },
  available() { try { const k = '__t'; localStorage.setItem(k,'1'); localStorage.removeItem(k); return true; }
                catch { return false; } },
};
```

### SP-FE-10.2 "최근 비교" 봉투(FR-43 스키마)

localStorage 최상위 키 **`loupit.recentComparisons`** 하나에 봉투 JSON을 둔다.

```
{ "v": 1, "items": [ RecentComparison, ... ] }          // items: 최신순, 길이 ≤ MAX_RECENT
```

```js
// store.js
const RECENT_KEY = 'loupit.recentComparisons';
const RECENT_V = 1;                                      // 봉투 스키마 버전(FR-43 R4)
const MAX_RECENT = 10;                                   // FR-43 R1 (상수로 조정 가능)

export const recent = {
  list() {                                               // 불러오기(손상 폐기)
    const env = store.get(RECENT_KEY);
    if (!isValidEnvelope(env)) { store.remove(RECENT_KEY); return []; }   // R4: 손상/버전불일치 폐기
    return env.items.filter(isValidRecord);              // 개별 손상 레코드도 무시
  },
  save(record) {                                          // 저장(선두 추가·FIFO·dedup)
    if (!store.available()) return false;                // FR-44: 저장 불가 시 생략
    const items = recent.list();
    const sig = signatureOf(record);
    const idx = items.findIndex(r => signatureOf(r) === sig);
    if (idx >= 0) items.splice(idx, 1);                  // R3: 동일 시그니처 제거(갱신·선두 이동)
    items.unshift(record);                               // 신규를 선두
    while (items.length > MAX_RECENT) items.pop();       // R2: 초과 시 말미(가장 오래된) 축출
    return store.set(RECENT_KEY, { v: RECENT_V, items });
  },
  removeById(id) {
    const items = recent.list().filter(r => r.id !== id);
    store.set(RECENT_KEY, { v: RECENT_V, items });
  },
  clear() { store.remove(RECENT_KEY); },                 // 전체 삭제
};

function isValidEnvelope(env) {
  return env && env.v === RECENT_V && Array.isArray(env.items);   // R4: v 불일치 → 무효
}
function isValidRecord(r) {                              // 필수 필드 검증(없으면 폐기)
  return r && typeof r.id === 'string' && typeof r.savedAt === 'string'
          && r.slots && r.input && r.result;
}
function signatureOf(r) {                                // R3: 양 슬롯 회사(또는 직접입력) + 핵심 입력
  const sid = s => (r.slots[s] && r.slots[s].comp_id != null ? 'c' + r.slots[s].comp_id : 'direct');
  return sid('a') + '|' + sid('b') + '|' + (r.result.priAxis || '');
}
```

## SP-FE-11 — URL 파라미터 프리필(정적 페이지 CTA → 비교 툴)

**구현**: FR-16 / FLOW SCR-52·55 / SP-GEN(생성 측 소유, 본 문서는 소비).

정적 회사 상세(F4)·인기 조합(F5) 페이지의 CTA가 넘긴 URL 파라미터를 부팅 시 읽어 슬롯을 사전 지정한다. **CTA 링크 생성은 SP-GEN**이 소유하고, 본 절은 **소비(파싱·해석·슬롯 반영)**만 규정한다.

### SP-FE-11.1 파라미터 계약

| 파라미터 | 의미 | 예 | 해석 |
| --- | --- | --- | --- |
| `a` | 슬롯 a 회사 지시자 | `?a=42` / `?a=samsung` / `?a=삼성전자` | comp_id(정수)→REF 매칭 / comp_eng_nm / comp_nm / 별칭 순 |
| `b` | 슬롯 b 회사 지시자 | `?b=17` | 동일 |
| `slot` | 단일 회사 진입 시 대상 슬롯 | `?a=42&slot=a` | 기본 `a`(현직) |

- 회사 상세 CTA(SCR-52): `/compare?a={comp_eng_nm|comp_id}`(기본 슬롯 a). 인기 조합(SCR-55): `/compare?a={..}&b={..}`.

### SP-FE-11.2 해석 알고리즘

```js
// app.js
function restoreFromPrefill() {
  const p = new URLSearchParams(location.search);
  for (const slot of ['a', 'b']) {
    const token = p.get(slot);
    if (!token) continue;
    const comp = resolveCompanyToken(token);             // REF 우선 해석
    if (comp) { App.state.matched[slot] = normalizeCompany(comp); fillBenefits(slot); initWsState(slot); reflectSlotLabel(slot, comp.comp_nm); }
    // 해석 실패 시 슬롯 미선택 유지(정상 검색 진입으로 폴백, FR-16 무효 지시자)
  }
  if (App.state.matched.a || App.state.matched.b) go('input', { push: false });  // 프리필 있으면 입력 뷰
}

function resolveCompanyToken(token) {
  const cs = App.state.REF.companies;
  if (/^\d+$/.test(token)) return cs.find(c => c.comp_id === Number(token)) || null;   // comp_id
  const t = token.toLowerCase();
  return cs.find(c => (c.comp_eng_nm || '').toLowerCase() === t)                        // 영문 식별자
      || cs.find(c => c.comp_nm === token)                                              // 정식명 완전일치
      || cs.find(c => (c.aliases || []).includes(token))                               // 별칭
      || null;
}
```
