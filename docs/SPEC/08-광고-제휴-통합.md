# 08. 광고·제휴 통합 — SP-ADS

**영역 ID 대역**: `SP-ADS`
**저장 경로**: `docs/SPEC/08-광고-제휴-통합.md`
**목적**: F6(광고/제휴 슬롯)을 개발자가 추측 없이 그대로 구현할 수 있는 수준으로 확정한다. (1) AdSense 설정 단일 주입 모듈(`adsConfig` — client id 플레이스홀더·슬롯 id 상수), (2) `page_type` 게이팅 결정 함수(`adPolicy`)와 마운트 오케스트레이터(`mountAds`), (3) 자동광고 스니펫 조건부 주입(ON=landing/company/combo, OFF=input/result/policy), (4) 수동 광고 슬롯 컴포넌트(예약 높이 CLS·"광고" 라벨·미노출 폴백), (5) 제휴 `affiliate.json` 스키마와 데이터 기반 제휴 카드 렌더(active·page_types 필터·이스케이프·`rel="sponsored nofollow noopener"`·http/https 스킴 제한), (6) 광고·쿠키 개인화 동의 배너 로직(localStorage/쿠키 상태·비개인화 폴백·서버 무전송), (7) 공정위 "광고" 표기 규칙, (8) 정적 페이지(F4/F5) 통합 DOM 계약, (9) 테스트 명세를 소유한다.

**상위 추적**: CONTEXT_BRIEF `§2-5`(수익화=AdSense+제휴·"광고" 표기)·`§2-6`(배치=콘텐츠 페이지 위주·입력 무광고·결과 절제)·`§2-2`(사용자 상태 localStorage 한정)·`§8`(AdSense 스니펫·플레이스홀더 client id·`affiliate.json`), FRD `09-광고-제휴.md`(**FR-70~FR-79**, 배치 표 정본), PRD `06-수익화.md`(**MON1~MON19**, 특히 MON3~MON18), PRD `07-비기능-요구사항.md`(NFR2·NFR5·NFR16·NFR17·NFR18·NFR19·NFR21·NFR22·NFR25·NFR29), USECASE `07-광고와정책.md`(UC-60·UC-61·UC-62), WIREFRAME `01-공통-레이아웃.md`(SCR-61 동의 배너·SCR-62 광고 슬롯/제휴 카드·§0 예약 높이·§5.1 배치 표), SPEC `01-개요와-아키텍처.md`(**SP-ARCH-6** 디렉토리 트리 `web/assets/js/ads.js`·`web/assets/data/affiliate.json`·**SP-ARCH-10** 불변식), SPEC `06-프론트엔드-구조.md`(**SP-FE-1** 모듈 그래프·`mountAds` 마운트 지점·**SP-FE-5** `bindGlobalUI`·**SP-FE-9** `escapeHtml`/`safeUrl`/`el`·**SP-FE-10** `store` localStorage 래퍼), SPEC `05-비교-계산엔진.md`(리포트 하단 슬롯 마운트 지점), SPEC `11-디자인-토큰`(SP-DSN, `styles.css` 색/타이포 토큰 — 본 문서는 클래스·예약 높이 계약만 소유).

**범위 경계**: 본 문서는 **광고·제휴 컴포넌트의 동작(설정 주입·게이팅·자동/수동 슬롯·제휴 렌더·표기·링크 속성·동의·폴백)과 그 DOM 계약**을 소유한다.
1. **색·타이포·간격 등 시각 토큰과 최종 스타일**은 SP-DSN(`styles.css`)이 소유한다. 본 문서는 **CSS 클래스명·예약 높이(min-height) 규격·"광고" 라벨 크기 하한**만 기능 계약으로 확정하고 실제 색·polish는 재정의하지 않는다.
2. **광고가 얹히는 정적 페이지 자체의 생성·본문·SEO 메타**는 SP-GEN(F4/F5, 회사 상세·인기 조합)이 소유한다. 본 문서는 그 페이지가 광고를 마운트하기 위해 **방출해야 하는 DOM 계약**(SP-ADS-9)만 규정하고 페이지 본문을 작성하지 않는다.
3. **정책 페이지 4종의 본문·게시**는 SP-POL(F7)이 소유한다. 본 문서는 동의 배너가 정책 페이지로 **링크**하는 연결(MON14)과 `policy` 무광고 게이팅만 규정한다.
4. **SPA 셸·부팅·라우팅·상태·localStorage 래퍼·렌더 안전 유틸**은 SP-FE(06)가 소유한다. 본 문서는 SP-FE가 제공하는 `store`(SP-FE-10)·`escapeHtml`/`safeUrl`/`el`(SP-FE-9)를 **소비**하고, `mountAds`/`initConsentBanner`를 SP-FE의 `bindGlobalUI`·뷰 전환 지점에 **마운트**한다.
5. **KPI 정의**는 `02-목표와-성공지표.md`가 소유(MON19), 본 문서는 중복 정의하지 않는다.
6. 로그인/회원/프로파일러/서버측 사용자 쓰기는 영구 제외이며 어떤 광고·제휴·동의 경로에도 없다(FR-01). 동의를 포함한 모든 상태는 브라우저(localStorage/쿠키)에만 존재하고 서버로 전송·저장하지 않는다(INV-4·NFR16·NFR17).

**전역 불변식(SP-ARCH-10 상속)**: INV-4(사용자 상태·동의는 클라이언트/localStorage에만, 서버 전송 0) · INV-1(서버 쓰기 엔드포인트 없음) · NFR21(데이터 문자열 무이스케이프 `innerHTML` 금지, `escapeHtml`/`textContent`) · **INV-ADS**(광고·제휴 컴포넌트의 마운트 여부는 오직 `page_type` 하나로 결정되며 판별 실패 시 무광고가 안전 기본값이다).

## SP-ADS-1 — 모듈 구성·공개 인터페이스

**구현**: FR-70·FR-71·FR-73 / SP-ARCH-6(트리) / SP-FE-1(모듈 그래프).

SP-ARCH-6 트리의 `web/assets/js/ads.js`(광고·제휴·게이팅·동의)를 정본으로 하고, 단일 설정을 관심사 분리로 co-located 서브모듈 `web/assets/js/adsConfig.js`에 둔다(SP-FE-1이 확립한 "app.js 하위 모듈" 패턴과 동일 근거 — 무빌드 ES 모듈, 단일 진실·단위 테스트 용이). 제휴 데이터는 정적 자원 `web/assets/data/affiliate.json`(SP-ARCH-6)이다.

### SP-ADS-1.1 파일·심볼 대응표

| 파일 | 역할 | 공개 심볼(대표) | 소유 |
| --- | --- | --- | --- |
| `web/assets/js/adsConfig.js` | **설정 단일 진실**: client id·슬롯 id 플레이스홀더·정책 플래그 | `adsConfig`, `isPlaceholder`, `AD_SLOT`, `SLOT_ID_MAP`, `SLOT_RESERVE` | 본 문서 SP-ADS-2 |
| `web/assets/js/ads.js` | **광고·제휴·동의 오케스트레이터**(부수효과 O: DOM/네트워크) | `mountAds`, `adPolicy`, `filterAffiliate`, `resolveSlotId`, `loadAffiliate`, `initConsentBanner`, `getConsent`, `setConsent`, `isPersonalized`, `parseConsent`, `detectPageType` | 본 문서 SP-ADS-3~8 |
| `web/assets/data/affiliate.json` | 제휴 항목 데이터(코드 무하드코딩) | — | 본 문서 SP-ADS-6(스키마) |

- **의존 방향**(비순환): `adsConfig.js`는 아무것도 import하지 않는다(순수 상수·순수 판별 함수). `ads.js`는 `adsConfig.js`와 SP-FE의 `dom.js`(`escapeHtml`·`safeUrl`·`el`, SP-FE-9)·`store.js`(`store`, SP-FE-10)만 import한다. `ads.js`는 `app.js`·계산·리포트 모듈을 import하지 않는다(호출은 상위에서 주입).
- **순수 서브셋(node:test 대상)**: `adPolicy`·`resolveSlotId`·`filterAffiliate`·`parseConsent`·`isPlaceholder`·`buildAffiliateAttrs`는 DOM/네트워크를 참조하지 않는 순수 함수로 분리하여 Node에서 직접 import·검증한다(SP-ADS-10). DOM/네트워크를 만지는 `mountManualSlot`·`ensureLoader`·`renderAffiliateCard`·`initConsentBanner`는 수동 브라우저 체크리스트로 검증한다.
- **추적**: FR-70·FR-73, SP-ARCH-6, SP-FE-1·9·10.

## SP-ADS-2 — AdSense 설정 단일 주입(`adsConfig`)

**구현**: **FR-70** / MON5·MON18 / NFR22 / 브리프 §8. client id·슬롯 id를 **단 한 곳**에서 플레이스홀더 상수로 관리하고, 승인·발급 후 이 파일만 실값으로 치환하면 전 슬롯에 반영된다. 슬롯별 HTML/JS에 리터럴 산재를 금지한다.

### SP-ADS-2.1 `adsConfig.js`

```js
// web/assets/js/adsConfig.js — 설정 단일 진실(FR-70). 승인 후 이 파일만 치환.
export const AD_CLIENT_PLACEHOLDER = 'ca-pub-XXXXXXXXXXXXXXXX';   // ca-pub- + 16자리
export const AD_SLOT_PLACEHOLDER   = 'XXXXXXXXXX';                // 10자리 슬롯 id

export const adsConfig = {
  // 승인 전: 플레이스홀더 유지 → 실제 광고 스크립트 미주입, 예약 슬롯만 렌더(MON18·FR-74)
  AD_CLIENT: AD_CLIENT_PLACEHOLDER,          // 승인 후 'ca-pub-1234567890123456'로 치환
  AUTO_ADS: true,                            // ON 페이지 자동광고 스니펫 사용(MON3)
  // 미동의/거부 폴백 정책(FR-79): 'nonpersonalized'(기본) | 'suppress'(광고 미노출)
  DENY_FALLBACK: 'nonpersonalized',
  AD_SLOT: {                                 // 수동 슬롯 식별자(FR-72). 초기 전부 플레이스홀더
    landing_bottom: AD_SLOT_PLACEHOLDER,
    company_mid:    AD_SLOT_PLACEHOLDER,
    company_bottom: AD_SLOT_PLACEHOLDER,
    combo_mid:      AD_SLOT_PLACEHOLDER,
    combo_bottom:   AD_SLOT_PLACEHOLDER,
    result_bottom:  AD_SLOT_PLACEHOLDER,
  },
};

// (page_type, position) → AD_SLOT 키 매핑. 배치 표(정본)와 1:1(FR-72·FR-73)
export const SLOT_ID_MAP = {
  landing: { content_bottom: 'landing_bottom' },
  company: { content_mid: 'company_mid', content_bottom: 'company_bottom' },
  combo:   { content_mid: 'combo_mid',   content_bottom: 'combo_bottom' },
  result:  { report_bottom: 'result_bottom' },
};

// 예약 높이 규격(CLS, FR-74). 색·여백은 SP-DSN; 여기서는 min-height 하한만 계약
export const SLOT_RESERVE = {               // px, {mobile, desktop}
  content_mid:    { mobile: 280, desktop: 280 },
  content_bottom: { mobile: 250, desktop: 250 },
  report_bottom:  { mobile: 250, desktop: 250 },
};

export const AD_LABEL_TEXT = '광고';        // 공정위 표기(FR-76). 대체: '유료 광고'

// 승인 전 판별: client id에 플레이스홀더 문자('X')가 남아 있으면 미승인 취급
export function isPlaceholder(id) { return !id || /X{4,}/i.test(id); }
```

### SP-ADS-2.2 규칙표

| 규칙 | 내용 | 근거 |
| --- | --- | --- |
| C-1 | client id 리터럴은 저장소·정적 산출물 전역에서 `adsConfig.js` **1곳에만** 존재(산재 0). 자동광고 스니펫(SP-ADS-4)·수동 슬롯(SP-ADS-5)은 `adsConfig.AD_CLIENT`만 참조. | FR-70·MON5 |

| C-3 | `isPlaceholder(AD_CLIENT)===true`인 승인 전에는 실제 로더 스크립트를 **주입하지 않고** 예약 슬롯 컨테이너만 렌더한다(MON18·FR-74). | FR-70·MON18 |

## SP-ADS-3 — `page_type` 게이팅 결정(`adPolicy`)·마운트(`mountAds`)

**구현**: **FR-73**(정본 배치 표)·FR-71·FR-72·FR-75 / MON7·MON8·MON9. 자동광고·수동 슬롯·제휴 컴포넌트의 마운트를 **오직 `page_type` 하나**로 게이팅한다(INV-ADS). 판별 실패는 무광고(안전 기본값).

### SP-ADS-3.1 결정 함수(순수, node:test 대상)

```js
// ads.js — 배치 표(FRD 09 정본)와 1:1. 창작·추가 배치 금지.
export const PAGE_TYPES = ['landing', 'company', 'combo', 'input', 'result', 'policy'];

export function adPolicy(pageType) {
  switch (pageType) {
    case 'landing': return { auto: 'ON',  manual: ['content_bottom'],               affiliate: 'optional' };
    case 'company': return { auto: 'ON',  manual: ['content_mid', 'content_bottom'], affiliate: 'on' };
    case 'combo':   return { auto: 'ON',  manual: ['content_mid', 'content_bottom'], affiliate: 'on' };
    case 'result':  return { auto: 'OFF', manual: ['report_bottom'],                affiliate: 'optional' };
    case 'input':   return { auto: 'OFF', manual: [],                               affiliate: 'none' }; // 무광고 강제
    case 'policy':  return { auto: 'OFF', manual: [],                               affiliate: 'none' };
    default:        return { auto: 'OFF', manual: [],                               affiliate: 'none' }; // 판별 실패=안전 기본값
  }
}
export function resolveSlotId(pageType, position) {
  const key = (SLOT_ID_MAP[pageType] || {})[position];   // 매핑 없으면 undefined → 슬롯 미렌더
  return key ? adsConfig.AD_SLOT[key] : null;
}
```

### SP-ADS-3.2 페이지 유형 판별(`detectPageType`)

```js
// 정적 페이지: <body data-page-type="company"> 에서 읽음(SP-ADS-9). SPA: 호출부가 명시 인자 전달.
export function detectPageType() {
  const t = (document.body && document.body.dataset && document.body.dataset.pageType) || null;
  return PAGE_TYPES.includes(t) ? t : 'default';   // 미상 → 'default'(무광고)
}
```

### SP-ADS-3.3 마운트 오케스트레이터

```js
export function mountAds(pageType = detectPageType(), root = document) {
  const policy = adPolicy(pageType);
  // input·policy·default: 자동·수동·제휴 전부 0 마운트(게이팅 자체 차단, MON8)
  if (policy.auto === 'OFF' && policy.manual.length === 0 && policy.affiliate === 'none') return;

  applyConsentSignal();                                  // 비개인화 신호를 로더 실행 전에 세팅(FR-79)
  const needLoader = policy.auto === 'ON' || policy.manual.length > 0;
  if (needLoader) ensureLoader();                        // 승인 전(isPlaceholder)엔 no-op(예약만)
  if (policy.auto === 'ON') enableAutoAds();

  for (const position of policy.manual) mountManualSlot(root, pageType, position);
  if (policy.affiliate !== 'none') mountAffiliate(root, pageType);
}
```

### SP-ADS-3.3 규칙표

| 규칙 | 내용 | 근거 |
| --- | --- | --- |
| G-1 | `adPolicy` 반환값은 배치 표(FRD 09·WIREFRAME §5.1)와 **1:1**. 표 외 위치·개수 추가 금지. | FR-73·MON7 |
| G-2 | `input`·`policy`·`default`는 `mountAds`가 **즉시 반환**하여 광고·제휴 DOM 노드 0·로더 스크립트 0. | FR-73·MON8·UC-60(1a)·UC-61(1b) |
| G-3 | `result`의 수동 슬롯은 `['report_bottom']` **정확히 1개**(리포트 판정 하단). 상단·중간 삽입 금지. | FR-72·MON9·UC-60 |
| G-4 | 판별 실패(`detectPageType`→`'default'`, 미지 값)는 광고 노출이 아니라 무광고로 폴백. | FR-73·INV-ADS |
| G-5 | `mountAds`는 뷰/페이지당 1회 호출(중복 슬롯·중복 로더 방지). `ensureLoader`는 내부적으로 로더 1회만 주입. | FR-71 |

- **SPA 마운트 지점**(SP-FE-3 뷰 전환): `input`·`search` 뷰에서는 호출하지 않거나 `mountAds('input')`(no-op). `report` 뷰 진입 성공 시 `mountAds('result')` 1회. 정적 페이지는 SP-ADS-9의 인라인 스크립트가 `mountAds()` 1회.
- **추적**: FR-71·72·73·75, MON7·MON8·MON9, SP-FE-3·5.

## SP-ADS-4 — 자동광고(Auto ads) 조건부 로드

**구현**: **FR-71** / MON3·MON7·MON8. 자동광고 스니펫을 사이트 전역이 아니라 `page_type`이 ON(landing/company/combo)인 페이지에만, 정적 본문 색인 **이후 클라이언트 스크립트로** 주입한다(비-JS 본문 무훼손, 브리프 §2-7).

### SP-ADS-4.1 로더·자동광고 활성화

```js
const LOADER_ID = 'adsbygoogle-loader';
const LOADER_SRC = 'https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js';

function ensureLoader() {
  if (isPlaceholder(adsConfig.AD_CLIENT)) return;                 // 승인 전: 로더 미주입(C-3)
  if (document.getElementById(LOADER_ID)) return;                 // 1회만(G-5)
  const s = document.createElement('script');
```

```js
function mountManualSlot(root, pageType, position) {
  const host = root.querySelector('[data-ad-position="' + position + '"]');   // SP-ADS-9 계약 위치
```

## SP-ADS-6 — 제휴 `affiliate.json` 스키마·제휴 카드 렌더

**구현**: **FR-75·FR-77** / NFR21·NFR29 / MON10·MON12 / SCR-62(6장) / §8. 제휴 항목을 코드가 아니라 `affiliate.json`에서만 읽어 렌더한다. 노출/미노출/링크 교체는 데이터만으로 제어(코드 무수정).

### SP-ADS-6.1 `affiliate.json` 스키마(정본)

| 필드 | 타입 | 제약 | 의미 |
| --- | --- | --- | --- |
| `version` | integer | ≥ 1 | 스키마 버전 |
| `items` | array | 0개 이상 | 제휴 항목 목록 |
| `items[].id` | string | 고유(unique)·kebab-case | 항목 식별자 |
| `items[].label` | string | 비어있지 않음·한국어 | 표시 라벨 |
| `items[].url` | string | `http`/`https` 스킴 필수·외부 링크 | 제휴 대상 URL |
| `items[].desc` | string \| null | 선택 | 보조 설명 |
| `items[].page_types` | string[] | 값 ∈ {`landing`,`company`,`combo`,`result`}, `input`·`policy` **불가** | 노출 대상 페이지 유형 |
| `items[].active` | boolean | — | `false`면 미노출 |

```json
{
  "version": 1,
  "items": [
    {
      "id": "resume-review",
      "label": "이력서 첨삭 서비스",
      "url": "https://partner.example.com/resume?ref=loupit",
      "desc": "이직 준비 관련 서비스",
      "page_types": ["company", "combo"],
      "active": true
    }
  ]
}
```

- 초기 배포 `affiliate.json`은 `{ "version": 1, "items": [] }`(빈 목록)로 시작할 수 있다(제휴 미확보 시 제휴 카드 미노출, 레이아웃 무영향).

### SP-ADS-6.2 로드·필터·렌더

```js
let _affiliateCache = null;
export async function loadAffiliate() {
  if (_affiliateCache) return _affiliateCache;
  try {
    const res = await fetch('/assets/data/affiliate.json', { credentials: 'omit' });   // 동일 오리진·무자격증명
    const data = await res.json();
    _affiliateCache = (data && Array.isArray(data.items)) ? data : { version: 1, items: [] };
  } catch { _affiliateCache = { version: 1, items: [] }; }                              // 실패 → 빈 목록(무크래시)
  return _affiliateCache;
}

// 순수 필터(node:test 대상): active && page_types 포함 && input/policy 아님 && url 스킴 유효
export function filterAffiliate(items, pageType) {
  if (pageType === 'input' || pageType === 'policy') return [];        // 게이팅(FR-73)
  const seen = new Set();
  return (items || []).filter(it => {
    if (!it || it.active !== true) return false;
    if (!Array.isArray(it.page_types) || !it.page_types.includes(pageType)) return false;
    if (it.page_types.includes('input') || it.page_types.includes('policy')) return false; // 위반 항목 배제
    if (typeof it.label !== 'string' || it.label.trim() === '') return false;
    if (safeUrl(it.url) == null) return false;                          // http/https만(NFR21)
    if (typeof it.id !== 'string' || seen.has(it.id)) return false;     // id 중복 배제
    seen.add(it.id);
    return true;
  });
}

// 순수 속성 빌더(node:test 대상): 광고성·안전 링크 속성(FR-77)
export function buildAffiliateAttrs(url) {
  return { href: safeUrl(url), rel: 'sponsored nofollow noopener', target: '_blank' };
}

async function mountAffiliate(root, pageType) {
  const host = root.querySelector('[data-affiliate-host]');            // SP-ADS-9 계약 위치
  if (!host) return;
  const { items } = await loadAffiliate();
  const list = filterAffiliate(items, pageType);
  host.replaceChildren();
  if (list.length === 0) return;                                       // 해당 없음 → 컴포넌트 미노출
  for (const it of list) host.append(renderAffiliateCard(it));
}

function renderAffiliateCard(it) {
  const card = el('div', { class: 'affiliate-card' });
  card.append(el('span', { class: 'ad-label', text: AD_LABEL_TEXT }));           // FR-76
  card.append(el('div', { class: 'affiliate-label', text: it.label }));          // 이스케이프(textContent)
  if (it.desc) card.append(el('div', { class: 'affiliate-desc', text: it.desc }));// null이면 생략
  const attrs = buildAffiliateAttrs(it.url);
  const a = el('a', { class: 'affiliate-cta', href: attrs.href, rel: attrs.rel, target: attrs.target, text: '바로가기 →' });
  card.append(a);
  return card;
}
```

## SP-ADS-7 — 광고·쿠키 개인화 동의(배너·상태·비개인화 폴백)

**구현**: **FR-78·FR-79** / NFR16·NFR17·NFR18·NFR25 / MON13·MON14·MON15 / SCR-61 / UC-62. 첫 방문 시 동의 배너로 광고/쿠키 목적·제3자(Google)를 고지하고 정책 페이지로 링크한다. 동의 상태는 **서버 저장 없이** 브라우저(localStorage/쿠키)에만 보관한다. 미동의/거부는 비개인화 광고 또는 미노출로 폴백하며 사이트 이용을 막지 않는다.

### SP-ADS-7.1 동의 상태 저장(localStorage 우선·쿠키 폴백)

```js
const CONSENT_KEY = 'loupit.adConsent';
const CONSENT_V = 1;

// 반환: true(개인화 동의) | false(거부) | null(미선택)
export function getConsent() {
  const parsed = parseConsent(store.get(CONSENT_KEY));        // store = SP-FE-10 localStorage 래퍼
  if (parsed !== null) return parsed;
  const c = readCookie(CONSENT_KEY);                          // 폴백: 1st-party 쿠키
  if (c === 'p') return true;
  if (c === 'n') return false;
  return null;
}
export function setConsent(personalized) {
  const ok = store.set(CONSENT_KEY, { v: CONSENT_V, personalized: !!personalized, ts: Date.now() });
  if (!ok) writeCookie(CONSENT_KEY, personalized ? 'p' : 'n', 365);   // localStorage 불가 시 쿠키
}
// 순수 파서(node:test 대상): 봉투 검증. 손상·버전불일치 → null
export function parseConsent(env) {
  if (env && env.v === CONSENT_V && typeof env.personalized === 'boolean') return env.personalized;
  return null;
}
export function isPersonalized() { return getConsent() === true; }

// 로더 실행 전 비개인화 신호 세팅(FR-79): 미동의/거부 → requestNonPersonalizedAds=1
function applyConsentSignal() {
  const ab = (window.adsbygoogle = window.adsbygoogle || []);
  ab.requestNonPersonalizedAds = isPersonalized() ? 0 : 1;
}
```

### SP-ADS-7.2 동의 배너 배선(`initConsentBanner`)

```js
// 배너 마크업(SCR-61)은 셸/정적 페이지가 방출(SP-ADS-9). 본 함수는 로직만.
export function initConsentBanner() {
  const banner = document.getElementById('consent-banner');
  if (!banner) return;
  if (getConsent() !== null) { banner.hidden = true; return; }        // 이미 결정 → 미노출(재방문)
  banner.hidden = false;                                              // 첫 방문 → 노출
  banner.querySelector('[data-consent="grant"]')?.addEventListener('click', () => decideConsent(true));
  banner.querySelector('[data-consent="deny"]') ?.addEventListener('click', () => decideConsent(false));
}
function decideConsent(personalized) {
  setConsent(personalized);
  const banner = document.getElementById('consent-banner');
  if (banner) banner.hidden = true;
  // 이번 세션 광고는 이미 비개인화로 로드됨. 개인화 전환은 다음 로드에 적용(FR-79 재방문 적용).
}
```

## SP-ADS-8 — 공정위 "광고" 표기(표시광고 명확성)

**구현**: **FR-76** / NFR19 / MON11·MON18 / §2-5. 모든 광고(AdSense 슬롯)·제휴 영역에 사용자가 인지 가능한 "광고"(또는 "유료 광고") 라벨을 부착하여 콘텐츠와 명확히 구분한다.

| 규칙 | 내용 | 근거 |
| --- | --- | --- |

## SP-ADS-9 — 정적 페이지·SPA 통합 DOM 계약

**구현**: FR-71·72·75·78 / SP-GEN 경계 / SP-FE 경계. 광고·제휴·동의 배너를 마운트하려면 호스트 페이지가 아래 DOM을 방출해야 한다. **본 문서는 이 계약(속성·id·클래스)을 소유**하고, 정적 페이지의 실제 방출은 SP-GEN(templates), SPA 셸 방출은 SP-FE가 수행한다.

### SP-ADS-9.1 호스트 DOM 계약

| 계약 요소 | 형식 | 방출 주체 | 용도 |
| --- | --- | --- | --- |
| 페이지 유형 | `<body data-page-type="company">` (값 ∈ PAGE_TYPES) | 정적: SP-GEN / SPA: 셸 또는 명시 인자 | `detectPageType`(SP-ADS-3.2) |
| 수동 슬롯 위치 | `<div data-ad-position="content_mid">`(빈 컨테이너) | SP-GEN(정적)/SP-FE(리포트) | `mountManualSlot` 대상 |
| 제휴 호스트 | `<div data-affiliate-host>`(빈 컨테이너) | SP-GEN/SP-FE | `mountAffiliate` 대상 |
| 동의 배너 | `#consent-banner`[hidden] + `[data-consent="grant"]`·`[data-consent="deny"]` + 정책 링크(`/privacy`·`/ads`) | 셸/SP-GEN(공통 파셜) | `initConsentBanner` |
| 광고 스크립트 마운트 | 아래 인라인 모듈 스크립트 | SP-GEN(정적 페이지) | 정적 페이지에서 광고 활성 |

### SP-ADS-9.2 정적 페이지 인라인 마운트(SP-GEN이 삽입)

```html
<!-- 정적 회사/조합 페이지 하단(SP-GEN 템플릿). 본문 색인 이후 광고 주입(§2-7) -->
<script type="module">
  import { mountAds, initConsentBanner } from '/assets/js/ads.js';
  initConsentBanner();     // 동의 배너(전역)
  mountAds();              // page_type = <body data-page-type>에서 자동 판별
</script>
```

- 배치 표대로 정적 페이지는 각 `data-ad-position` 컨테이너와 `data-affiliate-host`를 **본문 마크업 안에 예약 높이로** 미리 둔다(CLS). 컨테이너 위치·개수는 배치 표(정본)를 초과하지 않는다.
- **SPA 통합**: `report` 뷰의 `#view-report` 안에 `data-ad-position="report_bottom"`(판정 vdCard 하단) 1개와 선택적 `data-affiliate-host`를 두고, 뷰 진입 성공 시 `mountAds('result')`. `input`·`search` 뷰에는 어떤 `data-ad-*` 컨테이너도 두지 않는다(무광고 강제).
- **추적**: FR-71·72·75·78, SP-GEN·SP-FE 경계, §2-6·§2-7.
