# 10. 디자인 토큰·스타일 시스템 — SP-DS

**영역 ID 대역**: `SP-DS`
**저장 경로**: `docs/SPEC/10-디자인-토큰.md`
**목적**: loupit 프론트(무빌드 바닐라 CSS)의 **디자인 토큰 계층**을 개발자가 추측 없이 `web/assets/css/styles.css`의 `:root` 커스텀 속성으로 그대로 옮겨 담을 수 있는 수준으로 확정한다. 색 팔레트(다크 기본), 타입 스케일(Pretendard), 간격·radius·브레이크포인트·애니메이션 토큰, 배지/광고 라벨 색, 그리고 접근성(대비 AA) 계약을 소유한다. **옵션1 방침**에 따라 본 문서는 **토큰만** 정의하며, 픽셀 단위 hi-fi 시안·컴포넌트 최종 비주얼은 만들지 않는다. 실제 비주얼 polish(여백 미세조정·그림자·상태 스타일·색 톤 튜닝)는 **개발 착수 후 실제 앱을 보며 `styles.css`에서 반복**한다(SP-DS-1 R3).

**상위 추적**: CONTEXT_BRIEF `§8`(디자인 접근 옵션1 — 토큰만·다크·polish 후속·무빌드·Pretendard), `§2`(무로그인·무쓰기), WIREFRAME `01-공통-레이아웃.md`(§0 반응형·토큰 규약: 브레이크포인트 768·본문 max-width 960·본문 16/1.6·라벨 13·"광고" 라벨 11~12·CLS 예약, §7 배지 유형 카탈로그 official/est/만료, §8 버튼 위계, §9 폼 상태), PRD `07-비기능-요구사항.md`(**NFR2** LCP/INP/CLS, **NFR5** 폰트·슬롯 CLS·`font-display`, **NFR13** WCAG 2.1 AA·landmark, **NFR15** 본문 대비 ≥4.5:1, **NFR19** "광고" 표기), SPEC `01-개요와-아키텍처.md`(**SP-ARCH-6** 디렉토리 `web/assets/css/styles.css`·`web/assets/fonts/` Pretendard self-host, **SP-ARCH-8** ID-대역 지도), SPEC `06-프론트엔드-구조.md`(**SP-FE-1** `styles.css`=SP-DS 소유·`<head>` 폰트·CSS 우선 로드, **SP-FE-2** SPA 셸 클래스), SPEC 광고·제휴(**SP-ADS-5.2** `.ad-label` 폰트 하한 11px·`.ad-slot` 예약 높이 규격, `.ad-slot--content-mid` 280px·`--content-bottom`/`--report-bottom` 250px), SPEC 정적 생성기(SP-GEN, `base.html` `<head>` 폰트 preload·`font-display: swap`).

**범위 경계**: 본 문서는 **토큰(값)과 그 이름·의미·`:root` 구조**만 소유한다. 각 컴포넌트의 **최종 스타일 규칙(선택자·레이아웃·상태 시각화)** 은 해당 SPEC이 소유하고 본 문서 토큰을 **소비**한다 — 광고·제휴 컴포넌트 클래스 계약·예약 높이는 **SP-ADS**(본 문서는 그 클래스에 쓸 색·폰트 토큰만 제공), SPA 셸·뷰 클래스·라우팅은 **SP-FE**, 리포트 판정카드·배지 표시 마크업은 **SP-RPT**, 정적 페이지 본문 마크업·`base.html` `<head>`는 **SP-GEN**, 정책 페이지 본문은 **SP-POL**. 순수 계산(밴드 계수·판정)은 **SP-CALC**이며 본 문서는 그 결과를 **표시할 색·라벨 색만** 정의한다. 로그인/회원/프로파일러/서버 측 사용자 쓰기 UI는 영구 제외이므로 관련 토큰(아바타·계정 배지 등)을 정의하지 않는다(FR-01).

**대역·파일 번호 정합(작성 지시 vs SP-ARCH-8)**: SP-ARCH-8 ID-대역 지도는 본 문서를 대역 `SP-DSN`·파일 `11-디자인토큰.md`로 표기한다. 본 문서는 작성 지시에 따라 **대역 `SP-DS`·파일 `10-디자인-토큰.md`** 로 발행한다. **`SP-DS` ≡ `SP-DSN`**(동일 문서의 별칭)이며, 형제 SPEC(SP-FE·SP-ADS·SP-GEN·SP-ARCH 등)이 인용하는 **"SP-DSN(`styles.css` 색/타이포 토큰)"은 전부 본 문서를 가리킨다.** 형제 문서 파일 번호도 SP-ARCH-8 인덱스에서 드리프트했으므로(예: 광고=파일 08), 상호 인용은 **파일 번호가 아닌 안정 ID 대역**(SP-DS/SP-FE/SP-ADS…)으로 해석한다.

### SP-DS-1.1 원칙표

| ID | 원칙 | 내용 | 근거 |
| --- | --- | --- | --- |

### SP-DS-1.2 `:root` 계층 구조(개요)

`styles.css`는 아래 순서의 블록으로 구성한다. 상세 값은 각 SP-DS-N 절, 전체 스니펫은 SP-DS-10.

```
web/assets/css/styles.css
├─ @font-face (Pretendard self-host, woff2)      … SP-DS-4
├─ :root { … 토큰 단일 선언 … }
│    ├─ 1) 원시 색 팔레트  --bg-0..4 / --t1..4 / --blue..gold(+dim)   … SP-DS-2
│    ├─ 2) 시맨틱 색 별칭  --accent / --link / --border / --focus …    … SP-DS-2.4
│    ├─ 3) 상태·배지·광고 색  --badge-* / --ad-label-fg …               … SP-DS-3
│    ├─ 4) 타이포 --font-sans / --fs-* / --lh-* / --fw-*               … SP-DS-4
│    ├─ 5) 간격 --sp-1..8 / --container-max / --gutter-*               … SP-DS-5
│    ├─ 6) 반경·테두리·엘리베이션 --r-* / --bw / --shadow-*             … SP-DS-6
│    └─ 7) 모션 --dur-* / --ease-*                                     … SP-DS-8
├─ * , *::before, *::after { box-sizing:border-box }  (리셋 최소)
├─ @media (prefers-reduced-motion: reduce) { … }      … SP-DS-8
└─ (컴포넌트 스타일 규칙 — SP-FE/SP-RPT/SP-ADS 등이 var(--*) 소비, 본 문서 소유 아님)
```

- **추적**: 브리프 §8, SP-ARCH-6, SP-FE-1, NFR2·5·15·19.

## SP-DS-2 — 색 팔레트(다크 기본)

**구현**: 브리프 §8 / NFR13·NFR15 / WIREFRAME §7·§8. 값은 **AA 검증을 통과한 출발점**이며 polish 시 톤만 조정(SP-DS-1 R3). 대비 수치는 SP-DS-11 테스트가 회귀 검증한다.

### SP-DS-2.1 배경 층(`--bg-0` ~ `--bg-4`)

깊은 곳(페이지)→얕은 곳(상승 표면·테두리) 순으로 밝아진다. 카드·패널·경계선은 이 층으로만 구분(그림자 최소).

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `--bg-0` | `#0d0f12` | 페이지 최하단 배경(`body`), 최심 캔버스 |
| `--bg-1` | `#14171c` | 기본 표면(헤더/푸터/컨테이너 배경) |
| `--bg-2` | `#1c212a` | 상승 표면(카드·후보 패널·입력 필드 배경) |
| `--bg-3` | `#262d38` | 미세 경계·hover 표면·구분선(강) |
| `--bg-4` | `#39424f` | 강조 테두리·비활성 컨트롤 표면·트랙 |

### SP-DS-2.2 텍스트 계층(`--t1` ~ `--t4`)

| 토큰 | 값 | 용도 | 대비(on `--bg-0` / `--bg-2`) | AA |
| --- | --- | --- | --- | --- |
| `--t1` | `#f2f4f7` | 본문·제목 주 텍스트 | 17.4 / 14.7 | ✅ 본문/대형 |
| `--t2` | `#c3cad4` | 보조 텍스트·라벨·값 | 11.6 / 9.8 | ✅ 본문 |
| `--t3` | `#8b94a3` | 뮤트 텍스트·도움말·메타·"광고" 라벨·밴드 표기 | 6.3 / 5.3 | ✅ 본문 |
| `--t4` | `#646d7c` | **비활성·장식·플레이스홀더 전용**(본문 금지) | 3.7 / 3.1 | ⚠ 대형/비텍스트(≥3:1)만 |

- **규칙 T1**: 본문 텍스트(≤18px 일반)에는 `--t1`~`--t3`만 사용(모두 ≥4.5:1). `--t4`는 4.5:1 미만이므로 **비활성 컨트롤·순수 장식·placeholder**에 한정하고 정보 전달 본문에 쓰지 않는다(NFR15). SP-DS-11 대비 테스트가 `--t4` 본문 사용을 회귀 방지 대상으로 명시.

### SP-DS-2.3 액센트 팔레트(`--blue/amber/green/red/purple/gold` + `-dim`)

액센트는 (a) **텍스트/아이콘/테두리**로 쓸 때는 원색(`--x`, 어두운 배경 위 ≥4.5:1), (b) **채움(버튼/배지 배경)** 으로 쓸 때는 원색 배경 + **어두운 텍스트(`--bg-0`)** 를 얹는다(액센트는 흰 텍스트보다 어두운 텍스트와 대비가 크다). `-dim`은 **은은한 배경 채움/포커스 후광용** 반투명 틴트다.

| 토큰 | 값 | 시맨틱 용도 | 대비(원색 텍스트 on `--bg-0`) | 채움+`--bg-0` 텍스트 대비 |
| --- | --- | --- | --- | --- |
| `--blue` | `#4c8dff` | 주 액션·링크·정보·"우위 A"(중립 기준색) | 6.0 | 6.0 |
| `--amber` | `#f0a83c` | 경고·만료·주의(추정 밴드 강조) | 9.5 | 9.5 |
| `--green` | `#43c489` | 긍정·이득·"공식" 확인·승자(+) | 8.7 | 8.7 |
| `--red` | `#ff6161` | 부정·손실·검증 오류·"열위"(−) | 6.5 | 6.5 |
| `--purple` | `#a98cfb` | 보조 강조·브랜드 축(vdCard `brand`)·보조 카테고리 | 7.2 | 7.2 |
| `--gold` | `#d9b45a` | 신뢰·프리미엄 강조(공식 출처·검증일 강조 선택) | 9.7 | 9.7 |

`-dim` 변형(반투명 배경/후광 — `rgb(<원색> / 0.16)` 관례, 채움 배경·칩·배지 바탕):

```
--blue-dim:   rgb(76 141 255 / 0.16);
--amber-dim:  rgb(240 168 60 / 0.16);
--green-dim:  rgb(67 196 137 / 0.16);
--red-dim:    rgb(255 97 97 / 0.16);
--purple-dim: rgb(169 140 251 / 0.16);
--gold-dim:   rgb(217 180 90 / 0.16);
```

- **규칙 A1**: 액센트를 **채운 버튼(Primary, WIREFRAME §8 BTN-1)** 은 텍스트를 `--bg-0`(어두운) 으로 둔다(흰 텍스트 대비 미달 방지 — `--amber`/`--gold`는 흰 텍스트가 특히 저대비). SP-DS-11이 검증.
- **규칙 A2**: 색만으로 의미를 전달하지 않는다(색맹 대응). 승/패·경고는 **아이콘·텍스트(예: "⚠ 만료", "우위 A")** 를 병기한다(NFR13). 액센트 색은 보강 신호.
- **규칙 A3**: 승자 대비(vdCard) 기본 매핑 = 이득/승 `--green`, 손실/열위 `--red`, 동점 `--t3`, 브랜드 축 `--purple`. 확정 시맨틱 별칭은 SP-DS-2.4. (엔진 산출 winner→색 적용은 SP-RPT 소유.)

### SP-DS-2.4 시맨틱 색 별칭(원시 팔레트 → 의미)

컴포넌트는 원시 색이 아니라 **의미 별칭**을 우선 참조한다(polish 시 원시→별칭 매핑만 바꾸면 전역 반영).

```
--accent:        var(--blue);      /* 주 액션·활성 */
--accent-ink:    var(--bg-0);      /* 액센트 채움 위 텍스트(어두운) */
--link:          var(--blue);      /* 인라인 링크 */
--positive:      var(--green);     /* 이득·승·공식 */
--negative:      var(--red);       /* 손실·오류·열위 */
--warning:       var(--amber);     /* 경고·만료 */
--brand-axis:    var(--purple);    /* vdCard brand 축 */
--text:          var(--t1);
--text-muted:    var(--t3);
--surface:       var(--bg-1);
--surface-raised:var(--bg-2);
--border:        var(--bg-3);
--border-strong: var(--bg-4);
--focus:         var(--blue);      /* 포커스 링 색(SP-DS-9) */
```

### SP-DS-2.5 라이트 테마 확장 훅(선택·MVP 밖)

- 라이트 테마는 MVP 범위 밖(브리프 §8 다크 기본). 향후 도입 시 **원시 팔레트만 재선언**하도록 훅을 남긴다: `:root[data-theme="light"] { --bg-0:…; --t1:…; … }`. 컴포넌트 규칙은 별칭(`var(--surface)` 등)만 참조하므로 **재작성 없이** 전환된다. MVP는 `data-theme` 미부착 = 다크.

- **추적**: 브리프 §8, NFR13·NFR15, WIREFRAME §7·§8, SP-DS-11.

## SP-DS-3 — 배지·상태·광고 라벨 색

**구현**: WIREFRAME §7(배지 카탈로그 official/est/만료) / FR-38 밴드(±5/±20/+15) / SP-ADS-5.2(광고 라벨) / NFR19. 배지 **표시 마크업**은 SP-RPT, 밴드 **계수 계산**은 SP-CALC 소유. 본 절은 그 표시에 쓸 **색 토큰만** 정의한다.

### SP-DS-3.1 배지 색(전경/배경 쌍)

WIREFRAME §7: official=강조색·체크, est=중립 회색, 만료=경고 ⚠.

| 배지 | 전경 토큰 | 배경 토큰 | 매핑 | 근거 |
| --- | --- | --- | --- | --- |
| 공식(official) | `--badge-official-fg: var(--green)` | `--badge-official-bg: var(--green-dim)` | 확인·검증(체크) | WIREFRAME §7 BG-1 |
| 추정(est) | `--badge-est-fg: var(--t3)` | `--badge-est-bg: var(--bg-3)` | 중립 회색 | WIREFRAME §7 |
| 만료(expired) | `--badge-expired-fg: var(--amber)` | `--badge-expired-bg: var(--amber-dim)` | 경고 ⚠ | WIREFRAME §7 BG-4 |
| 정성(qualitative) | `--badge-qual-fg: var(--t2)` | `--badge-qual-bg: var(--bg-3)` | 금액 대신 서술("정성적") | WIREFRAME §7 BG-6 |

### SP-DS-3.2 밴드 라벨 색(불확실성 ±)

밴드 텍스트(예: `(추정 ±20%)`, `(공식 ±5%)`, 만료 가산)는 **뮤트 톤**으로 값에 종속 표기하되, 만료 상태는 경고색을 얹는다(SP-CALC가 계수·상태를, SP-RPT가 문구·색 적용을 소유; 본 절은 색 토큰 제공).

```
--band-label-fg:    var(--t3);     /* 일반 밴드 표기(±5%/±20%) */
--band-expired-fg:  var(--amber);  /* 만료 가산(+15%) 강조 */
```

- 밴드 계수 정본은 SP-CALC(DEC-2: 명시 ±5% / 추정 ±20% / 만료 +15% 가산). 본 절은 색만 정의하며 계수를 재정의하지 않는다.

### SP-DS-3.3 광고·제휴 라벨 색(NFR19·FR-76·SP-ADS-5.2)

"광고" 라벨은 **은폐 금지·인지 가능**해야 한다. 색은 뮤트이되 대비 하한을 만족하고, 크기 하한은 SP-ADS가 소유(11px)하되 토큰으로 제공한다.

```
--ad-label-fg:      var(--t3);        /* 대비 6.3:1(on bg-0) — 인지 가능, 은폐 아님 */
--ad-label-fs:      12px;             /* WIREFRAME §0 "광고" 11~12px, SP-ADS 하한 11px 준수 */
--ad-slot-bg:       var(--bg-1);      /* 슬롯 컨테이너 배경(광고/콘텐츠 구분) */
--ad-slot-border:   var(--bg-3);      /* 슬롯 경계(콘텐츠 위장 방지, SP-ADS M-5) */
```

- **규칙 AD1**: `--ad-label-fg`는 `--t4`(<4.5:1)를 쓰지 않는다 — 라벨 은폐로 해석될 수 있음(NFR19). `--t3` 이상 대비 유지. `--ad-label-fs` ≥ 11px(SP-ADS-5.2 하한). 값은 SP-ADS `.ad-label` 규칙이 소비.
- **추적**: WIREFRAME §7, FR-38, FR-76·NFR19, SP-ADS-5.2, SP-CALC(DEC-2).

## SP-DS-7 — 브레이크포인트

**구현**: 브리프 §8 / WIREFRAME §0(모바일 <768 / 데스크톱 ≥768, 모바일 우선).

CSS 커스텀 속성은 미디어쿼리 조건에 직접 못 쓰므로 **정본 값은 상수로 고정**하고 문서 계약으로 강제한다(토큰은 참고용 별칭).

| 이름 | 조건 | 의미 |
| --- | --- | --- |
| (기본) | `< 768px` | **모바일**(1열 세로). 기본 스타일 = 모바일 우선 |
| `desktop` | `@media (min-width: 768px)` | **데스크톱**(가로·다열, 컨테이너 가운데 정렬) |

```
--bp-desktop: 768px;   /* 참고 토큰(문서 정본). 미디어쿼리엔 리터럴 768px 사용 */
```

- **규칙 BP1**: 브레이크포인트는 **768px 단일 분기**만 사용(중간 단계 없음, MVP). 추가 분기는 polish 시 도입 가능하나 토큰 문서에 등재 후 사용.
- **규칙 BP2**: 모든 컴포넌트는 **모바일 우선**(기본=모바일, `min-width:768px`로 데스크톱 오버라이드). WIREFRAME 각 화면의 "모바일/데스크톱" 와이어가 이 분기에 대응.
- **추적**: 브리프 §8, WIREFRAME §0.

@media (prefers-reduced-motion: reduce) {
