## 1. 목적

TASK는 6단계 **SPEC**(SP-* 구현 계약)을 개발자가 그대로 착수할 수 있는 **구현 작업(리프)으로 분해**한다. 각 도메인 SP-대역을 최소 독립 기능 리프로 쪼개고, 리프마다 **구현 + 대응 테스트**를 짝지어 TDD(red-green-refactor)로 완료한다. **DoD(완료 기준) = 구현 + 테스트 green 둘 다**(SP-TEST-1). 작업은 계층(도메인 → 그룹 T-nn.g → 리프 T-nn.g.i)으로 나뉘고, 각 리프는 진행 마커(`- [ ]`/`- [-]`/`- [v]`)로 상태를 관리한다.

## 5. 규약 리마인더

- **DoD**: 리프 완료 = 구현 + 대응 테스트 green 둘 다(SP-TEST-1). 한쪽만이면 진행중.
- **진행 마커**: `- [ ]` 미착수 · `- [-]` 진행중 · `- [v]` 완료(구현+테스트 green).
- **결정 처리**: 애매하면 임의 판단 금지, **AskUserQuestion으로 확인** 후 진행(특히 DG-1~4).
- **범위 불변식(전 도메인 고정)**: 로그인/회원/프로파일러/서버 측 사용자 쓰기 **부재**(INV-1·4) · 비교 계산 **100% 클라이언트**(SP-ENGINE) · 회사 등록 **~96(≠200)**(INV-6) · DEC-2 밴드 **stated ±5% / estimated ±20% / 만료 +15%**(INV-5).
