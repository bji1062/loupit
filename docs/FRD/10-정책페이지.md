# 정책·고지 페이지 (FRD)

**목적**: F7(정책 페이지)의 **정적 고지 문서 4종**(개인정보처리방침·이용약관·면책조항·광고/제휴 고지)과 **정정요청 연락 경로**를 필드·규칙 단위로 정밀 명세한다. 구체적으로 (1) 4종 정책 페이지의 존재·전역 접근(푸터)·정적 배포, (2) 개인정보처리방침의 필수 포함 항목(비로그인·서버 무저장·localStorage 한정·AdSense 쿠키/제3자 고지), (3) 이용약관의 필수 포함 항목(비교·정보 도구 성격, 비-채용 플랫폼), (4) 데이터 정확성 면책조항(참고용·정확성 한계·배지 의미·불확실성 밴드), (5) 광고·제휴 고지("광고" 표기 원칙·제휴 관계·공정위 표시광고 규정 정합), (6) 신고 기능 부재 하의 정정요청 연락 경로(임시안), (7) 정책 페이지의 얇은 유틸리티 페이지 광고·색인 정합성, (8) 동의 배너 ↔ 정책의 단일 진실 연결(정책 측 계약)을 확정한다. 본 문서는 **"어떤 정책 문서를 어떤 필수 항목으로 게시·연결하는가"**를 규정하며, 배지·불확실성 밴드의 산출 규약과 광고·동의 컴포넌트의 동작은 재정의하지 않고 각 소유 FRD(전역 규약 `01`·데이터 계약 `02`·광고/제휴 담당 문서)를 인용한다.

**상위 추적**: FRD → USECASE → PRD → 브리프. 상위 근거 = USECASE `07-광고와정책.md`(**UC-62·UC-63**, 연계 UC-42·UC-60·UC-61·UC-A4), PRD `05-기능-요구사항.md`(**F7**, 연동 F6), PRD `06-수익화.md`(MON7-6·MON11·MON13·MON14·MON15·MON17·MON18), PRD `07-비기능-요구사항.md`(NFR5·NFR6·NFR9·NFR10·NFR12·NFR13·NFR16·NFR17·NFR18·NFR19·NFR22·NFR24·NFR25·NFR29), PRD `08-데이터-요구사항.md`(D3.9·D3.10·D4.1·D4.2·D4.6), PRD `10-리스크.md`(R3·R5), RESEARCH `benefit-scraping.md`(§5 법적 안전장치 5·6). 브리프 근거: §1(제품 정의), §2-1(비로그인·서버 무저장), §2-2(localStorage 한정), §2-5(수익화·"광고" 표기), §2-6(정책 페이지 광고 최소화), §7(불확실성 밴드). 전역 규약·용어는 FRD `01-개요와-FR규칙.md`(FR-01·FR-05·FR-07), 배지·금액 신뢰도 계약은 FRD `02-데이터-계약.md`(FR-D9), 불확실성 밴드 산식(금액 신뢰도 `amt_source` 기준, DEC-2)은 FRD `05-비교엔진.md`(FR-38)를 인용하며 재정의하지 않는다.

**범위 경계**: 본 문서는 F7의 **정책·고지 문서(콘텐츠)와 그 게시·접근·정합 규칙**만 다룬다.
- **광고·제휴 컴포넌트의 동작**(AdSense 자동/수동 슬롯 렌더, `affiliate.json` 제휴 링크, 개인화 **동의 배너의 표시·저장·개인화/폴백** 메커니즘 MON13·MON15, "광고" 라벨 부착)은 **F6 광고·제휴 담당 FRD 문서(FR-7x 대역)**가 소유한다. 본 문서는 그 컴포넌트가 **링크로 참조하는 정책 문서의 내용·존재**(개인정보처리방침·광고/제휴 고지)와 **동의↔정책 단일 진실 연결의 정책 측 계약**만 규정한다(FR-87, MON14).
- **배지·불확실성 밴드의 산출 규약**(밴드는 출처 배지가 아니라 **금액 신뢰도 `amt_source`** 기준: 명시 금액 ±5%/추정 금액 ±20%/만료 +15% 확대 — DEC-2 디커플링. 배지 자체는 `badge_cd`·만료로 파생)은 전역 규약 FR-05·계산 엔진 **05 FR-38**·데이터 계약 FR-D9가 소유한다. 본 문서(면책조항, FR-83)는 그 배지·밴드의 **의미를 사용자에게 고지하는 문안 요건**만 규정한다.
- **정적 페이지 생성 파이프라인·`<head>` 메타·`sitemap.xml`·`robots.txt`의 생성 방식**은 F4 정적 생성(FRD `07`) 및 그와 동일한 빌드타임 규약(UC-A4)을 따른다. 본 문서는 정책 페이지가 그 파이프라인에 **포함되어야 함**(4종 URL 게시·sitemap 등록·비-JS 본문)을 규정하되 생성기 구현은 재정의하지 않는다.
- 프로파일러와 익명 경로 서버 측 사용자 쓰기는 영구 제외(§2-1·§2-3·§2-4)이며, 로그인·회원은 이 익명 게시·열람·정정요청 경로에 등장하지 않는다(로그인·재직 인증은 복지 등록·수정 기여 SC14 한정 — FRD 13, FR-01). 정책 열람·정정요청 접수는 페이지 자체에서 **서버로의 사용자 데이터 쓰기를 발생시키지 않으며**(NFR16), 정정 반영은 런타임 쓰기가 아니라 **빌드타임 재생성**(D4.2)으로만 이뤄진다.

**FR-ID 대역**: 본 문서는 **FR-8x 대역**(FR-80~FR-87)을 소유한다(작성 지시). 부여한 ID는 재사용하지 않는다(안정 ID, 브리프 §9). 본 문서의 헤드라인 커버 UC는 **UC-63**(정책·고지 페이지 열람)이며, **UC-62**(광고·쿠키 개인화 동의/거부)의 **정책 측 요구**(개인정보처리방침·광고/제휴 고지 링크 대상, 단일 진실 연결)를 함께 커버한다. 하위 문서(FLOW/WIREFRAME/SPEC/TASK)가 이 ID를 인용한다.

> **대역 정합 비고**: FRD `01` §2.3 초안은 FR-8x를 F8(읽기 전용 API)에 잠정 배정했으나, 실제 문서 구성에서 **F8의 API 계약은 FRD `02-데이터-계약.md`가 FR-Dx 대역**(FR-D1~FR-D11: `reference/all`·`companies/search`·`companies/{id}` 스키마)으로 소유한다. 이에 따라 FR-8x가 비어 F7 정책 페이지에 배정되었다. FRD `01` §2.2-6이 예약한 "F7 분리 대역 승격" 경로에 해당한다. `01`에서의 `FR-82` 표기는 명명 형식 예시일 뿐 소유 항목이 아니므로 충돌하지 않는다.

## 공통 규약 (전 FR 적용)

- **정적·빌드타임 게시**: 정책 페이지 4종은 빌드타임에 정적 HTML로 생성되어 Nginx가 직접 서빙한다(UC-A4, NFR1 정합). 요청 시 Uvicorn 호출·DB 조회·서버 렌더링이 없으며, 문안 갱신은 재빌드·재배포로만 이뤄진다(§2-6, D4.2). 런타임 쓰기 경로 없음(§2-4).
- **비-JS 본문**: 정책 본문은 JS 실행 없이 최종 HTML 마크업에 완성된 상태로 포함되어 브라우저·크롤러가 그대로 표시·색인할 수 있어야 한다(NFR12·NFR24, UC-94/UC-A3). JS는 (있다면) 향상 목적으로만 쓰이며 정책 본문 표시나 링크 이동을 막지 않는다.
- **얇은 유틸리티 페이지·광고 최소화**: 정책 페이지는 자동광고 OFF/최소·수동 슬롯 없음이다(MON7-6). 클릭 유도·과도 광고 없이 AdSense 프로그램 정책에 정합한다(MON18). 광고 밀도 "없음"이므로 광고로 인한 레이아웃 이동은 발생하지 않으며 폰트 로딩 CLS만 억제한다(NFR5).
- **전역 접근(푸터)**: 4종 정책 페이지는 사이트 전역 내비게이션(푸터 등)에서 접근 가능해야 한다(MON17, NFR18). 정책 링크는 순수 `<a href>`로 제공되어 JS 없이도 이동 가능하다.
- **서버 무저장·로컬 한정**: 정책 열람·정정요청 접수 전 과정에서 사용자 데이터의 서버 전송·저장이 없다(NFR16, §2-1). 광고·동의 상태 등 사용자 상태는 브라우저(localStorage/광고 쿠키)에만 존재한다(§2-2, NFR17, FR-07).
- **단일 진실 원칙**: 쿠키·제3자(Google) 사용·광고 목적·데이터 정확성·"광고" 표기 등 **동일 사실은 한 정책 문서에서 정의**하고, 동의 배너·광고 표기·타 페이지는 이를 **참조·링크**한다(중복 정의로 인한 불일치 금지, MON14).
- **접근성·시맨틱·한국어**: 시맨틱 HTML5 landmark(`header`/`nav`/`main`/`footer`)와 단일 `<h1>`(정책명) 이하 heading 위계를 사용하고 WCAG 2.1 AA를 목표로 한다(NFR13). `<html lang="ko">`·`<meta charset="utf-8">`, UTF-8 인코딩(NFR29). 각 페이지는 고유한 `<title>`·`meta description`을 갖는다(NFR6).
- **시크릿 미노출**: 정적 산출물·저장소에 DB 자격증명·PAT·배포 토큰을 노출하지 않으며 AdSense client id는 플레이스홀더를 유지한다(NFR22, MON5). 정책 페이지에 게시하는 연락처 등 상수는 운영자(A5)가 배포 시 주입한다.
- **창작 금지·근거 한정**: 정책 문안은 브리프·상위문서에 근거한 사실만 담는다. 법률 자문·수치·연락처 등 브리프에 없는 값은 본 FRD가 **필수 항목의 존재·의미**로 규정하되, 구체 문구·연락처 실값은 운영자가 확정하는 상수로 남긴다(플레이스홀더 규약).

## [FR-80] 정책·고지 페이지 집합·전역 접근(푸터)·정적 배포

**설명**: 개인정보처리방침·이용약관·면책조항(데이터 정확성)·광고/제휴 고지의 **4종 정책 페이지가 각각 독립 URL로 정적 게시**되고, 사이트 전역 내비게이션(푸터)에서 접근 가능해야 한다. 이는 AdSense 승인·규정 준수의 전제 요건이다.

**상세 동작**

- 정상 흐름
  1. 빌드타임 정적 생성이 4종 정책 페이지를 각각 고유 URL로 산출하고 배포한다(UC-A4, A5).
  2. 모든 페이지(랜딩·회사 상세·조합·비교 툴·정책)의 **푸터**에 4종 정책 페이지 링크를 순수 `<a href>`로 노출한다(MON17).
  3. 사용자는 전역 푸터의 정책 링크 또는 동의 배너의 정책 링크(UC-62, FR-87)로 정책 페이지에 진입한다.
  4. 요청한 정책 유형에 해당하는 정적 페이지가 반환된다.
- 대안/예외
  - 특정 정책 페이지 미게시·404: **금지 상태**다. 4종 중 하나라도 미게시·404이면 AdSense 정책 위반 위험(MON17 미충족)이며, 배포 단계(UC-A4)의 생성물 검증에서 4종 URL 존재·푸터 링크 존재를 확인하여 방지한다(리스크 R1 계열, NFR9 sitemap 검증과 정합).
  - 정책 문안 갱신: 재빌드·재배포로만 반영한다(D4.2, §2-4 런타임 쓰기 없음).

**입력**

| 입력 | 소스 | 타입/제약 |
| --- | --- | --- |
| 정책 문안 4종 | 운영자 확정 콘텐츠(A5) | 개인정보처리방침 / 이용약관 / 면책조항 / 광고·제휴 고지 각 1문서 |
| 정책 URL 경로 | 빌드 설정 | 4종 각 고유 정적 경로(예: `/policy/privacy`, `/policy/terms`, `/policy/disclaimer`, `/policy/ads`; 실경로는 SPEC 확정) |
| 푸터 링크 목록 | 빌드 설정(전역 레이아웃) | 4종 정책 페이지로의 `<a href>` 링크 |

**출력·상태**

- 산출물: 정책 정적 페이지 4개 + 전역 푸터의 정책 링크 세트. 서버·DB 사용자 상태 변화 0건.
- 사용자에게 4종 정책·고지 내용이 전역에서 열람 가능한 상태.

**검증·비즈니스 규칙**

1. 정책 페이지 **정확히 4종**(개인정보처리방침·이용약관·면책·광고/제휴 고지)이 각각 존재한다(F7 수용기준 1, §3 F7).
2. 4종 URL이 모든 페이지의 푸터에서 접근 가능하다(MON17, NFR18). 링크는 JS 없이 이동 가능한 `<a href>`다(NFR24).
3. 4종 URL이 `sitemap.xml`에 등록된다(NFR9, FR-86과 연동).
4. 4종 중 누락·404가 0이다(생성물 검증 대상, UC-63 2b 방지).

**추적**: UC-63 / F7 / MON17·MON18 / NFR18·NFR9·NFR12·NFR24 / §2-6·§3(F7)·D4.2 / 연계 UC-A4(배포)·FR-86·FR-87.

## [FR-83] 데이터 정확성 면책조항 필수 포함 항목(배지·밴드 의미)

**설명**: 면책조항 페이지는 복지·연봉·근무조건 데이터가 **참고용이며 실제와 다를 수 있음**과, **배지(공식/추정)·만료·불확실성 밴드**의 의미를 사용자에게 고지해야 한다(D4.1, §7, R3). 배지·밴드의 **산출 규약**은 전역 규약 FR-05·계산 엔진 05 FR-38(DEC-2)·데이터 계약 FR-D9가 소유하며, 본 페이지는 그 **의미의 고지 문안**만 규정한다. 특히 불확실성 밴드는 출처 배지가 아니라 **금액 신뢰도(`amt_source`)** 를 기준으로 산정되므로(DEC-2 디커플링, 05 FR-38), 면책 문안도 amt_source 기준으로 고지하여 실제 산출 밴드와 어긋나지 않게 한다.

**상세 동작**

- 정상 흐름 — 페이지는 다음 필수 항목을 고지한다.

| # | 필수 포함 항목 | 근거 |
| --- | --- | --- |
| D-1 | 복지·연봉·근무조건 데이터는 **참고용**이며 **실제와 다를 수 있다**. 채용 의사결정의 최종 확인 책임은 사용자에게 있다 | D4.1 |
| D-2 | 각 복지 항목은 **출처 신뢰도 배지**를 가진다 — **공식(official)**: 회사 공식 공개 정보 기반 / **추정(est)**: 집계·프리셋 등 추정 기반 | §4·§7, FR-05, FR-D9, RESEARCH §4 |
| D-3 | 배지가 공식이라도 **금액이 추정치일 수 있다**(출처 신뢰도와 금액 신뢰도의 분리). 추정 금액은 넓은 오차로 표시된다 | RESEARCH DEC-2·§4, FR-D9 |
| D-4 | 복지 금액에는 **불확실성 밴드**가 적용되며, 밴드는 출처 배지가 아니라 **금액 신뢰도(`amt_source`)** 를 기준으로 한다(D-3의 출처·금액 분리 원칙) — **명시 금액 ±5% / 추정 금액 ±20% / 만료 시 +15% 확대(가산)**. 따라서 출처가 **공식**이어도 금액이 추정치면 밴드는 **±20%가 유지**된다 | §7, 05 FR-38·DEC-2, FR-05(개정)·FR-D9, D3.9 |
| D-5 | 데이터에는 **검증일·만료 시점**이 있으며, 만료된 항목은 "재확인 필요"로 표기되고 오차가 확대된다 | D4.3, D3.10, FR-D9 |
| D-6 | 비교 계산은 **사용자 브라우저에서 수행**되며 서비스는 특정 회사가 절대적으로 유리하다고 **단정하지 않는다** | §2-8, R3(결정 8) |
| D-7 | 데이터 정정을 원하는 회사·사용자를 위한 **정정 요청 경로 안내**(FR-85로 링크) | R5, D4.6 |

- 대안/예외
  - 밴드 수치·기준은 계산 엔진 **05 FR-38**(DEC-2, `amt_source` 기준: 명시 0.05·추정 0.20·만료 +0.15 가산)과 그와 정합한 전역 규약 FR-05(개정)·FR-D9가 고정한 값과 **동일하게** 서술한다. 본 페이지가 다른 계수 또는 **다른 기준(예: 출처 배지 기준)**을 서술하면 계산(05 FR-38·FR-D9)과 불일치 → 금지.

**입력**

| 입력 | 소스 | 타입/제약 |
| --- | --- | --- |
| 정확성 면책 사실 | D4.1, R3 | 참고용·의사결정 책임 |
| 배지·밴드 의미 | §7·§4, 05 FR-38·DEC-2, FR-05·FR-D9, RESEARCH §4 | 배지(공식/추정)·만료·±밴드(`amt_source` 기준: 명시/추정)·금액 신뢰도 분리 |
| 정정 경로 링크 | FR-85 | 정정요청 페이지·연락처로의 링크 |

**출력·상태**

- 면책조항 정적 페이지 1종(D-1~D-7 포함). 서버 사용자 상태 변화 0건.

**검증·비즈니스 규칙**

1. D-4의 밴드 계수·기준이 계산 엔진 **05 FR-38**(DEC-2)의 고정값과 **정확히 일치**한다 — 밴드는 **금액 신뢰도 `amt_source` 기준**(명시 `stated` 0.05·추정 `estimated` 0.20)이고 만료는 기준 계수에 **+0.15 가산**(치환 아님)이며, **출처 배지(`badge_cd`) 기준으로 서술하지 않는다**(디커플링, 불일치 금지). 정합된 전역 규약 FR-05(개정)·FR-D9와도 일치한다.
2. D-2·D-3이 배지·금액 신뢰도 디커플링(FR-D9, 05 FR-38, RESEARCH DEC-2)과 정합.
3. 면책조항이 실제 밴드 산식(**05 FR-38**·DEC-2)과, 비교 리포트·회사 상세 페이지의 배지·출처·만료 UI 표시(FRD `06` FR-41·FRD `07` FR-54·UC-42, 정합된 FR-05 개정)와 **모순되지 않는다**(F3·F4 정합). 특히 **'배지 official + 금액 estimated'** 항목의 불확실성을 **과소 고지하지 않는다**(DEC-2가 회피하려는 R3 법적 노출 방지).
4. F7 수용기준 4(비교 데이터·복지 정보의 정확성 한계 명시)를 충족한다.

**추적**: UC-63·UC-42 / F7(F3 정합) / — / D4.1·D4.3·D3.9·D3.10·§7·§2-8·R3, RESEARCH §4.2·DEC-2 / 연계 05 FR-38·FR-05·FR-D9·FR-85·UC-34.

## [FR-84] 광고·제휴 고지 필수 포함 항목("광고" 표기·제휴)

**설명**: 광고·제휴 고지 페이지는 loupit의 **광고·제휴 관계**와 **"광고" 표기 원칙**을 고지하여 공정위 표시광고 규정 취지에 정합해야 한다(MON11, §2-5, NFR19). 개별 광고·제휴 컴포넌트의 라벨 부착 동작은 F6 담당 문서가 소유하고, 본 페이지는 그 **원칙의 고지**를 규정한다.

**상세 동작**

- 정상 흐름 — 페이지는 다음 필수 항목을 고지한다.

| # | 필수 포함 항목 | 근거 |
| --- | --- | --- |
| A-1 | 서비스는 **Google AdSense 광고**와 **제휴(affiliate) 링크**로 운영 비용을 충당한다(유료 구독·결제 없음) | MON1, §2-5 |
| A-2 | 모든 광고·제휴 영역에는 **"광고"(또는 "유료 광고")** 라벨을 사용자가 인지 가능하게 표기한다(숨김 금지) | MON11, NFR19 |
| A-3 | 제휴 링크 클릭 시 **제휴사로부터 수수료를 받을 수 있다**(제휴 관계 고지) | MON10·MON11, §2-5 |

  - "광고" 라벨을 눈에 띄지 않게 숨기려는 배치는 **금지**(표시광고 명확성 요건, MON11). 본 페이지는 라벨을 인지 가능한 색·크기로 표기하는 원칙을 명문화한다.

| "광고" 표기 원칙 | MON11, NFR19 | 공정위 표시광고 규정 |
| 제3자 광고 쿠키 사실 | MON13·MON14 | Google 쿠키·개인화 동의 |

**출력·상태**

- 광고·제휴 고지 정적 페이지 1종(A-1~A-5 포함). 서버 사용자 상태 변화 0건.

**검증·비즈니스 규칙**

## [FR-85] 정정요청 연락 경로(신고 기능 前 임시안)

**설명**: MVP는 런타임 신고·정정 수집 기능이 없으므로(§3 신고 기능 제외, D4.6), 정책 페이지에 **회사·사용자의 데이터 정정 요청을 위한 오프라인 연락 경로**를 게시하고, 접수 시 **수작업 큐레이션 → 빌드타임 재배포**로 반영하는 임시안을 명세한다(R5, RESEARCH §5-6).

**상세 동작**

- 정상 흐름
  1. 정책 페이지(면책조항 또는 별도 고지 영역)에 **정정 요청 연락 경로**를 게시한다(예: 이메일 주소 또는 문의 링크; 실값은 운영자가 배포 시 주입하는 상수, 플레이스홀더 규약).
  2. 회사·사용자가 그 경로(사이트 외부 채널)로 정정·삭제 요청을 보낸다. **loupit 서버로의 사용자 데이터 쓰기는 발생하지 않는다**(§2-4, NFR16).
  3. 운영자(A5)가 요청을 검토하여 해당 `TCOMPANY_BENEFIT` 행을 **수정 또는 무효화**하고(필요 시 출처 `BADGE_SRC_CD`를 `manual`로), **재시드 → 정적 재생성 → 재배포**로 반영한다(D4.2).

- 처리 상태표

| 단계 | 주체 | 동작 | 반영 시점 |
| --- | --- | --- | --- |
| 접수 | 회사/사용자 → A5 | 외부 채널로 정정 요청(서버 쓰기 없음) | 즉시(오프라인) |
| 검토·수정 | A5 | 해당 복지 행 수정/무효화(수작업 큐레이션) | 다음 빌드 |
| 반영 | 빌드 파이프라인 | 재시드 → 재생성 → 재배포(D4.2) | 배포 주기 종속 |

- 대안/예외
  - 런타임 신고 폼·API를 **두지 않는다**(§3, §2-4). 페이지 내 어떤 요소도 서버로 사용자 입력을 저장하는 쓰기 경로를 만들지 않는다.
  - **SLA 한계 고지**: 정정 반영은 **배포 주기에 종속**됨을 잔여 리스크로 고지한다(R5, OI-7). 즉시 반영을 보장하지 않는다.

**입력**

| 입력 | 소스 | 타입/제약 |
| --- | --- | --- |
| 정정 요청 연락처 | 운영자 확정 상수(A5) | 이메일/문의 링크 등(플레이스홀더까지 허용, 배포 시 실값 주입) |
| 정정 대상 데이터 | `TCOMPANY_BENEFIT` 행 | 수정/무효화 대상(수작업 큐레이션) |

**출력·상태**

- 정정 요청 연락 경로가 정책 페이지에 게시된 상태. 페이지 자체는 서버 사용자 쓰기 0건.
- 접수·반영은 오프라인 큐레이션 + 빌드타임 재배포로 처리되어 참조 데이터가 갱신된 상태.

**검증·비즈니스 규칙**

1. 정책 페이지에 **정정 요청 연락 경로가 게시**되어 있다(R5 완화, RESEARCH §5-6). 미게시 금지.
2. 정정 요청 처리에 **런타임 서버 쓰기 경로가 없다**(§2-4, NFR16, D4.6). 반영은 빌드타임 재생성만(D4.2).
3. 연락처 실값 미확정 시 플레이스홀더를 유지하되, 배포(UC-A4) 전 실값 주입을 확인한다(운영 절차).
4. SLA(배포 주기 종속) 한계가 고지된다(R5, OI-7 잔여 리스크).

**추적**: UC-63 / F7 / NFR16 / D4.6·D4.2·R5, RESEARCH §5-6 / 연계 FR-83(면책 링크)·UC-A4(재배포).

## [FR-86] 정책 페이지 광고·색인 정합성(얇은 유틸리티 페이지)

**설명**: 정책 페이지는 **얇은 유틸리티 페이지**로서 자동광고 OFF/최소·수동 슬롯 없음이며(MON7-6), 클릭 유도·과도 광고 없이 AdSense 프로그램 정책에 정합하고(MON18), JS 없이 크롤러가 색인 가능한 정적 문서여야 한다(NFR12·NFR24, UC-A3).

**상세 동작**

- 정상 흐름
  1. 정책 페이지에 **자동광고 스니펫을 로드하지 않거나 최소화**하고 **수동 슬롯을 배치하지 않는다**(MON7-6, MON8 페이지 유형 게이팅과 정합).
  2. 페이지는 정적 HTML로 서빙되어 크롤러가 JS 없이 본문(정책 문안)을 색인한다(NFR12·NFR24, UC-A3). 필수 정책 게시 사실이 AdSense 승인 심사에서 확인된다(MON17·MON18).
  3. 각 정책 페이지는 고유 `<title>`·`meta description`(NFR6), 시맨틱 landmark·heading 위계(NFR13), `<html lang="ko">`·UTF-8(NFR29)로 렌더된다.
  4. 4종 정책 URL이 `sitemap.xml`에 등록되고 `robots.txt`가 사이트 루트에 제공된다(NFR9·NFR10).

- 대안/예외
  - 정책 페이지에 광고를 조밀하게 배치하려는 시도: **금지**. 얇은 페이지 광고 회피(MON18)·정책 페이지 무·최소 광고(MON7-6) 위반이며 AdSense 승인 리스크(R1).

**입력**

| 입력 | 소스 | 타입/제약 |
| --- | --- | --- |
| 페이지 유형 판별 | 빌드/런타임 게이팅 | 정책 유형 → 자동광고 OFF/최소·슬롯 없음(MON7-6·MON8) |
| SEO 메타 | 빌드 설정 | 페이지별 고유 title·description(NFR6) |
| sitemap 엔트리 | 빌드 설정 | 4종 정책 URL(NFR9) |

**출력·상태**

- 광고 없음/최소·크롤러 색인 가능·sitemap 등록된 정책 정적 페이지. 광고 로드로 인한 레이아웃 이동 없음(NFR5).

**검증·비즈니스 규칙**

1. 정책 페이지에 수동 광고 슬롯이 **0개**이고 자동광고가 OFF/최소다(MON7-6, MON8).
2. JS 무효화 상태에서 정책 본문이 표시·색인 가능하다(NFR12·NFR24, UC-94/UC-A3).
3. 4종 정책 URL이 `sitemap.xml`에 존재하고 누락 0(NFR9, FR-80과 연동).
4. 각 페이지가 고유 title·description(NFR6)·`lang="ko"`·UTF-8(NFR29)·시맨틱 landmark(NFR13)를 만족한다.
5. AdSense client id는 플레이스홀더를 유지하고 산출물에 시크릿 미포함(NFR22, MON5).

**추적**: UC-63 / F7 / MON7-6·MON8·MON17·MON18 / NFR5·NFR6·NFR9·NFR10·NFR12·NFR13·NFR22·NFR24·NFR29 / §2-6 / 연계 UC-A3(색인)·FR-80.

## [FR-87] 동의 배너 ↔ 정책 단일 진실 연결(정책 측 계약)

**설명**: 광고·쿠키 개인화 동의 배너(UC-62, F6 담당 문서 소유)와 정책 페이지가 **쿠키·제3자(Google)·광고 목적·동의 선택권**이라는 동일 사실을 **단일 진실로 공유**하도록, 본 문서는 그 **정책 측 계약**(동의 배너가 링크하는 정책 문서의 내용·도달성)을 규정한다(MON14). 동의 배너의 표시·저장·개인화/폴백 **동작**(MON13·MON15)은 F6 담당 문서가 소유한다.

**상세 동작**

- 정상 흐름
  1. 동의 배너는 **개인정보처리방침(FR-81)**·**광고/제휴 고지(FR-84)**로 링크한다(MON14). 본 문서는 그 링크 대상 두 페이지가 존재하고(FR-80) 쿠키·제3자·광고 목적·동의 선택권 사실을 담도록 보장한다(FR-81 P3·P4, FR-84 A-4).
  2. 사용자가 동의 배너의 정책 링크로 진입하면 배너가 참조한 사실과 **동일한 사실**을 정책 페이지에서 확인한다(단일 진실, UC-62 → UC-63).
  3. 동의/거부에 따른 개인화/비개인화/미노출은 F6 담당 문서가 처리하며, 그 상태는 브라우저(localStorage/쿠키)에만 저장된다(§2-2, FR-07). 정책 페이지는 이 사실(P5·A-4)을 고지한다.

- 대안/예외
  - 배너와 정책 문서의 사실이 어긋나는 경우: **금지**. 쿠키·제3자·목적·선택권은 정책 문서(FR-81·FR-84)를 단일 진실로 하고 배너는 이를 요약·링크한다(불일치 금지).
  - 정책 링크 대상 미게시: FR-80 4종 게시 요건 위반(UC-63 2b)이며 동의 배너 링크가 404를 향하면 안 된다.

**입력**

| 입력 | 소스 | 타입/제약 |
| --- | --- | --- |
| 동의 배너 링크 요구 | MON14 | 개인정보처리방침·광고/제휴 고지로의 링크 |
| 단일 진실 사실 | FR-81·FR-84 | 쿠키·제3자·광고 목적·동의 선택권 |

**출력·상태**

- 동의 배너가 링크하는 정책 문서가 존재·도달 가능하고 동일 사실을 담은 상태. 동의 상태는 클라이언트 한정(서버 저장·크로스 디바이스 동기화 없음).

**검증·비즈니스 규칙**

1. 동의 배너의 정책 링크 대상(개인정보처리방침·광고/제휴 고지)이 존재하고 도달 가능하다(FR-80, MON14). 링크가 404를 향하지 않는다.
2. 배너가 고지하는 쿠키·제3자·목적·선택권이 정책 문서(FR-81 P3·P4·P5, FR-84 A-4)와 **동일**하다(단일 진실, MON14).
3. 동의 상태가 서버로 전송·저장되지 않는다(§2-1·§2-2, NFR16·NFR17). 본 계약은 상태 저장 메커니즘을 F6 담당 문서에 위임하되, 정책 측 고지가 그 동작과 정합함을 보장한다.

**추적**: UC-62·UC-63 / F6·F7 / MON13·MON14·MON15 / NFR18·NFR16·NFR17 / §2-2·§2-5 / 연계 FR-81·FR-84·FR-07·UC-60(개인화/폴백).
