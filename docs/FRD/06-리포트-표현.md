# 리포트 표현·로컬저장 (FRD)

**목적**: F3(클라이언트 비교 리포트)의 **표현 계층**과 **로컬 지속성**을 필드·규칙 단위로 정밀 명세한다. 구체적으로 (1) 계산된 리포트를 화면에 구성·렌더링하는 방식(판정카드·총보상·시간조정 가치·워라밸·자율성·카테고리별 복지 델타·정성 복지 표), (2) 복지 배지·불확실성 밴드의 표시, (3) 입력 변경 시 재계산·리렌더 트리거, (4) "최근 비교"의 localStorage 저장·불러오기·삭제(스키마·최대 개수), (5) localStorage 불가 시 폴백, (6) 렌더 시 XSS 이스케이프를 확정한다. 본 문서는 **"어떻게 표시·저장하는가"**를 규정하며, 실효연봉·시간가치·워라밸 요인·카테고리 합산·vdCard 판정 등 **계산식(§7)은 재정의하지 않고** 상위 F3 계산 FR(FR-3x, 문서 05)의 출력을 인용해 렌더한다.

**상위 추적**: FRD → USECASE → PRD → 브리프. 상위 근거 = USECASE `04-비교리포트.md`(UC-34·UC-35·UC-36, 그리고 표현 대상인 UC-30~UC-33 출력), PRD `05-기능-요구사항.md`(F3), PRD `07-비기능-요구사항.md`(NFR2·NFR5·NFR13~15·NFR16·NFR17·NFR21·NFR25·NFR26), 브리프 §7(비교 알고리즘 출력·`compare`/`calc`/`renderBenCatCompare`/`renderQualCompare`)·§4(데이터 모델·배지·만료)·§2-2(localStorage)·§2-6(결과 화면 광고)·§2-8(클라이언트 계산). 전역 규약·용어·상태 모델은 FRD `01-개요와-FR규칙.md`(FR-01·FR-03·FR-04·FR-05·FR-06·FR-07, §4 용어, §5 상태 모델), 데이터 필드 계약은 FRD `02-데이터-계약.md`(FR-D5·FR-D8·FR-D9)를 인용하며 재정의하지 않는다.

**범위 경계**: 본 문서는 F3의 **표현·재계산 트리거·로컬 저장**만 다룬다. 리포트 지표의 **계산 로직**(실효연봉=연봉+순복지, 시간조정 가치 `getWSHours`/`getOTPay`, 워라밸 요인 `REMOTE_SAVE`, 카테고리 합산 `getBenTotal`, vdCard 판정, 밴드 계수 산출)은 F3 계산 FR(FR-3x, 문서 05, UC-30~UC-33)이 소유한다. 입력 필드의 정의·검증은 F2(FR-2x, 문서 04, UC-20~25)가 소유하며, 본 문서는 입력 **변경 이벤트**를 재계산 트리거로만 참조한다. 결과 화면의 광고 슬롯 콘텐츠는 F6(FR-6x, 문서 07)이 소유하고, 본 문서는 레이아웃 이동 방지를 위한 **고정 크기 예약**만 규정한다. 로그인/회원/프로파일러/서버측 사용자 쓰기(영구 제외, §2-1·§2-3·§2-4)는 어떤 표현·저장 경로에도 등장하지 않는다(FR-01). 리포트 표현·저장 전 과정에서 계산 입력·결과가 서버로 전송·저장되지 않으며(§2-8, NFR16), 지속 저장은 브라우저 localStorage에만 발생한다(§2-2, NFR17, FR-07).

**FR-ID 대역**: 본 문서는 **FR-4x 대역**(FR-40~FR-45)을 소유한다(작성 지시). 부여한 ID는 재사용하지 않는다(안정 ID). 본 문서의 헤드라인 커버 UC는 **UC-34·UC-35·UC-36**이며(FR-41·FR-42·FR-43·FR-44), 리포트 구성(FR-40)과 렌더 안전(FR-45)은 UC-30~UC-34의 출력 표시를 지지한다. 하위 문서(FLOW/WIREFRAME/SPEC/TASK)가 이 ID를 인용한다.

## 공통 규약 (전 FR 적용)

- **표현 vs 계산 분리**: 본 대역의 FR은 F3 계산 FR(FR-3x)이 산출한 값을 **표시**만 한다. 어떤 FR도 실효연봉·시간가치·밴드 계수를 재정의하지 않는다. 표시 대상 값이 미산출(입력 부족)이면 해당 지표만 "미산출"로 표기하고 리포트 전체를 무효화하지 않는다(FR-42, UC-35 1a).
- **단위·포맷**: 모든 금액은 **만원 단위 정수**로 계산되며(§4, FR-04), 표시 시 한국어 "만/억" 포맷으로 렌더한다(예: `12000` → "1억 2,000만원"). 통근시간은 분 단위(FR-24).
- **두 관점 대비**: 모든 지표는 슬롯 a(현직)·b(이직처)를 나란히 대비하는 형태로 표시하고, 필요한 지표는 델타(a-b 또는 b-a)를 함께 제시한다(§7 출력: 두 관점 비교).
- **미전송·로컬 한정**: 표현·재계산·저장 전 과정에서 계산 입력·결과가 서버로 전송·저장되지 않는다(§2-8, NFR16). 지속 저장은 localStorage 한정이며 쿠키·서버 세션을 사용하지 않는다(광고 쿠키 제외, NFR17, FR-07).
- **오프라인 렌더**: 참조 번들(REF) 로드 성공 후에는 네트워크 없이도 리포트 렌더·재계산이 동작한다(§7, NFR26).
- **렌더 안전(횡단)**: 회사·복지 문자열(`comp_nm`·`benefit_nm`·`note_ctnt`·`qual_desc_ctnt`·`aliases`·`industry_nm`)과 복원된 localStorage 콘텐츠는 `textContent`/DOM API/이스케이프로 삽입하며 데이터 보간 `innerHTML`을 금지한다(NFR21, 상세 FR-45).
- **레이아웃 안정**: 리포트·배지·광고 슬롯 삽입으로 인한 레이아웃 이동을 방지한다. 동적 영역은 고정 크기를 예약한다(CLS ≤ 0.1, NFR5). 재계산 리렌더는 INP ≤ 200ms를 목표로 한다(NFR2).
- **접근성**: 리포트는 시맨틱 HTML(heading 위계, 카테고리 델타는 `<table>`)로 구성하고 본문 대비 ≥ 4.5:1을 유지하며, 배지는 색상만이 아니라 텍스트 라벨로도 의미를 전달한다(NFR13·NFR15).

## [FR-40] 리포트 표현 구성·렌더링

**설명**: F3 계산 FR(FR-3x, 문서 05)이 산출한 a/b 리포트 출력을 화면에 **구성·렌더링**한다. 판정 카드(vdCard)·총보상·시간조정 가치·워라밸·자율성·카테고리별 복지 델타·정성 복지 표를 두 관점(a/b) 대비 레이아웃으로 표시한다. 본 FR은 표시 배치·구조·안정성만 규정하고 값 계산은 재정의하지 않는다(§7, FR-3x 인용).

**상세 동작**
- 정상
  1. UC-30으로 `compare()`가 a/b 리포트 출력을 산출하면(FR-3x), 본 FR이 이를 리포트 화면에 렌더한다.
  2. 리포트는 다음 구성요소를 포함한다(§7 출력 정의, "리포트 표현 대상" 표):
     - **판정 카드(vdCard)**: 우선순위(`curPri`)·희생요소(`curSacrifice`) 기준 "누가 유리/무엇을 택할지" 판정과 근거(UC-33). 두 관점 우열이 근소하면 과잉 단정을 피하는 표기를 렌더한다(UC-33 3a).
     - **총보상 카드**: 실효연봉(연봉+순복지)을 a·b·델타로 표시. 각 금액에 배지·불확실성 밴드를 병기한다(FR-41, UC-31·UC-34).
     - **시간조정 가치 카드**: 주간근무시간·야근수당을 반영한 시간당 가치를 a/b 대비로 표시(UC-31).
     - **워라밸·자율성 카드**: 재택절약·유연·무제한휴가 반영 요인을 a/b 대비로 표시(UC-31).
     - **카테고리별 복지 델타 표**: 9개 카테고리(`compensation`/`flexibility`/`work_env`/`time_off`/`health`/`family`/`growth`/`leisure`/`perks`)의 a·b 합산과 델타를 `<table>`로 표시(UC-32, `renderBenCatCompare`).
     - **정성 복지 표**: 정성 항목(`qual_yn=true`, 금액 없음)을 금액 델타 대신 설명(`qual_desc_ctnt`) a/b 대비로 표시(UC-32, `renderQualCompare`).
  3. 각 구성요소는 슬롯 a(현직)·b(이직처)를 나란히 대비하고, 정량 지표는 델타를 함께 표시한다.
  4. 사용자는 각 지표 상세를 조회한다(UC-31~UC-34). 지표 상세 확인의 유즈케이스 흐름은 USECASE가, 표시 배치는 본 FR이 규정한다.
- 대안·예외

  | 분기 | 조건 | 처리 |
  |------|------|------|
  | 2a | 특정 지표가 미산출(입력 부족, 예: 연봉 `null`) | 해당 카드만 "미산출" 상태로 렌더하고 나머지 리포트는 정상 표시(리포트 전체 무효화 금지, UC-35 1a) |
  | 2b | 한 슬롯이 회사 미선택 직접 입력(UC-15) | 프리셋 초기값 없이 사용자 입력 값만으로 산출된 지표를 렌더(UC-30 3a). 브랜드 축 판정 제한 시 vdCard가 그 사유를 표기(UC-33 2a) |
  | 2c | 한 카테고리에 정성 항목만 존재 | 해당 카테고리는 금액 델타 대신 정성 설명으로 렌더(UC-32 3a) |
  | 4a | 결과 화면 진입 시 광고 슬롯 | 결과 화면은 절제된 광고 슬롯을 노출할 수 있으나(§2-6) 슬롯 콘텐츠는 F6 소유. 본 FR은 CLS 방지를 위한 고정 크기 예약만 렌더(NFR5) |

**입력**
| 필드 | 타입 | 제약 | 출처 |
|------|------|------|------|
| a/b 리포트 출력 | `compare()` 결과 객체(관점별 지표·판정·밴드) | FR-3x 산출 | F3 계산(문서 05, §7) |
| 표시 원문 문자열 | `comp_nm`·`benefit_nm`·`qual_desc_ctnt`·`note_ctnt` 등 | 이스케이프 대상(FR-45) | REF(FR-D4·FR-D5) |

**출력·상태**
- a/b 비교 리포트가 위 구성요소로 클라이언트에 렌더링·조회된다(서버 전송·저장 없음, NFR16).
- 리포트 렌더는 클라이언트 표시 상태만 생성하며 공용 상태(§5)를 변경하지 않는다(읽기 표시).

**검증·비즈니스 규칙**
- 렌더 구성요소는 §7 출력 정의(총보상·시간조정 가치·워라밸·자율성·카테고리 델타·정성·vdCard)를 빠짐없이 포함한다.
- 지표 값·계수는 FR-3x 계산 결과를 그대로 표시하며 본 FR이 재계산하지 않는다(표현·계산 분리).
- 카테고리 표는 9종 집합만 헤더로 사용하고 미상 카테고리는 `perks`로 폴백된 항목을 표시한다(FR-D8·FR-D5).
- 시맨틱 HTML·heading 위계·`<table>` 사용, 대비 ≥ 4.5:1(NFR13·NFR15). 동적 영역 고정 크기 예약으로 CLS ≤ 0.1(NFR5).
- 모든 표시 문자열은 FR-45 규약(이스케이프/`textContent`)으로 삽입한다(NFR21).

## [FR-41] 복지 배지·불확실성 밴드 표시

**설명**: 복지 금액·총보상 옆에 **출처 배지**(공식/추정/만료)와 **금액 신뢰도**(`amt_source`), **불확실성 밴드(±)**를 표시하여 사용자가 수치의 신뢰수준을 이해하게 한다. 배지 파생·밴드 계수는 전역 규약(FR-05)·데이터 계약(FR-D9)이 소유하고, 본 FR은 그 결과의 **표시**를 규정한다(UC-34, §4·§7).

**상세 동작**
- 정상
  1. 각 복지 항목·총보상 금액에 **출처 배지**를 표시한다. 배지는 `badge_cd`와 `expires_dtm`으로 파생한다(FR-D9.1):

     | `badge_cd` | `expires_dtm` | 파생 상태 | 표시 라벨 |
     |------------|---------------|-----------|-----------|
     | `official` | 없음/미경과 | 공식 | **공식** |
     | `official` | 경과(만료) | 만료 | **만료 · 재확인 필요** |
     | `est`(또는 그 외) | 무관 | 추정 | **추정** |

  2. **금액 신뢰도**(`amt_source ∈ {stated, estimated, none}`, FR-D5·FR-D9)를 함께 표시한다: `stated`=명시 금액, `estimated`=추정 금액("추정치" 표기), `none`=정성(금액 없음). 출처 배지(`official`)와 금액 신뢰도는 독립 축이므로 "공식 · 추정치" 조합이 표시될 수 있다(DEC-2, FR-D9). 프리셋 폴백 유래 항목(FR-06, `benefit_presets`)은 "유형 기준" 추정 성격을 함께 드러낸다.
  3. **불확실성 밴드(±)**: 체크된 금전 복지의 밴드를 합산해 총보상을 단일값이 아닌 **범위**(예: "실효연봉 6,200만원 ±약 500만원")로 표시한다. 밴드 계수 산출은 FR-05·FR-D9.2가 소유하며, 만료 항목은 밴드가 확대되고 "재확인 필요"가 병기된다.
  4. 사용자는 배지·밴드를 통해 추정치일수록 범위가 넓음을 인지한다(UC-34 3·4).
- 대안·예외

  | 분기 | 조건 | 처리 |
  |------|------|------|
  | 1a | 프리셋으로 채워진 항목 | 프리셋 배지(주로 `est`) 기준으로 "추정"·"유형 기준"을 표시(UC-34 1a, FR-06·FR-D3) |
  | 1b | 항목 만료(`expires_dtm` 경과) | "만료 · 재확인 필요" 라벨 + 밴드 확대 표시(UC-34 2a, FR-D9) |
  | 3a | 정성 항목(`qual_yn=true`, `amt_source=none`) | 금액 밴드 미적용. 배지·출처(`badge_src_cd`)만 표시(UC-34 3a, FR-D5·FR-D9) |
  | 1c | 미체크 항목 | 밴드 합산·총보상 표시에서 제외(FR-05) |

**입력**
| 필드 | 타입 | 제약 | 상태변수/출처 |
|------|------|------|----------------|
| `badge_cd` | ∈ {`official`,`est`} | FR-D5 | `benS[s][i].badge_cd` |
| `amt_source` | ∈ {`stated`,`estimated`,`none`} | FR-D5·FR-D9 | `benS[s][i].amt_source` |
| `expires_dtm` | ISO8601 \| null | 경과 판정 입력 | `benS[s][i].expires_dtm` |
| `badge_src_cd` / `badge_src_url_ctnt` | 출처 유형/URL | 출처표기(정성 포함) | `benS[s][i]`(FR-D5) |
| 밴드 계수/범위 | number | FR-05·FR-D9.2 산출 | F3 계산(문서 05) |

**출력·상태**
- 항목별 배지 라벨(공식/추정/만료·재확인 필요) + 금액 신뢰도 표기 + 총보상의 ± 범위 표시. 표시 상태만 생성(공용 상태 불변).

**검증·비즈니스 규칙**
- 배지 파생은 FR-D9.1 표를 따른다(`official`+만료 → "만료·재확인 필요", 그 외/`est` → "추정").
- 배지는 색상만이 아니라 **텍스트 라벨**로도 의미를 전달한다(NFR15 대비·비색상 전달).
- **밴드 기준 오픈이슈(OI-6)**: PRD D3.9(배지 기준 밴드 official ±5% / est ±20% / 만료 ±15%, FR-05)와 RESEARCH §4.2(금액 신뢰도 기준 밴드, FR-D9.2)의 상충은 SPEC/TASK에서 하나로 확정된다. 본 FR은 **표시 요구**(± 범위 노출·만료 시 확대·"재확인 필요")만 규정하고 계수 기준은 확정된 계산(FR-05/FR-D9)을 그대로 렌더한다.
- 배지 규약은 F7 데이터 정확성 면책조항과 정합해야 한다(UC-34, FR-05).
- 출처 URL(`badge_src_url_ctnt`)을 링크로 렌더할 경우 스킴을 `http`/`https`로 제한하고 이스케이프한다(FR-45, NFR21).

## [FR-42] 입력 변경 재계산·리렌더 트리거

**설명**: 리포트 조회 중 사용자가 F2 입력(연봉·복지 체크/금액·근무형태·통근시간·우선순위·희생요소)을 변경하면, 변경된 입력으로 리포트를 **재계산·리렌더**한다. 재계산은 F3 계산 FR(`compare()`·실시간 요약 `calc()`)이 수행하고, 본 FR은 **변경 감지→재계산 호출→리렌더** 트리거와 갱신 표시를 규정한다(UC-35, §7). 서버 호출은 발생하지 않는다(§2-8).

**상세 동작**
- 정상
  1. 조회 중 F2 입력 하나 이상이 변경되면(F2 상태 `salS`/`benS`/`wsState`/`cmtS`/`curPri`/`curSacrifice`/`selectedRate` 변경) 본 FR이 재계산을 트리거한다.
  2. 변경 입력으로 `calc()`(실시간 요약)·`compare()`(확정 리포트)를 재실행한다(FR-3x, §7). 서버 호출 없음.
  3. 총보상·시간조정 가치·워라밸·자율성·카테고리별 복지 델타·판정(vdCard)·불확실성 밴드가 갱신된 값으로 다시 렌더된다(FR-40·FR-41).
  4. 사용자가 갱신된 a/b 비교 결과를 확인한다.
- 대안·예외

  | 분기 | 조건 | 처리 |
  |------|------|------|
  | 2a | 우선순위(`curPri`)만 변경 | 판정 축만 전환되어 vdCard를 재렌더한다(다른 지표 값 동일, UC-35 2a·UC-33) |
  | 2b | 회사 교체·슬롯 재지정(UC-14) | 새 회사의 실데이터/프리셋으로 `benS[s]` 초기값이 바뀌고 전체 지표를 재계산·리렌더한다(UC-35 2b, FR-06) |
  | 1a | 변경으로 필수값(예: 연봉) 비워짐 | 해당 지표 카드만 "미산출"로 표시하고 입력 보완을 안내(리포트 전체 무효화 금지, UC-35 1a·FR-40 2a) |

**입력**
| 필드 | 타입 | 제약 | 상태변수 |
|------|------|------|----------|
| F2 입력 변경 이벤트 | 상태 변경 신호 | `salS`/`benS`/`wsState`/`cmtS`/`curPri`/`curSacrifice`/`selectedRate` 중 하나 이상 | FRD 01 §5, FR-2x |

**출력·상태**
- 변경 입력을 반영한 최신 리포트가 렌더된다. 재계산은 클라이언트에서만 발생하고 서버 저장이 없다(§2-8, NFR16).
- 공용 상태(§5)는 F2가 변경하며, 본 FR은 변경을 감지해 렌더 표시만 갱신한다.

**검증·비즈니스 규칙**
- 재계산·리렌더는 서버 호출 0(§2-8, NFR16). 로드 성공 후 오프라인에서도 동작(NFR26).
- 리렌더는 고정 크기 예약을 유지해 레이아웃 이동을 유발하지 않으며(CLS ≤ 0.1, NFR5), 입력 응답성 INP ≤ 200ms를 목표로 한다(NFR2).
- 필수값 결측은 부분 미산출로 처리하고 크래시하지 않는다(FR-40 2a).
- `curPri` 변경은 vdCard 축만 전환하며 나머지 지표 값을 재계산하지 않는다(효율·정합, UC-35 2a).

**추적**: UC-35 / F3(F2 입력 연동) / FR-03·FR-40·FR-41 / NFR2·NFR5·NFR16·NFR26 / PRD-SEG-1·SEG-2 / §7(`compare`·`calc`)·§2-8.

## [FR-43] 최근 비교 localStorage 저장·불러오기·삭제

**설명**: 사용자가 조회한 비교를 "최근 비교"로 **브라우저 localStorage에만** 저장하고, 재방문 시 불러와 복원하며, 항목별·전체 삭제를 제공한다. 서버 계정·서버 저장은 존재하지 않고 전 과정이 로그인 없이 기기 내에서만 동작한다(§2-1·§2-2, FR-07). 본 FR은 저장 **스키마·최대 개수·조작 규칙**을 확정한다(UC-36, 공통 로컬 저장 UC-A2의 F3 맥락 구체화).

**상세 동작**
- 정상
  1. **저장**: 사용자가 조회한 비교를 "최근 비교"로 저장하면, 비교 입력 스냅샷과 결과 요약을 localStorage 레코드로 직렬화해 기록한다(서버 전송 없음, §2-2). 프로파일러 상태는 저장 대상이 아니다(FR-D10).
  2. **불러오기**: 재방문·재진입 시 최근 비교 목록을 localStorage에서 읽어 표시하고, 선택 시 저장 시점 입력으로 리포트를 복원해 조회 상태(UC-30)로 재진입한다.
  3. **삭제(항목)**: 특정 최근 비교의 `id`로 해당 레코드를 제거하고 목록에서 뺀다.
  4. **삭제(전체)**: 모든 레코드를 제거하여 목록을 비운다.
  5. UI는 저장·불러오기·삭제가 **로그인·서버 통신 없이 이 기기 내에서만** 유지됨(크로스 디바이스 동기화 없음)을 고지한다(§2-2, PRD-CTX-4).
- 대안·예외

  | 분기 | 조건 | 처리 |
  |------|------|------|
  | 1a | localStorage 비활성/차단/용량 초과 | 저장을 생략하고 비교·조회·재계산은 정상 동작(FR-44, NFR25) |
  | 1b | 최대 개수 초과 저장 | 가장 오래된 레코드를 축출(FIFO)하고 신규를 선두에 추가(아래 규칙 R2) |
  | 1c | 동일 비교 재저장 | 동일 시그니처(슬롯 회사 + 핵심 입력) 레코드가 있으면 갱신·선두 이동(중복 방지, 규칙 R3) |
  | 2a | 저장된 최근 비교 없음 | 빈 상태로 목록 미표시(UC-36 2a) |
  | 2b | 복원 후 최신 참조 갱신 원함 | 복원은 저장 시점 입력 기준. 사용자가 원하면 FR-42 재계산으로 갱신(UC-36 2b) |
  | 2c | 스키마 버전 불일치·손상 레코드 | 해당 레코드를 무시·폐기하고 크래시하지 않는다(규칙 R4·FR-45) |

**최근 비교 저장 스키마**

localStorage 최상위 키 **`loupit.recentComparisons`** 하나에 봉투(envelope) JSON을 저장한다.

```
{
  "v": 1,                 // 스키마 버전(int). 불일치 시 마이그레이션/폐기(R4)
  "items": [ RecentComparison, ... ]   // 최신순(newest-first), 길이 ≤ MAX_RECENT
}
```

`RecentComparison` 레코드 필드:

| 필드 | 타입 | 널 | 설명 |
|------|------|:---:|------|
| `id` | string | N | 안정 식별자(삭제 키). 저장 시각 기반 또는 랜덤 |
| `savedAt` | string(ISO8601) | N | 저장 시각 |
| `label` | string | N | 표시 라벨. `slots.a.comp_nm`·`slots.b.comp_nm` 조합(예: "삼성전자 vs SK하이닉스"), 미선택 슬롯은 "직접 입력" 폴백. 렌더 시 이스케이프(FR-45) |
| `slots` | object | N | `{a: SlotSummary, b: SlotSummary}` — 회사 요약 |
| `input` | object | N | 복원용 입력 스냅샷(아래) |
| `result` | object | N | 목록·복원 표시용 결과 요약(아래) |

`SlotSummary`: `{comp_id: int|null, comp_nm: string|null, comp_tp_cd: string|null}` (직접 입력 슬롯은 `comp_id=null`, §5 `matched` 요약).

`input`(복원 스냅샷, F2 상태): `{ salS:{a,b}, selectedRate, benS:{a,b}, wsState:{a,b}, cmtS:{a,b}, curPri, curSacrifice }`. `benS` 항목은 복원에 필요한 필드(`benefit_cd`·`benefit_nm`·`benefit_amt`·`benefit_ctgr_cd`·`badge_cd`·`amt_source`·`checked`·`qual_yn`·`expires_dtm?`)만 포함한다(FR-D5·FR-D8, §5).

`result`(요약): `{ totalComp:{a,b}(만원), band:{a,b}(±만원, 선택), verdict: string(판정 요약, 렌더 시 이스케이프), priAxis: curPri }`.

**최대 개수·조작 규칙**

| 규칙 | 내용 |
|------|------|
| R1(MAX_RECENT) | 최근 비교 최대 보관 개수 = **10**. localStorage 용량 보호 + "최근 목록" UX 근거로 설정하며, SPEC 단계에서 상수로 조정 가능하다. |
| R2(축출) | 저장 시 신규를 `items` 선두에 추가하고, 길이가 MAX_RECENT를 초과하면 말미(가장 오래된)를 제거한다(FIFO). |
| R3(중복 방지) | 동일 시그니처(양 슬롯 `comp_id`/직접입력 + 핵심 입력) 레코드가 있으면 새로 추가하지 않고 기존 레코드를 갱신·선두 이동한다. |
| R4(버전·검증) | 읽기 시 `v`가 현재 버전과 다르거나 필수 필드가 없으면 해당 레코드/봉투를 폐기하고 빈 목록으로 취급한다(무크래시, FR-45 신뢰 경계). |

**입력**
| 필드 | 타입 | 제약 | 상태변수 |
|------|------|------|----------|
| 저장 트리거 | 사용자 "최근 비교 저장" 액션 | — | (UI) |
| 저장 대상 상태 | F2 입력 + F3 결과 요약 | §5 상태·`compare` 출력 | `matched`(요약)·`benS`·`wsState`·`curPri`·`curSacrifice`·`selectedRate`·`cmtS`·`salS` |
| 삭제 대상 | 레코드 `id` 또는 전체 | 존재하는 `id` | `items[].id` |

**출력·상태**
- 최근 비교가 해당 기기 localStorage(`loupit.recentComparisons`)에만 저장·유지·삭제된다. 서버 저장·크로스 디바이스 동기화 없음(§2-1·§2-2, PRD-CTX-4).
- 불러오기는 저장 스냅샷을 §5 공용 상태로 복원해 조회 상태(UC-30)로 재진입한다.

**검증·비즈니스 규칙**
- 저장소는 localStorage 한정, 서버 전송 0(NFR16·NFR17). 저장 실패는 흡수하고 비교는 정상 동작(FR-44, NFR25).
- 스키마는 위 봉투·레코드 구조를 따르고, 최대 개수 R1(10)·축출 R2·중복 R3·검증 R4 규칙을 만족한다.
- 복원된 레코드의 모든 문자열(`label`·`comp_nm`·`benefit_nm`·`verdict` 등)은 신뢰 불가로 간주해 렌더 시 이스케이프한다(FR-45).
- 프로파일러 상태(`pfResult`·`pfJob` 등)는 저장·복원 대상이 아니다(FR-D8·FR-D10).

## [FR-44] localStorage 불가 시 폴백

**설명**: localStorage가 비활성(사생활 모드)·차단·용량 초과여서 저장·복원이 불가능할 때, 비교·조회·재계산은 정상 동작하고 **저장·복원만 생략**한다. 저장 실패로 앱이 크래시하지 않는다(UC-36 1a, NFR25, FR-07).

**상세 동작**
- 정상(폴백)
  1. 저장·읽기 접근은 `try/catch`로 감싸고, 예외(접근 거부·`QuotaExceededError` 등) 발생 시 흡수한다.
  2. 저장 불가로 판정되면 "최근 비교" 저장 UI를 비활성 또는 미제공으로 처리하고, 저장이 이 브라우저에서 불가함을 사용자에게 알린다.
  3. 비교 리포트 생성(UC-30)·조회(UC-31~34)·재계산(UC-35)은 저장 가능 여부와 무관하게 정상 동작한다.
- 대안·예외

  | 분기 | 조건 | 처리 |
  |------|------|------|
  | 1a | 저장 중 `QuotaExceededError` | 예외 흡수. 오래된 레코드 축출(FR-43 R2) 후에도 초과 시 저장 생략, 비교는 계속 |
  | 1b | 읽기 접근 거부(사생활 모드) | 빈 목록으로 취급하고 복원 미제공, 비교는 정상 |
  | 1c | JSON 파싱 실패·손상 데이터 | 해당 봉투 폐기(FR-43 R4), 크래시 없음 |

**입력**: 없음(런타임 환경·예외).

**출력·상태**
- 저장·복원 미제공, 비교 기능 정상. 어떤 사용자 데이터도 서버로 대체 전송되지 않는다(NFR16).

**검증·비즈니스 규칙**
- 저장·읽기 실패는 전부 `try/catch`로 흡수하여 무크래시(NFR25).
- 저장 불가 시 비교·조회·재계산 경로에 영향 없음(기능 저하는 "최근 비교" 저장·복원에 한정).
- 서버 세션·쿠키를 저장 폴백으로 사용하지 않는다(광고 쿠키 제외, NFR17).

**추적**: UC-36 1a / F3 / FR-07·FR-43 / NFR25·NFR16·NFR17 / PRD-CTX-3·PRD-CTX-4 / §2-2.

## [FR-45] 리포트 렌더 XSS 이스케이프

**설명**: 리포트·배지·최근 비교 목록 렌더 시 회사·복지 데이터와 복원된 localStorage 콘텐츠를 **HTML 이스케이프/`textContent`**로 삽입하여 XSS를 차단한다. 데이터를 보간한 `innerHTML` 직접 삽입을 금지한다(NFR21, §7 렌더 안전). 본 FR은 표현 계층 전반(FR-40·FR-41·FR-43)의 횡단 렌더 안전 규약이다.

**상세 동작**
- 정상
  1. 렌더 대상 문자열을 `textContent`/DOM 생성 API 또는 이스케이프 함수로 삽입한다: `comp_nm`·`benefit_nm`·`note_ctnt`·`qual_desc_ctnt`·`aliases`·`industry_nm`(REF, FR-D4·FR-D5) 및 최근 비교의 `label`·`verdict`·회사명(FR-43 복원 데이터).
  2. 출처 URL(`badge_src_url_ctnt`)을 링크로 렌더할 경우 스킴을 `http`/`https`로 제한하고 `javascript:` 등 위험 스킴을 차단하며, 속성값을 이스케이프한다.
  3. 복원된 localStorage 콘텐츠는 **신뢰 불가 입력**으로 간주해 렌더 전 이스케이프·스키마 검증(FR-43 R4)을 거친다.
- 대안·예외

  | 분기 | 조건 | 처리 |
  |------|------|------|
  | 1a | 복지 설명에 HTML 특수문자·태그 포함 | 이스케이프하여 문자 그대로 표시(태그 미실행) |
  | 2a | 출처 URL이 위험 스킴 | 링크화하지 않고 무해 텍스트로 표시하거나 제거 |
  | 3a | 손상·조작된 localStorage 페이로드 | 스키마 검증 실패 시 폐기(FR-43 R4), 렌더 안 함 |

**입력**
| 필드 | 타입 | 제약 | 출처 |
|------|------|------|------|
| 렌더 문자열 | string | 이스케이프 필수 | REF(FR-D4·FR-D5), 복원 레코드(FR-43) |
| 렌더 URL | string(URL) | `http`/`https`만 허용 | `badge_src_url_ctnt`(FR-D5) |

**출력·상태**
- 이스케이프·검증을 거친 안전한 DOM 텍스트/속성. 스크립트 실행 없음.

**검증·비즈니스 규칙**
- 데이터·사용자·복원 문자열을 보간한 `innerHTML` 직접 삽입 금지, `textContent`/이스케이프 사용(NFR21).
- XSS 페이로드 주입 테스트를 통과한다(NFR21 수용 기준).
- URL 렌더는 스킴 화이트리스트(`http`/`https`)를 적용한다.
- 복원 localStorage 콘텐츠는 신뢰 경계 밖으로 취급해 이스케이프·검증한다(FR-43 R4).

**추적**: UC-30~34·UC-36(표시 지지) / F3 / FR-40·FR-41·FR-43·FR-D5 / NFR21 / §7(렌더 안전)·§4.
