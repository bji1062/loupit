# 광고·제휴 (FRD)

**목적**: F6(광고/제휴 슬롯)을 필드·규칙 단위로 정밀 명세한다. 구체적으로 (1) Google AdSense 설정(client id·슬롯 id 플레이스홀더)의 **단일 위치 주입**, (2) **자동광고(Auto ads) 조건부 로드**와 **수동 광고 슬롯(Manual ad unit) 배치**, (3) **페이지 유형별 광고 배치 게이팅 매트릭스**(콘텐츠 페이지 위주·비교 입력 무광고·비교 결과 하단 1개), (4) **광고 슬롯 CLS 방지**(예약 높이·미노출 폴백), (5) **제휴 `affiliate.json` 스키마와 데이터 기반 컴포넌트 렌더**, (6) 광고·제휴 영역의 **"광고" 표기**(공정위 표시광고 규정 준수)와 **광고성 링크 속성**(`rel="sponsored nofollow"`), (7) **광고·쿠키 개인화 동의**의 고지·상태 보관·정책 연결과 **미동의/비개인화 폴백**을 확정한다. 본 문서는 **"어느 페이지에 무엇을 어떤 규칙으로 렌더/게이팅하고, 표기·동의·폴백을 어떻게 강제하는가"**를 규정하며, 광고 노출 KPI(MON19)와 AdSense 승인 콘텐츠 볼륨(MON16)·필수 정책 페이지(F7·MON17)의 소유는 각 담당 문서에 위임하고 인용만 한다.

**상위 추적**: FRD → USECASE → PRD → 브리프. 상위 근거 = USECASE `07-광고와정책.md`(UC-60·UC-61·UC-62, 연계 UC-63), PRD `05-기능-요구사항.md`(F6, 연동 F4·F5·F7), PRD `06-수익화.md`(MON1~MON19, 특히 하위 문서 인용 규칙상 FRD 소유 대상인 **MON3~MON15**), PRD `07-비기능-요구사항.md`(NFR2·NFR5·NFR16·NFR17·NFR18·NFR19·NFR21·NFR22·NFR25·NFR29), 브리프 §2-5(수익화·"광고" 표기)·§2-6(광고 배치 정책)·§2-7(SEO=승인 콘텐츠)·§8(AdSense 스니펫·플레이스홀더 client id·`affiliate.json`). 전역 규약·용어는 FRD `01-개요와-FR규칙.md`(FR-01 서버 무쓰기·FR-07 로컬 지속성·§4 용어 "광고" 표기)를 인용하며 재정의하지 않는다.

**범위 경계**: 본 문서는 F6의 **광고·제휴 컴포넌트 동작(노출·게이팅·표기·링크 속성·동의·폴백)**만 다룬다.
1. 광고가 배치되는 콘텐츠 페이지 **자체의 생성·본문·SEO**는 F4(회사 상세, FRD `07-회사페이지-정적생성.md` FR-58)·F5(인기 조합)가 소유한다. 본 문서는 그 페이지 위에서 마운트되는 **광고/제휴 컴포넌트의 배치·게이팅·표기 규칙**을 소유하고, 정적 페이지 문서는 자기 마크업 내 **슬롯 위치·고정 크기 예약**만 규정한다(FR-58은 본 문서 FR-71~FR-76을 역참조).
2. **정책·고지 페이지 4종(개인정보처리방침·이용약관·면책·광고/제휴 고지)의 게시·본문**은 F7(UC-63)이 소유한다. 본 문서는 동의 배너가 정책 페이지로 **링크**하는 연결(MON14)과 정책 페이지의 **무광고 게이팅**(MON7-6)만 규정하고 정책 본문을 작성하지 않는다.
3. 광고 노출·유입 **KPI 정의**는 `02-목표와-성공지표.md`가 소유하며 본 문서는 중복 정의하지 않는다(MON19). AdSense **승인 요건**(콘텐츠 볼륨 MON16·정책 페이지 게시 MON17)은 F4/F5/F7 소유이며, 본 문서는 승인 전 **플레이스홀더 유지**(MON5·MON18)와 **프로그램 정책 정합**(클릭 유도 금지·콘텐츠/광고 구분·얇은 페이지 회피, MON18)만 규정한다.
4. 로그인/회원/프로파일러/서버측 사용자 쓰기(영구 제외, §2-1·§2-3·§2-4)는 어떤 광고·제휴·동의 경로에도 등장하지 않는다(FR-01). 광고 노출·제휴 클릭·동의·폴백 전 과정에서 사용자 데이터의 **서버 저장·전송이 없으며**(NFR16), 개인화 동의를 포함한 모든 상태는 **브라우저(localStorage/쿠키)에만** 존재한다(§2-2, FR-07·MON13).

**FR-ID 대역**: 본 문서는 **FR-7x 대역**(FR-70~FR-79)을 소유한다(작성 지시). FRD `01` §2.3 인덱스는 F6·F7 병합 기준으로 FR-6x를 지시하나, 인기 비교 조합(F5) 문서가 FR-6x를 소유하는 실제 배정에 맞춰 본 F6 광고·제휴 문서는 **FR-7x**로 배정된다(FR-01 §2.2-4 안정 ID, 중복·재사용 금지). 헤드라인 커버 UC는 **UC-60·UC-61·UC-62**(정책 열람 UC-63은 F7 문서가 소유, 본 문서는 연계 인용). 하위 문서(FLOW/WIREFRAME/SPEC/TASK)가 이 ID를 인용한다.

## 공통 규약 (전 FR 적용)

- **페이지 유형이 배치의 단일 축**: 모든 광고/제휴 컴포넌트의 마운트 여부·구성은 **현재 페이지 유형(`page_type`)** 하나로 게이팅한다(MON8). `page_type` 열거값은 아래 6종으로 고정한다.

  | `page_type` | 대응 화면 | 대응 기능 | MON7 행 |
  | --- | --- | :---: | :---: |
  | `landing` | 랜딩(홈) | 랜딩 | MON7-1 |
  | `company` | 회사 상세 정적 페이지 | F4 | MON7-2 |
  | `combo` | 인기 비교 조합 정적 페이지 | F5 | MON7-3 |
  | `input` | 비교 입력 화면 | F2 | MON7-4 |
  | `result` | 비교 결과(리포트) 화면 | F3 | MON7-5 |
  | `policy` | 정책·고지 페이지 | F7 | MON7-6 |

  회사 랭킹/리스트(MON7-7·SC12)는 **2차 검토·MVP 미포함**이므로 본 열거에 넣지 않는다(창작 금지). 게이팅은 페이지 유형 판별 실패 시 **광고 없음(안전 기본값)**으로 처리한다.
- **서버 무쓰기·로컬 한정**: 광고 노출·제휴 클릭·동의 전 과정에서 사용자 데이터의 서버 전송·저장이 없다(NFR16, FR-01). 개인화 동의 상태만 브라우저(localStorage/쿠키)에 보관한다(FR-07·MON13). AdSense가 설정하는 광고 쿠키는 제3자(Google) 도메인 소관이며 loupit 서버 저장이 아니다(NFR17 "광고 쿠키 제외" 단서와 정합).
- **표기 명확성(공정위)**: 모든 광고·제휴 영역은 사용자가 인지 가능한 **"광고"(또는 "유료 광고")** 라벨을 부착한다(MON11·NFR19). 라벨을 배경과 구분되지 않는 색·과소 크기로 숨기지 않는다(표시광고 명확성 요건, FR-76).
- **렌더 안전(XSS 방지)**: `affiliate.json`에서 읽은 문자열(`label`·`desc`·`url`)은 마크업에 삽입 시 **HTML 이스케이프**하며, `url`은 `http`/`https` 스킴으로 제한한다(NFR21). 원문 무이스케이프 삽입을 금지한다(FR-75).
- **레이아웃 안정(CLS)**: 광고/제휴 슬롯은 광고 유닛 크기에 상응하는 **예약 높이**를 가져 삽입 전후 레이아웃 이동을 억제한다(CLS ≤ 0.1, NFR2·NFR5). 광고 미노출(미승인·차단기·네트워크 실패·미동의 폴백)에서도 레이아웃이 깨지지 않는다(FR-74·MON6).
- **한국어·인코딩**: 광고 라벨·동의 문안·제휴 라벨은 한국어이며 UTF-8로 렌더한다(NFR29).
- **설정의 단일 진실**: client id·슬롯 id는 코드에 산재 하드코딩하지 않고 단일 설정 위치에서 주입한다(FR-70·MON5). 개별 제휴 링크는 코드가 아니라 `affiliate.json`에서만 온다(FR-75·MON10).

## [FR-70] AdSense 설정 단일 주입(client id·슬롯 id 플레이스홀더)

- **설명**: AdSense `data-ad-client`(ca-pub-XXXX)와 개별 슬롯 id를 **단일 설정 위치**에서 플레이스홀더 상수로 관리하고, 승인·발급 후 그 한 곳만 실값으로 치환한다. 하드코딩 산재를 금지한다.
- **상세 동작**
  - 정상: 광고 설정은 단일 설정 모듈(예: `adsConfig`)에 `AD_CLIENT`(플레이스홀더 `ca-pub-XXXXXXXXXXXXXXXX`)와 슬롯 id 상수(`AD_SLOT_*`)로 존재한다. 자동광고 스니펫(FR-71)·수동 슬롯(FR-72)은 이 상수만 참조한다. 승인·발급 후 설정 모듈의 값을 치환하면 전 슬롯에 반영된다(재빌드/재배포).
  - 대안/금지: 슬롯별 `.html`·`.js`에 client id·슬롯 id를 직접 문자열로 반복 삽입하지 않는다. 플레이스홀더 상태에서는 실제 광고가 아니라 슬롯 컨테이너(예약 높이)만 유지한다(승인 전, FR-74·MON18).
- **입력**: 없음(빌드/배포 설정). 설정 필드: `AD_CLIENT`(string, `ca-pub-` 접두 16자리 형식, 초기=플레이스홀더), `AD_SLOT_*`(string, 슬롯 식별자, 초기=플레이스홀더).
- **출력·상태**: 전 광고 컴포넌트가 참조하는 단일 client id·슬롯 id 소스. 승인 전에는 플레이스홀더 유지.
- **검증·규칙**: (1) 저장소·정적 산출물 전역 검색에서 client id 리터럴이 설정 모듈 **1곳에만** 존재(산재 0). (2) client id는 시크릿이 아닌 **공개 플레이스홀더**로 취급되어 저장소 스캔 정책과 정합한다(NFR22 "AdSense client id는 플레이스홀더"). (3) 승인 전 상태에서 실제 광고 스크립트가 아닌 예약 슬롯만 렌더(MON18).
- **추적**: UC-60(사전조건 2), F6, NFR22, MON5·MON18, §8.

## [FR-71] 자동광고(Auto ads) 페이지 유형별 조건부 로드

- **설명**: 자동광고 스니펫을 **사이트 전역이 아니라 페이지 유형별 조건부**로만 로드한다. 자동광고 ON 페이지에만 스니펫을 삽입하고, 제외 페이지에는 삽입하지 않는다.
- **상세 동작**
  - 정상: 현재 `page_type`이 배치 표(공통 규약)에서 **자동광고 ON**인 유형(`landing`·`company`·`combo`)일 때만 AdSense 자동광고 스니펫을 삽입하고 `AD_CLIENT`(FR-70)를 주입한다.
  - 대안/금지: `input`은 자동광고 **OFF**(스니펫 미삽입, MON8). `result`는 자동광고 OFF(수동 절제 슬롯만, FR-72·MON9). `policy`는 OFF/최소(얇은 유틸리티 페이지, MON7-6). 이들 페이지에는 자동광고 스크립트를 **로드하지 않는다**.
- **입력**: `page_type`(열거 6종), 동의 상태(FR-78, 개인화/비개인화 신호).
- **출력·상태**: 자동광고 ON 페이지 = 스니펫 1회 삽입(중복 삽입 없음). OFF 페이지 = 스니펫 부재.
- **검증·규칙**: (1) `input`·`result`·`policy` DOM에 자동광고 스니펫이 존재하지 않음(정적/네트워크 검증). (2) 자동광고는 정적 페이지 본문 색인 이후 **클라이언트 스크립트로 주입**되어 비-JS 본문 색인에 영향 없음(UC-60 3b, §2-7). (3) 동의 미획득 시 비개인화/미노출로 폴백(FR-79).
- **추적**: UC-60(기본 흐름 1~3·대안 1a·2a·3b), F6, MON3·MON7·MON8.

## [FR-72] 수동 광고 슬롯(Manual ad unit) 배치

- **설명**: 콘텐츠 페이지의 **지정 위치**(본문 중간·하단)에 명시적 광고 슬롯을 배치하며, 위치·개수는 배치 표를 따른다. 비교 결과 화면은 **리포트 하단 1개**로 제한한다.
- **상세 동작**
  - 정상: `page_type`별 지정 위치에 수동 슬롯을 렌더한다 — `landing`: 콘텐츠 하단 1; `company`·`combo`: 본문 중간 1 + 하단 1; `result`: 리포트(판정 vdCard) **하단 1개만**. 각 슬롯은 `AD_SLOT_*`(FR-70)와 예약 높이(FR-74)를 갖고 "광고" 표기(FR-76)를 부착한다.
  - 대안/금지: `input`·`policy`는 수동 슬롯 **없음**. `result`의 슬롯을 판정 상단·본문 사이에 삽입하지 않는다(판정 열람 방해 금지, MON9).
- **입력**: `page_type`, `AD_SLOT_*`, 슬롯 위치 지정(본문중간/하단/리포트하단).
- **출력·상태**: 페이지별 배치 표에 부합하는 수동 슬롯 집합(개수·위치 준수). 각 슬롯에 "광고" 라벨 부착.
- **검증·규칙**: (1) `result` 화면 수동 슬롯 수 = **정확히 1**(리포트 하단), 상단·중간 삽입 0(MON9). (2) `input`·`policy` 수동 슬롯 0(MON7-4·MON7-6). (3) `company`·`combo`는 본문 중간·하단 각 1(MON7-2·MON7-3). (4) 슬롯 배치가 클릭 유도·콘텐츠 위장 형태가 아니어야 한다(콘텐츠/광고 구분, MON18·FR-76).
- **추적**: UC-60(기본 흐름 2, 대안 1a·2a), F6, NFR19, MON4·MON7·MON9.

## [FR-73] 페이지 유형별 광고 배치 게이팅 매트릭스

- **설명**: 자동광고(FR-71)·수동 슬롯(FR-72)·제휴(FR-75) 컴포넌트의 마운트를 **페이지 유형 판별 하나로 게이팅**하여 배치 표(정본)를 강제한다. 콘텐츠 페이지 위주·비교 입력 무광고·비교 결과 절제를 구현 수준에서 보장한다.
- **상세 동작**
  - 정상: 페이지 진입 시 `page_type`을 판별하고, 아래 결정 규칙으로 각 컴포넌트 마운트 여부를 결정한다.

    ```
    function adPolicy(page_type):
      switch page_type:
        landing -> { auto: ON,  manual: ["content_bottom"],           affiliate: optional }
        company -> { auto: ON,  manual: ["content_mid","content_bottom"], affiliate: on }
        combo   -> { auto: ON,  manual: ["content_mid","content_bottom"], affiliate: on }
        input   -> { auto: OFF, manual: [],                            affiliate: none }   # 무광고 강제
        result  -> { auto: OFF, manual: ["report_bottom"],             affiliate: optional_report_bottom }
        policy  -> { auto: OFF, manual: [],                            affiliate: none }
        default -> { auto: OFF, manual: [],                            affiliate: none }   # 판별 실패 = 안전 기본값
    ```
  - 대안/금지: `input`에서는 자동광고·수동 슬롯·제휴 컴포넌트를 **렌더링하지 않는다**(마운트 자체를 게이팅, MON8). `policy`는 무광고로 AdSense 정책 정합 유지(MON7-6·MON18). 배치 표 외 위치·개수 추가를 금지한다.
- **입력**: `page_type`(열거 6종). 판별 불가 시 `default`.
- **출력·상태**: 페이지별 `{auto, manual[], affiliate}` 마운트 결정. `input`/`policy`는 광고 컴포넌트 0.
- **검증·규칙**: (1) `input` 페이지의 광고·제휴 컴포넌트 마운트 = **0**(자동·수동·제휴 전부, UC-60 1a·UC-61 1b·MON8). (2) 결정 규칙이 배치 표(정본)와 1:1 일치. (3) 판별 실패는 광고 노출이 아니라 **무광고**로 폴백(오노출 방지). (4) 클릭 유도 금지·콘텐츠와 광고 명확 구분·얇은 페이지 광고 회피(MON18).
- **추적**: UC-60(기본 흐름 2·대안 1a·2a)·UC-61(대안 1b), F6, MON7·MON8·MON9·MON18.

## [FR-74] 광고 슬롯 CLS 방지·예약 높이·미노출 폴백

- **설명**: 광고 슬롯 컨테이너는 광고 유닛 크기에 상응하는 **예약 높이/폴백**을 가져, 광고 미노출(미승인·광고 차단기·네트워크 실패·미동의 폴백)에서도 레이아웃이 깨지지 않게 한다(CLS 방지).
- **상세 동작**
  - 정상: 각 슬롯 컨테이너는 삽입될 광고 유닛 규격에 맞춘 **예약 높이**(CSS min-height 등)를 미리 확보한다. 광고가 로드·삽입되어도 예약 높이 범위 안에서 채워져 삽입 전후 레이아웃 이동이 발생하지 않는다(CLS ≤ 0.1).
  - 대안: 광고 미노출(승인 전 플레이스홀더·차단기·네트워크 실패·미동의 미노출 폴백) 시에도 컨테이너는 레이아웃을 유지하거나 **초기 렌더 시점에** 안정적으로 접혀 사후 이동을 유발하지 않는다. 페이지 본문·리포트 판정은 광고 상태와 무관하게 정상 표시된다.
- **입력**: 슬롯 규격(예약 높이), 광고 로드 결과(성공/미노출), 승인 상태(FR-70), 동의 상태(FR-79).
- **출력·상태**: 광고 상태와 무관하게 레이아웃 무손상. 미노출 시 빈 예약 컨테이너 또는 안정적 접힘.
- **검증·규칙**: (1) 광고 로드 실패·차단기·미승인 시 페이지 레이아웃 이동 없음(CLS ≤ 0.1, NFR2·NFR5). (2) 광고 미노출이 본문·리포트 판정 표시를 방해하지 않음(무크래시). (3) 승인 전에는 실제 광고 대신 예약 슬롯만 유지(FR-70·MON18).
- **추적**: UC-60(대안 3a), F6, NFR2·NFR5, MON6.

## [FR-75] 제휴 `affiliate.json` 스키마·데이터 기반 컴포넌트 렌더

- **설명**: 제휴 항목을 **`affiliate.json` 데이터에서 읽어 렌더**하고, 개별 제휴 링크를 코드에 하드코딩하지 않는다. 노출·미노출·링크 교체가 데이터만으로 제어되어 코드 수정이 불필요하다.
- **상세 동작**
  - 정상: 페이지 로드 시 `affiliate.json`을 읽어 현재 `page_type`에 해당하는 활성 항목만 필터링해 제휴 컴포넌트로 렌더한다. 각 항목은 "광고" 표기(FR-76)와 광고성 링크 속성(FR-77)을 적용한다.
  - 대안: 현재 `page_type`에 해당 항목이 없거나 전부 비활성이면 제휴 컴포넌트를 **노출하지 않는다**(UC-61 1a). `input`·`policy`에는 어떤 항목도 렌더하지 않는다(게이팅, FR-73).
- **입력**: `affiliate.json`(정적 자원, 서버 쓰기 없음)와 현재 `page_type`.

  **`affiliate.json` 스키마**

  | 필드 | 타입 | 제약 | 의미 |
  | --- | --- | --- | --- |
  | `version` | integer | ≥ 1 | 스키마 버전 |
  | `items` | array | 0개 이상 | 제휴 항목 목록 |
  | `items[].id` | string | 고유(unique), kebab-case | 항목 식별자 |
  | `items[].label` | string | 비어있지 않음, 한국어 | 표시 라벨(예: "이력서 첨삭 서비스") |
  | `items[].url` | string | `http`/`https` 스킴 필수, 외부 링크 | 제휴 대상 URL |
  | `items[].desc` | string \| null | 선택 | 보조 설명 |
  | `items[].page_types` | string[] | 값 ∈ {`landing`,`company`,`combo`,`result`}, `input`·`policy` **불가** | 노출 대상 페이지 유형 |
  | `items[].active` | boolean | — | `false`면 미노출 |

  ```json
  {
    "version": 1,
    "items": [
      {
        "id": "resume-review",
        "label": "이력서 첨삭 서비스",
        "url": "https://partner.example.com/resume?ref=loupit",
        "desc": "이직 준비 관련 서비스",
        "page_types": ["company", "combo"],
        "active": true
      }
    ]
  }
  ```
- **출력·상태**: 현재 `page_type`에 대해 `active===true && page_types.includes(page_type)`인 항목만 렌더된 제휴 컴포넌트. 해당 없음 시 컴포넌트 부재.
- **검증·규칙**: (1) 코드에 개별 제휴 링크(URL) 하드코딩 0 — 링크는 `affiliate.json`에서만 온다(MON10). (2) `page_types`에 `input`·`policy`가 포함된 항목은 게이팅 위반이므로 렌더 대상에서 제외(FR-73·MON8과 정합). (3) `label`·`desc`·`url`은 렌더 시 HTML 이스케이프, `url`은 `http`/`https` 스킴만 허용(NFR21). (4) 데이터 교체(항목 추가·삭제·`active` 토글·URL 변경)만으로 노출/미노출/링크가 바뀌고 코드 수정이 불필요(F6 수용기준 4). (5) `id` 중복 금지.
- **추적**: UC-61(기본 흐름 1·대안 1a·1b), F6, NFR21·NFR29, MON10, §8.

## [FR-76] 광고·제휴 "광고" 표기(공정위 표시광고 명확성)

- **설명**: 모든 광고(AdSense 슬롯)·제휴 영역에 사용자가 인지 가능한 **"광고"(또는 "유료 광고")** 라벨을 부착하여 콘텐츠와 명확히 구분한다(공정위 표시·광고 규정 취지 준수).
- **상세 동작**
  - 정상: 각 자동광고 영역·수동 슬롯·제휴 컴포넌트에 "광고" 라벨을 인지 가능한 위치·색·크기로 부착한다. 라벨은 광고 영역이 콘텐츠가 아님을 드러낸다(MON11·MON18 콘텐츠/광고 구분).
  - 대안/금지: 라벨을 배경과 구분되지 않는 색·과소 크기로 숨기거나 시각적으로 감추는 배치를 금지한다(표시광고 명확성 요건 위반, UC-61 2a). 라벨 미부착 광고/제휴 렌더를 금지한다.
- **입력**: 광고/제휴 영역 존재(FR-71·FR-72·FR-75), 라벨 문안("광고"/"유료 광고").
- **출력·상태**: 모든 광고·제휴 영역에 "광고" 표기 부착. 라벨 없는 광고 영역 0.
- **검증·규칙**: (1) 렌더된 모든 광고/제휴 영역에 "광고" 라벨 존재(누락 0, NFR19). (2) 라벨은 숨김(hidden·시각 은폐) 처리 금지 — 인지 가능한 색·크기 유지(MON11 주의). (3) 노출만으로도(클릭 없이) 표기 요건 충족(UC-61 3a). (4) 라벨 문안·정확한 위치는 WIREFRAME/SPEC에서 확정하되 본 명확성 규칙을 준수한다.
- **추적**: UC-60(기본 흐름 4)·UC-61(기본 흐름 2·대안 2a·3a), F6, NFR19, MON11·MON18, §2-5.

## [FR-77] 광고성 링크 속성(rel="sponsored nofollow"·noopener)

- **설명**: 외부 제휴/광고 링크에 **광고성 링크 속성**과 안전한 외부 이동 속성을 부여한다. SEO 색인 정책과 유료 링크 표기 규정을 동시에 충족한다.
- **상세 동작**
  - 정상: 제휴 컴포넌트가 렌더하는 외부 링크(`<a>`)에 `rel="sponsored nofollow"`를 부여하고, 새 탭 이동 시 `target="_blank"`와 `rel="noopener"`(및 필요 시 `noreferrer`)를 함께 적용한다. 사용자가 "광고"임을 인지한 상태(FR-76)에서 외부 대상으로 이동한다.
  - 대안: 외부 이동은 loupit 서버에 사용자 데이터를 저장하지 않는다(NFR16). `affiliate.json`의 `url` 스킴이 `http`/`https`가 아니면 링크를 렌더하지 않는다(FR-75와 정합).
- **입력**: 제휴 항목 `url`(FR-75), 링크 렌더 컨텍스트.
- **출력·상태**: 광고성·안전 속성이 적용된 외부 링크. 유료 링크가 SEO 신뢰를 전달하지 않도록 표기.
- **검증·규칙**: (1) 모든 제휴/광고 외부 링크에 `rel`에 `sponsored`와 `nofollow` 포함(MON12). (2) 새 탭 링크는 `rel`에 `noopener` 포함(탭내빙 방지). (3) `url`은 `http`/`https` 스킴만(NFR21). (4) 링크 속성 규정과 "광고" 표기(FR-76)를 동시 충족(§2-7 SEO·§2-5 표기).
- **추적**: UC-61(기본 흐름 4·5), F6, MON12, §2-7.

## [FR-78] 광고·쿠키 개인화 동의 고지·상태·정책 연결

- **설명**: AdSense가 쿠키를 사용하므로 첫 방문 시 **광고/쿠키 사용 고지 및 동의 UI**(동의 배너)를 제공하고, 동의 상태를 **서버 저장 없이** 브라우저(localStorage/쿠키)에만 보관하며, 배너를 정책 페이지로 연결한다.
- **상세 동작**
  - 정상: 첫 방문 시 동의 배너가 광고/쿠키 사용 목적·제3자(Google) 사용 사실을 고지한다(MON13). 배너는 **개인정보처리방침·광고/제휴 고지(F7 정책 페이지)**로 링크하여 동의↔정책을 단일 진실로 연결한다(MON14, UC-63). 사용자가 개인화 광고에 대해 **동의 또는 거부**를 선택하면, 선택 상태가 브라우저(localStorage/쿠키)에만 저장된다.
  - 대안: 재방문 시 저장된 동의 상태를 읽어 배너 재노출 없이 적용한다(기기 내 한정, 서버 계정 복원 아님, UC-62 4a). localStorage/쿠키 비활성·차단 시 상태를 보관하지 못해 매 방문 고지하고 기본은 비개인화/미노출 폴백(FR-79·UC-62 4b). 해외(EEA/UK 등) CMP 요건은 브리프 미확정 → 2차 재검토(창작 금지, 06-수익화.md §5).
- **입력**: 방문 상태(첫/재방문), 저장된 동의 상태(localStorage/쿠키), 사용자 선택(동의/거부).
- **출력·상태**: 동의 상태 = {개인화 동의 | 거부/미동의} 브라우저 로컬 보관(서버 전송·저장 없음). 동의 배너 → 정책 페이지 링크.
- **검증·규칙**: (1) 동의 상태가 서버로 전송·저장되지 않음(네트워크 검증, NFR16·NFR17). (2) 동의 배너에 개인정보처리방침·광고/제휴 고지 링크 존재(MON14, F7 정책 페이지·NFR18). (3) 쿠키·광고 목적·제3자(Google) 사용 사실이 정책 페이지에 명시됨을 전제(F7 소유, 본 FR은 링크 연결만 보장). (4) 미선택 상태는 비개인화/미노출 폴백으로 간주하고 사이트 이용은 정상(FR-79·UC-62 3a).
- **추적**: UC-62(기본 흐름 1~4·대안 3a·4a·4b·1a), F6·F7(연계 UC-63), NFR17·NFR18, MON13·MON14.

## [FR-79] 미동의/비개인화 폴백·재방문 적용

- **설명**: 사용자가 개인화 광고에 동의하지 않아도 사이트 이용이 가능해야 하며, 이 경우 **비개인화 광고 또는 광고 미노출**로 폴백한다. 폴백 상황에서도 슬롯 레이아웃은 무손상(FR-74)이다.
- **상세 동작**
  - 정상: 동의 상태(FR-78)에 따라 광고 종류를 결정한다 — 동의 시 개인화 광고, 거부/미동의 시 **비개인화 광고 또는 광고 미노출**. 어느 선택에서도 검색·비교·열람 등 사이트 이용은 정상 동작한다.
  - 대안: 폴백(비개인화/미노출) 시에도 슬롯 컨테이너는 예약 높이/폴백으로 레이아웃 이동 없이 유지된다(FR-74·MON6·UC-62 6). 미선택 상태는 미동의로 간주해 폴백한다(UC-62 3a). 어떤 폴백에서도 배치·표기 규칙(FR-73·FR-76)은 동일 적용된다(UC-60 4a).
- **입력**: 동의 상태(개인화 동의 | 거부/미동의 | 미선택), `page_type` 배치 결정(FR-73).
- **출력·상태**: 개인화/비개인화/미노출 중 하나의 광고 표현. 폴백 시 레이아웃 무손상, 사이트 이용 정상.
- **검증·규칙**: (1) 미동의/거부에서도 사이트 전 기능 이용 가능(MON15). (2) 폴백은 비개인화 광고 또는 미노출 중 하나이며 레이아웃 이동 없음(FR-74·MON6). (3) 폴백 상태에서도 노출된 광고에는 "광고" 표기가 유지된다(FR-76). (4) 개인화 여부는 UC-62의 클라이언트 동의 상태만을 따르고 서버 상태에 의존하지 않는다(NFR16).
- **추적**: UC-62(기본 흐름 5·6·대안 3a)·UC-60(대안 4a), F6, NFR25, MON15·MON6.
