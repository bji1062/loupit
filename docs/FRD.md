# FRD — loupit

> 이 문서는 **얇은 인덱스**다. 요약과 참조 파일 목록, FR 마스터표(FR-ID), UC→FR 커버리지 매트릭스만 담는다. 각 FR의 상세 동작·입력·출력·검증 규칙은 개별 `FRD/xx.md` 파일에 있으며 여기서는 서술하지 않는다.

## 1. 목적

loupit FRD는 USECASE가 정의한 각 유즈케이스(UC-*)를 **충족시키기 위해 시스템이 제공해야 하는 기능**을 구현 이전 수준에서 정밀 명세한 문서군이다. 13개 문서가 전역 규칙·데이터 계약·기능별 대역(FR-0x·FR-Dx·FR-1x~FR-9x·FR-Ex·FR-10x·11x)을 1:1로 나눠 소유한다. 모든 FR은 자신이 충족하는 UC-*와 상위 기능 F1~F8(및 SC14 참여)을 인용해 **FRD → USECASE → PRD → 브리프** 추적성을 유지하며(명명·추적 규칙은 [FRD/01-개요와-FR규칙.md](FRD/01-개요와-FR규칙.md) §2·§3), 익명 열람·비교는 로그인 없이 유지된다 — 비밀번호·OAuth·소셜 로그인·프로파일러는 영구 제외이며(SC10, FR-01), 복지 등록·수정 기여(SC14)만 무비밀번호 이메일 코드 로그인 + 재직 인증을 요구한다(FR-10x·11x, [13-참여-API.md](FRD/13-참여-API.md)).
