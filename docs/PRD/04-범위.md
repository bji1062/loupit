# 범위 (Scope)

**목적**: loupit 1차 릴리스(MVP)에 포함되는 기능과 명시적으로 제외되는 기능을 확정하여, 하위 문서(USECASE/FRD/FLOW/SPEC/TASK)가 인용할 범위 경계를 고정한다.

**참조 브리프 섹션**: §2(확정 핵심 결정), §3(MVP 범위 F1~F8), §9(문서 규칙·ID). 각 항목은 관련 기능 코드 F#와 결정 번호를 인용한다.

---

## 1. In-scope (MVP 1차 포함)

브리프 §3의 F1~F8을 SC ID로 고정한다. 각 항목은 1차 릴리스 필수 범위다.

| ID | 범위 항목 | 근거 | 요지 |
|----|-----------|------|------|
| SC1 | 회사 검색·선택 | F1 | 이름/별칭 매칭 검색(§6 `/companies/search`, 최대 20건), 현직 a / 이직처 b 대상 선택 |
| SC2 | 비교 입력 폼 | F2 | 연봉, 복지 체크·금액, 근무형태(재택/유연/야근/임금유형), 통근시간, **우선순위 직접 선택**(연봉/워라밸/복지) |
| SC3 | 클라이언트 비교 리포트 | F3 | 총보상·시간가치·워라밸·카테고리별 복지·우선순위 판정. 계산 100% 클라이언트(결정 8) |
| SC4 | 회사 상세 페이지 ~96개(정적) | F4 | 복지 데이터가 있는 회사만 대상. 기업정보·복지·근무형태 + "이 회사로 비교하기" CTA. 빌드타임 생성, SEO 색인(결정 7) |
| SC5 | 인기 비교 조합 페이지(정적) | F5 | 자주 찾는 회사쌍 사전 생성(예: 삼성전자 vs SK하이닉스). 빌드타임 생성, SEO 색인(결정 7) |
| SC6 | 광고·제휴 슬롯 | F6 | AdSense(자동+수동 슬롯) + `affiliate.json` 기반 제휴 컴포넌트(결정 5). 배치는 결정 6 정책 준수 |
| SC7 | 정책 페이지 | F7 | 개인정보처리방침, 이용약관, 면책조항(데이터 정확성), 광고·제휴 고지(결정 5, 표시광고 규정) |
| SC8 | 읽기 전용 API(익명 열람) | F8 | `/api/v1/reference/all`, `/companies/search`, `/companies/{id}`, `/health` — 익명 열람은 GET 전용·무인증(결정 4). 기여용 쓰기·세션 경로는 SC14(INV-1 개정)로 분리 |

### In-scope 경계 제약(결정 인용)

- **비교 계산은 서버로 전송하지 않는다** — 서버는 참조 데이터만 제공(결정 8, SC3).
- **사용자 데이터는 브라우저 localStorage에만** 저장(예: "최근 비교"). 서버 저장 없음(결정 2).
- **광고 배치**: 랜딩·회사 페이지·인기조합·리스트는 배치, 비교 툴 입력 화면은 최소화, 비교 결과 화면은 절제된 슬롯 허용(결정 6, SC6).
- **서버 쓰기는 기여 경로에 한정**(SC14) — 익명 열람·비교는 무쿠키·무세션 GET 유지(SC8, INV-4). 복지 등록·수정만 세션(라우트 의존성, 미들웨어 아님)+재직 인증을 요구하는 쓰기 경로다(INV-1 개정, 2026-07-21 결정).

### 1.1 In-scope 증분 — 인증 기여 (2026-07-21 결정)

loupit은 라이브 운영 중이며, SC14는 MVP-1(SC1~SC8) 이후 추가되는 **증분 범위**다. 열람·비교는 로그인 없이 그대로 유지되고, 복지 데이터 등록·수정에 한해서만 로그인·재직 인증을 요구한다(나무위키식 참여 모델 — 익명 열람 + 인증 기여).

| ID | 범위 항목 | 근거 | 요지 |
|----|-----------|------|------|
| SC14 | 인증 기여자의 복지 등록·수정(참여) | 2026-07-21 결정 · 참여 문서군(USECASE/09 · FRD/13 · SPEC/13, SP-AUTH) | 무비밀번호 이메일 코드 로그인 + 회사 도메인 이메일 재직 인증 후 복지 항목 등록·수정. **열람·비교는 로그인 불필요(익명 유지)**. 서버 쓰기는 이 기여 경로에 한정하며, 세션은 라우트 의존성(미들웨어 아님)으로 표현한다 |

## 2. Out-of-scope (제외)

| ID | 제외 항목 | 분류 | 근거 | 비고 |
|----|-----------|------|------|------|
| SC9 | 프로파일러(가치관 진단 설문 → 페르소나) | **영구 제외** | 결정 3, §3 | 사용자가 우선순위를 직접 선택(SC2)으로 대체. 프로파일러 데이터 이식 안 함(§5) |
| SC10 | 비밀번호 로그인·OAuth/소셜 로그인·소셜피드 | **영구 제외** | 결정 1(2026-07-21 범위 축소), §3 | 비밀번호·소셜(OAuth·카카오/네이버/구글)·JWT·소셜피드 없음. 무비밀번호 이메일 코드 로그인 + 재직 인증은 기여 기능 한정으로 SC14(In-scope)에서 별도 정의 |
| SC11 | 복지 카테고리 가이드 콘텐츠 | 2차 이후 | §3 | 1차 미포함 |
| SC12 | 회사 랭킹/리스트 페이지 | 2차 검토 | §3 | 2차에서 도입 여부 검토(결정 6은 리스트 광고 배치를 전제하나 페이지 자체는 2차) |
| SC13 | 사용자 저장·신고 기능 | 2차 이후 | 결정 1·2, §3 | 서버 측 저장/신고 없음. 클라이언트 localStorage "최근 비교"(SC1~SC3)와 혼동 금지 |

### 제외에 따른 데이터 모델 정리(§4 인용)

여전히 사용하지 않는 테이블: TSOCIAL_ACCOUNT(소셜 로그인 없음), TEMAIL_VERIFICATION(구 URL 토큰 방식 미재사용 — TAUTH_CODE로 대체), TPROFILER_RESULT, TCOMPARISON, TBENEFIT_REPORT(구 제보 워크플로 미재사용 — TBENEFIT_EDIT_LOG로 대체), TCOMPANY_BENEFIT_BADGE_LOG(구 배지로그 미재사용), TCOMPARISON_FEED, TDAILY_STAT, TPOPULAR_CASE, 및 프로파일러 계열(TPROFILE, TPROFILE_JOB_FIT, TJOB_GROUP, TJOB, TPROFILER_QUESTION, TQUESTION_SCENARIO).

**SC14(참여, 2026-07-21) 도입으로 신규 7테이블을 추가한다**(SP-AUTH 대역): TMEMBER(무비밀번호·PII 최소 — 로그인 이메일 + 닉네임뿐), TSESSION(불투명 토큰의 해시만 저장), TAUTH_CODE(로그인·재직 인증 6자리 코드), TCOMPANY_EMAIL_DOMAIN(회사 도메인), TEMPLOY_VERIFICATION(재직 인증 — 회사 이메일 원문 파기·HMAC 해시만 보관), TEMPLOY_VRF_REQUEST(도메인 미등록 회사의 수동 승인 큐), TBENEFIT_EDIT_LOG(편집 이력 append-only). 익명 참조 5종 + TCOMPARE_LOG는 그대로 유지(§4).

## 3. MVP vs 2차 로드맵

| 항목 | ID | MVP(1차) | 2차 이후 | 상태 근거 |
|------|----|:--------:|:--------:|-----------|
| 회사 검색·선택 | SC1 | O | — | F1 |
| 비교 입력(우선순위 직접선택) | SC2 | O | — | F2 |
| 클라이언트 비교 리포트 | SC3 | O | — | F3 |
| 회사 상세 페이지 ~96개 | SC4 | O | — | F4 |
| 인기 비교 조합 페이지 | SC5 | O | — | F5 |
| 광고·제휴 슬롯 | SC6 | O | — | F6 |
| 정책 페이지 | SC7 | O | — | F7 |
| 읽기 전용 API | SC8 | O | — | F8 |
| 복지 카테고리 가이드 콘텐츠 | SC11 | — | O(도입) | §3 제외→2차 |
| 회사 랭킹/리스트 페이지 | SC12 | — | △(검토) | §3 2차 검토 |
| 사용자 저장·신고 기능 | SC13 | — | O(검토) | §3, 단 서버저장은 결정 1과 상충 → 재검토 필요 |
| 프로파일러 | SC9 | X | X | 결정 3, 영구 제외 |
| 비밀번호·OAuth/소셜 로그인·소셜피드 | SC10 | X | X | 결정 1, 영구 제외(2026-07-21 축소) |
| 인증 기여자 참여(복지 등록·수정) | SC14 | — | O | 2026-07-21 결정, 증분 릴리스 |

범례: O=포함, △=도입 여부 검토, X=영구 제외, —=해당 없음.

**주의**: SC9(프로파일러)는 로드맵 어느 단계에서도 재도입하지 않는다(결정 3, 변경 금지). SC10은 2026-07-21 결정으로 **범위가 축소**되었다 — 비밀번호·OAuth/소셜 로그인·소셜피드는 여전히 영구 제외이나, 기여 기능용 **무비밀번호 이메일 코드 로그인 + 재직 인증**은 SC14로 In-scope다(열람·비교는 익명 유지, 서버 쓰기는 기여 경로 한정). SC13(사용자 저장·신고)은 여전히 2차 후보이며, SC14의 편집 이력(TBENEFIT_EDIT_LOG, 공개 이력)과는 다른 기능이다.
