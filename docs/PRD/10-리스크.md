# 리스크와 완화

**목적**: loupit MVP의 사업·제품·법적 리스크를 식별하고, 각 리스크에 영향·가능성 등급과 검증가능한 완화책을 부여하여 하위 문서(FRD/SPEC/TASK)와 운영이 참조할 리스크 경계를 고정한다.

**참조 브리프 섹션**: §2(핵심 결정 5·6·7), §2-1(로그인 전면 제거), §2-2(localStorage 저장), §3(F4·F5·F6·F7, 제외: 신고 기능·리스트 페이지 2차), §4(TCOMPANY_BENEFIT 배지·출처·검증일·만료일 컬럼), §5(복지 SQL 96개 + 유형 프리셋 폴백), §7(복지 불확실성 밴드), §8(AdSense 스니펫·플레이스홀더 client id).

**상위 추적**: 본 문서는 PRD 항목이다. 완화책은 `02-목표와-성공지표.md`의 지표(GM*)·목표(G*)와 `06-수익화.md`의 수익화 요구(MON*)를 인용하며, 신규 요구는 정의하지 않고 기존 ID로 추적한다. 하위 문서는 리스크 ID(R*)를 인용해 완화 구현을 명시한다.

---

## 1. 등급 기준

리스크는 **영향**(발생 시 손실 규모)과 **가능성**(발생 확률)의 2축으로 등급화한다. 모두 3단계.

| 축 | 낮음 | 중간 | 높음 |
|----|------|------|------|
| **영향** | UX·지표 부분 저하, 우회 가능 | 수익·유입 지표 유의미 하락 | 수익원 차단 또는 법적·신뢰 훼손 |
| **가능성** | 완화 전제 시 예외적 | 완화 없으면 발생 가능 | 완화 없으면 발생 개연 큼 |

노출도 = 영향 × 가능성. R1·R2·R4는 수익 인과 사슬(SEO→UV→광고) 직결이므로 최우선 관리 대상이다.

## 3. 리스크 상세

각 리스크의 조기경보 신호(트리거)와 완화 후 잔여 리스크. 트리거 충족 시 완화책을 강화하거나 대응을 개시한다.

| ID | 조기경보 신호(트리거) | 잔여 리스크 |
|----|----------------------|-------------|
| R1 | AdSense 반려 통지; 정책 페이지 누락·오류 사이트 상태 | 재신청까지 수익 개통 지연. 승인 전에는 매출 0 전제(MON18) |
| R2 | GSC 색인율 하락(GM2 < 목표); "크롤링됨-현재 색인 안 됨" 다수; 수동 조치 통지 | 프리셋 폴백 회사(실복지 SQL 없는 ~104개)는 차별화 여지가 제한적이라 저정보 잔존 가능 |
| R3 | 사용자 문의·정정 요청 유입; est 비중 높은 회사에서 값 편차 제보 | 만료(EXPIRES_DTM 경과) 복지의 시점 불일치. 밴드·배지로 고지하나 오독 가능 |
| R4 | 월 UV 곡선이 가정치(GM1) 하회; organic 신규 유입 정체 | SEO는 외부(검색엔진 알고리즘) 종속이라 통제 불가 요인 잔존. 유료 채널 없음(MVP) |
| R5 | 특정 기업 반복 이의; 법무성 연락 | 신고 기능 부재(§3)로 접수는 정책 페이지 연락 경로에 의존. 정정 반영은 배포 주기에 종속 |
| R6 | 이탈률 상승(GM6 악화); CLS 저하(GM7 < 양호); 비교 완료율 하락(GM5) | 자동광고(MON3) 배치 위치는 Google 재량이라 콘텐츠 페이지 내 국소 UX 변동 잔존 |

**상충 조정 원칙**: R6(광고 UX) 완화와 수익(광고 노출 GM4) 사이 상충 시, 결정 6(비교 입력 최소화·콘텐츠 페이지 위주)을 우선 기준으로 삼는다. R2(중복 콘텐츠 회피)와 F5(조합 페이지 대량 생성) 상충 시, 페이지 고유성(고유 데이터·canonical)을 색인 규모보다 우선한다.

**하위 문서 인용 규칙**: FRD는 R3(배지·출처·밴드 표시)·R6(광고 게이팅)을, SPEC/TASK는 R1(MON5·MON16~18)·R2(canonical·정적 생성기 고유성 검증)·R5(정정 반영 데이터 갱신)을 인용해 구현·검증한다.
