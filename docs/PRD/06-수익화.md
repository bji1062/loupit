# 수익화 모델 (Monetization)

**목적**: loupit의 수익원(Google AdSense + 제휴)과 광고 배치·동의·AdSense 승인 요건을 확정하여, 하위 문서(USECASE/FRD/FLOW/SPEC/TASK)가 인용할 수익화 요구사항 경계를 고정한다.

**참조 브리프 섹션**: §2.5(수익화=AdSense+제휴, '광고' 표기), §2.6(광고 배치=콘텐츠 페이지 위주·비교 입력 최소화), §2.7(SEO 콘텐츠=AdSense 승인 기반), §3(F6 광고·제휴 슬롯, F7 정책 페이지, SC12 리스트 2차 검토), §8(AdSense 스니펫·플레이스홀더 client id·affiliate.json). 각 요구사항은 결정 번호와 기능 코드 F#를 인용한다.

---

## 1. 수익 모델 개요

| ID | 요구사항 | 근거 | 요지 |
|----|----------|------|------|
| MON1 | 수익원은 광고와 제휴 2종으로 한정 | 결정 5, F6 | (1) Google AdSense, (2) `affiliate.json` 기반 제휴 링크. **유료 구독·결제·유료회원은 없음**(로그인 전면 제거, 결정 1) |
| MON2 | 수익의 뼈대는 SEO 콘텐츠 | 결정 7, F4·F5 | 회사 상세 ~96개(복지 데이터가 있는 회사) + 인기 비교 조합 정적 페이지가 검색 유입과 광고 노출·AdSense 승인 콘텐츠를 동시에 공급. 광고 자체가 아니라 콘텐츠가 수익 기반 |

제약: 사용자 데이터를 수익화(판매·서버 저장)하지 않는다. 사용자 입력은 브라우저 localStorage에만 존재하며 서버로 전송·저장하지 않는다(결정 1·2).

## 3. 광고·제휴 배치 원칙 (페이지 유형별)

**원칙**(결정 6): 광고는 **콘텐츠 페이지 위주**로 배치한다. **비교 툴 입력 화면에서는 광고를 최소화**(UX 보호), 비교 결과 화면에는 **절제된 슬롯**만 허용한다.

| ID | 페이지 유형 | 대응 기능 | 자동광고 | 수동 슬롯 | 제휴 컴포넌트 | 광고 밀도 | 근거 |
|----|-------------|-----------|:--------:|-----------|:-------------:|:---------:|------|
| MON7-1 | 랜딩(홈) | 랜딩 | ON | 콘텐츠 하단 1개 | 선택 | 중 | 결정 6(콘텐츠 페이지) |
| MON7-2 | 회사 상세 페이지(정적) | F4 | ON | 본문 중간 1 + 하단 1 | O(관련 서비스) | 상 | 결정 6·7, 핵심 SEO 콘텐츠 |
| MON7-3 | 인기 비교 조합 페이지(정적) | F5 | ON | 본문 중간 1 + 하단 1 | O(관련 서비스) | 상 | 결정 6·7, 핵심 SEO 콘텐츠 |
| MON7-4 | 비교 입력 화면 | F2 | **OFF** | **없음** | 없음 | **없음** | 결정 6(입력 화면 최소화, UX 보호) |
| MON7-5 | 비교 결과 화면 | F3 | OFF | 절제된 1개(리포트 하단) | 선택(리포트 하단) | 저 | 결정 6(절제된 슬롯 허용) |
| MON7-6 | 정책 페이지 | F7 | OFF/최소 | 없음 | 없음 | 없음 | 얇은 유틸리티 페이지, AdSense 정책 정합성 |
| MON7-7 | 회사 랭킹/리스트 페이지 | SC12(2차) | ON | 리스트 중간·하단 | 선택 | 중 | 결정 6은 리스트 배치를 전제하나 **페이지 자체는 2차 검토**(§3). MVP 미포함 |

범례: O=배치, 선택=콘텐츠 적합 시 배치, ON/OFF=자동광고 로드 여부, 밀도(없음<저<중<상).

관련 보조 요구사항:

| ID | 요구사항 | 근거 | 요지 |
|----|----------|------|------|
| MON8 | 입력 화면 무광고를 구현 수준에서 강제 | 결정 6, MON7-4 | 비교 입력 뷰에서는 자동광고 스니펫·수동 슬롯·제휴 컴포넌트를 **렌더링하지 않는다**. 페이지 유형 판별로 광고 컴포넌트 마운트를 게이팅 |
| MON9 | 결과 화면 슬롯은 리포트 하단 1개로 제한 | 결정 6, MON7-5 | 리포트 판정(vdCard) 상단·본문 사이 삽입 금지. 판정 열람을 방해하지 않는 하단 1슬롯만 |

## 4. 제휴(Affiliate) 모델

| ID | 요구사항 | 근거 | 요지 |
|----|----------|------|------|
| MON10 | 제휴는 `affiliate.json` 데이터 기반 컴포넌트로 렌더 | 결정 5, §8, F6 | 제휴 항목(라벨·링크·대상 페이지 유형 등)을 `affiliate.json`에서 읽어 렌더. 코드에 개별 제휴 링크 하드코딩 금지. 스키마 상세는 FRD/SPEC에 위임 |
| MON11 | 모든 제휴·광고 링크에 "광고" 표기 명시 | 결정 5 | 제휴 컴포넌트 및 유료 배치에 **"광고"(또는 "유료 광고")** 라벨을 사용자가 인지 가능하게 표기. 공정위 표시·광고 규정(추천·보증 등에 관한 표시·광고 심사지침 취지) 준수 |
| MON12 | 제휴 링크는 광고성 링크 속성 부여 | 결정 5, 결정 7(SEO) | 외부 제휴/광고 링크에 `rel="sponsored nofollow"`, 외부 이동 `target`/`rel="noopener"` 적용. SEO 색인 정책과 유료 링크 표기 규정 동시 충족 |

주의: 제휴 표기 라벨은 눈에 띄지 않는 색·크기로 숨기지 않는다(표시광고 규정의 명확성 요건). "광고" 표기 문안·위치는 WIREFRAME/SPEC에서 확정한다.

## 5. 광고·쿠키 동의 (Consent)

| ID | 요구사항 | 근거 | 요지 |
|----|----------|------|------|
| MON13 | 광고·분석 쿠키 동의 고지 제공 | 결정 5, F7 | AdSense는 쿠키를 사용하므로, 첫 방문 시 광고/쿠키 사용 고지 및 동의 UI를 제공한다. 동의 상태는 서버 저장 없이 브라우저(localStorage/쿠키)에만 보관(결정 1·2) |
| MON14 | 동의·정책의 단일 진실 연결 | F7 | 동의 배너는 개인정보처리방침·광고/제휴 고지(F7 정책 페이지)로 링크한다. 쿠키·광고 목적·제3자(Google) 사용 사실을 정책 페이지에 명시 |
| MON15 | 미동의/비개인화 폴백 | 결정 5 | 사용자가 개인화 광고에 동의하지 않아도 사이트 이용이 가능해야 하며, 이 경우 비개인화 광고 또는 광고 미노출로 폴백한다(레이아웃은 MON6 준수) |

비고: MVP는 한국 사용자 대상이나, 해외(EEA/UK 등) 유입 시 Google이 요구하는 동의 처리(CMP) 필요 여부는 실제 트래픽·정책에 따라 2차에서 재검토한다(브리프 미확정 → 창작 금지).

## 6. AdSense 승인 요건

승인은 수익화의 전제 조건이다. 아래 요건을 릴리스 전 충족한다.

| ID | 요구사항 | 근거 | 충족 방식 |
|----|----------|------|-----------|
| MON16 | 승인 가능한 콘텐츠 분량 확보 | 결정 7, F4·F5 | 회사 상세 ~96개(복지 데이터가 있는 회사) + 인기 비교 조합 정적 페이지를 빌드타임 생성하여 독창적·탐색 가능한 콘텐츠 볼륨 확보. 승인 신청은 정적 콘텐츠 배포 후 진행 |
| MON17 | 필수 정책 페이지 게시 | F7 | 개인정보처리방침·이용약관·면책조항(데이터 정확성)·광고/제휴 고지를 게시하고 전역 내비게이션(푸터 등)에서 접근 가능하게 한다 |
| MON18 | AdSense 프로그램 정책 정합성 | 결정 5·6 | 클릭 유도 금지, 콘텐츠와 광고 명확 구분, 얇은 페이지 광고 회피(MON7-6), 정상 탐색 구조·오류 없는 사이트 유지. 승인 전에는 실제 광고 대신 슬롯 플레이스홀더 유지(MON5) |
