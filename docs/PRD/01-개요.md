# 제품 개요

**목적**: loupit 재설계의 배경·문제정의·비전·핵심 가치제안을 정의하여 하위 문서(USECASE/FRD 등)가 참조할 상위 근거를 확정한다.

**참조 브리프 섹션**: 1(제품 정체성), 2(확정 핵심 결정), 3(MVP 범위), 5(보존 실데이터).

---

## 1. 제품 정체성 요약

| 항목 | 내용 |
| --- | --- |
| 이름 | jobcho.wiki (도메인 jobcho.wiki, HTTPS) |
| 한 줄 정의 | 로그인 없는 한국 회사 복지·연봉·근무조건 비교 도구 + 회사별 정보 페이지. 광고로 수익화 |
| 기원 | 기존 `/home/ubuntu/job_change`("직장 선택 OS", 로그인 기반 풀스택)의 재설계 |
| 언어 | UI/콘텐츠 한국어, 코드 영어 |
| 수익 모델 | Google AdSense(자동광고 + 수동 슬롯) + 제휴(affiliate) 링크 |

## 2. 배경: 왜 재설계하는가

기존 job_change는 회원가입·로그인·OAuth·JWT·소셜피드·프로파일러(가치관 진단)를 갖춘 로그인 기반 풀스택 제품이었다. 코드는 완전히 새로 작성하되, 큐레이션된 회사·복지 데이터(기업유형 6종, 복지 데이터가 있는 회사 ~96개, 상세 복지 SQL 96개 파일)는 보존한다. 재설계의 두 축은 다음과 같다.

1. **로그인 마찰 제거**: 비교 도구를 쓰기 위한 회원가입·로그인 단계를 없애 진입장벽을 제거한다. 인증·회원·소셜피드를 전면 제거하고, 사용자 데이터는 서버에 저장하지 않고 브라우저 localStorage("최근 비교" 등)에만 둔다.
2. **광고 수익화 전환**: 로그인 기반의 불명확한 수익 경로를 AdSense + 제휴 링크 기반 광고 모델로 전환한다. 이를 위해 회사별 정적 페이지 ~96개(복지 데이터가 있는 회사)와 인기 비교 조합 페이지를 빌드타임 생성하여 검색 색인·AdSense 승인 콘텐츠를 확보한다.

부수적으로, 인증 미들웨어·쓰기 엔드포인트·프로파일러 등 상태ful 기능을 제거하고 백엔드를 **슬림 읽기 전용 FastAPI + MySQL**로 단순화하여 운영·유지보수 부담을 낮춘다.

## 3. 문제 정의

| # | 문제 (기존/시장) | loupit의 해결 |
| --- | --- | --- |
| P1 | 로그인·회원가입이 단순 비교 사용의 진입장벽이 되어 이탈·검색유입 손실 유발 | 로그인 0단계. 회원가입 없이 즉시 비교, 개인정보 서버 저장 없음 |
| P2 | 수익 경로 불명확(사용자 데이터 수집형 풀스택) | 콘텐츠 페이지 기반 AdSense + 제휴 링크로 수익화 |
| P3 | 인증·회원·소셜피드·프로파일러로 인한 코드·인프라 복잡도와 유지보수 부담 | 읽기 전용 슬림 백엔드(쓰기 엔드포인트·인증 미들웨어 없음) |
| P4 | 진단 설문(프로파일러) 의존이 사용 흐름을 무겁게 함 | 프로파일러 제외, 사용자가 비교 우선순위(연봉/워라밸/복지)를 직접 선택 |
| P5 | 흩어진 회사 복지·근무조건 정보 탐색 비용 | 큐레이션 데이터를 회사 상세 페이지·검색·비교로 제공 |

> 데이터 정확성은 완전 보장 대상이 아니며, 복지 정보에는 공식/추정 배지와 불확실성 밴드를 표기하고 면책조항을 제공한다.

## 4. 비전

loupit은 한국 직장인이 로그인이나 개인정보 제공 없이도, 큐레이션된 실데이터를 바탕으로 두 직장(현직 vs 이직처)의 총보상·시간가치·워라밸·복지를 자기 기준으로 즉시 비교하고, 회사별 복지·근무조건 정보를 검색으로 찾아볼 수 있는 무료 도구가 되는 것을 지향한다. 사용자에게는 마찰 없는 의사결정 보조를, 운영자에게는 SEO 트래픽에 기반한 광고·제휴 수익 구조를 제공한다.

## 5. 핵심 가치제안

각 항목은 하위 문서가 인용할 수 있도록 안정적 ID를 부여한다.

| ID | 가치제안 | 근거 |
| --- | --- | --- |
| PRD-OV-1 | **로그인 없는 즉시 사용·프라이버시** — 회원가입/로그인 없이 바로 비교. 사용자 데이터는 서버 미저장, 브라우저 localStorage에만 보관 | 인증·회원 전면 제거, localStorage 정책 |
| PRD-OV-2 | **실증 데이터 기반 비교** — 복지 데이터가 있는 회사(약 96개) + 상세 복지 데이터, 공식/추정 배지와 불확실성 밴드로 신뢰 수준을 명시 | 보존 큐레이션 데이터, 복지 배지·밴드 |
| PRD-OV-3 | **사용자 주도 우선순위 + 클라이언트 계산** — 진단 설문 없이 사용자가 우선순위를 직접 선택하고, 비교 계산은 100% 클라이언트에서 수행(빠름·프라이버시) | 프로파일러 제외, 클라이언트 사이드 계산 |
| PRD-OV-4 | **SEO 자산 기반 광고 수익 구조** — 회사·인기 조합 정적 페이지로 검색 유입을 만들고 AdSense·제휴로 수익화. 광고는 콘텐츠 페이지 위주로 배치하고 비교 입력 화면 광고는 최소화 | SEO 정적 페이지, 광고 배치 정책 |
| PRD-OV-5 | **투명한 광고·데이터 고지** — 광고/제휴에 "광고" 표기(공정위 표시광고 규정 준수), 데이터 정확성 면책 및 정책 페이지 제공 | 광고 표기 정책, 정책 페이지(F7) |

## 6. 대상·범위 경계

- **대상 사용자**: 이직/재직 판단을 위해 두 직장의 보상·근무조건을 비교하려는 한국 직장인, 특정 회사 복지·근무형태를 찾는 검색 유입 사용자.
- **범위**: MVP 상세는 별도 문서(범위/기능 정의)에서 다룬다. 프로파일러는 영구 제외한다. 사용자 계정·저장·신고 기능, 회사 랭킹/리스트 페이지 등은 본 릴리스에서 제외(2차 이후 검토).
