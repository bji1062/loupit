# 타깃 사용자

**목적**: loupit의 핵심 사용자 세그먼트·대표 페르소나·이용 맥락을 정의하여, 이후 USECASE/FRD/FLOW 문서의 사용자 기준점을 제공한다.

**참조한 브리프 섹션**: §1(제품 정체성), §2(핵심 결정: 로그인 제거·localStorage), §3(MVP 범위 F1~F8·우선순위 직접 선택), §7(비교 알고리즘 입력: 현직 a / 이직처 b), §8(기술 스택: 모바일·SEO 정적 페이지).

**상위 추적**: 본 문서는 PRD 항목이며, 항목 ID는 문서 구분 접두사로 부여한다 — 세그먼트 `PRD-SEG-*`, 페르소나 `PRD-PER-*`, 니즈 `PRD-NEED-*`, 이용 맥락 `PRD-CTX-*`. 하위 USECASE(UC-*) 문서가 본 문서의 사용자 ID를 인용한다(USECASE 자체 ID인 UC-*와 혼동하지 않도록 구분한다).

---

## 1. 개요

loupit은 **로그인 없이** 한국 회사의 복지·연봉·근무조건을 비교하는 도구이자, 회사별 정보 페이지 모음이다(§1). 따라서 타깃은 "가입 절차 없이, 검색으로 유입되어, 모바일에서 즉시 정보를 얻으려는 사용자"다. 사용자별 서버 계정·저장이 없으므로(§2), 모든 세그먼트는 **1회성·비로그인 방문**을 기본 전제로 한다.

## 2. 사용자 세그먼트

| ID | 세그먼트 | 정의 | 주 진입점(MVP) | 핵심 동기 |
|----|---------|------|---------------|-----------|
| PRD-SEG-1 | 이직 고려 직장인 | 현직 재직 중이며 특정 회사로의 이직을 저울질하는 사용자 | 비교 툴(F2), 회사 상세(F4)의 "이 회사로 비교하기" CTA | 현직(a) vs 이직처(b)의 실질 처우 차이 확인 |
| PRD-SEG-2 | 신입/경력 오퍼 비교자 | 복수 오퍼 또는 오퍼 vs 현직을 비교해 수락 여부를 결정하려는 사용자 | 비교 툴(F2), 인기 비교 조합(F5) | 총보상·워라밸·복지 우선순위(연봉/워라밸/복지)에 따른 판정 |
| PRD-SEG-3 | 회사 복지 정보 검색 유입자 | 특정 회사의 복지·근무형태를 알아보려 검색엔진에서 유입된 사용자 | 회사 상세 ~96p(복지 있는 회사, F4), 인기 비교 조합(F5) — SEO 정적 페이지 | 한 회사의 복지·근무형태·기업정보 열람 |

세그먼트 우선순위: PRD-SEG-1·PRD-SEG-2가 비교 툴(F2/F3)의 핵심 사용자, PRD-SEG-3은 SEO 유입 및 광고 노출(F4/F5/F6)의 주 트래픽원. 세 세그먼트는 배타적이지 않으며, PRD-SEG-3이 비교 CTA를 통해 PRD-SEG-1/PRD-SEG-2로 전환되는 흐름을 가정한다.

## 3. 대표 페르소나

간결한 대표상 3인. 각 페르소나는 세그먼트 하나에 대응한다.

| ID | 페르소나 | 대응 세그먼트 | 상황 | 기대 결과 |
|----|---------|-------------|------|-----------|
| PRD-PER-1 | 3년차 개발자 "지원" | PRD-SEG-1 | 현직에 만족하지만 스타트업 오퍼를 받고 연봉 상승분이 복지·야근 감소를 상쇄하는지 불확실 | 실효연봉·시간당 가치·워라밸 비교로 "옮길 가치가 있나" 판단 |
| PRD-PER-2 | 취업 준비생 "민재" | PRD-SEG-2 | 대기업·중견 오퍼 2곳을 놓고 브랜드보다 워라밸을 우선하고 싶음 | 우선순위를 워라밸로 직접 선택(F2)한 뒤 판정 카드로 선택지 좁히기 |
| PRD-PER-3 | 정보 탐색자 "수아" | PRD-SEG-3 | 지인이 언급한 회사의 복지·재택 여부가 궁금해 검색으로 유입 | 회사 상세 페이지에서 복지·근무형태 확인 후 "이 회사로 비교하기"로 전환 |

페르소나 공통점: **로그인 의향 없음**, 모바일 우선, 짧은 세션. 이는 창작이 아니라 §2(계정 없음)·§8(모바일·바닐라 JS)에서 도출된 제약이다.

## 4. 핵심 니즈

| ID | 니즈 | 근거(브리프) | 관련 세그먼트 |
|----|------|-------------|--------------|
| PRD-NEED-1 | 가입·로그인 없이 즉시 이용 | §2-1(인증 전면 제거) | PRD-SEG-1, PRD-SEG-2, PRD-SEG-3 |
| PRD-NEED-2 | 두 직장(현직 a / 이직처 b)의 처우를 한 화면에서 비교 | §3-F2·F3, §7 | PRD-SEG-1, PRD-SEG-2 |
| PRD-NEED-3 | 자신이 중요시하는 축(연봉/워라밸/복지)을 직접 골라 판정 | §2-3(프로파일러 제외·직접 선택), §3-F2 | PRD-SEG-1, PRD-SEG-2 |
| PRD-NEED-4 | 특정 회사의 복지·근무형태·기업정보를 검색으로 바로 열람 | §3-F4·F5 | PRD-SEG-3 |
| PRD-NEED-5 | 최근 비교 기록을 기기 내에서 유지(서버 저장 없음) | §2-2(localStorage) | PRD-SEG-1, PRD-SEG-2 |

## 5. 이용 맥락 (설계 가정)

아래는 위 세그먼트·니즈에서 도출한 이용 맥락 가정이며, FLOW/WIREFRAME이 준수할 제약이다.

| ID | 맥락 가정 | 함의 |
|----|----------|------|
| PRD-CTX-1 | 주 디바이스는 **모바일 웹** | 비교 입력·리포트·회사 페이지 모두 모바일 우선 레이아웃. §8(빌드 없는 바닐라 JS) |
| PRD-CTX-2 | 주 진입은 **검색엔진 유입**(랜딩 경유 아님) | 회사 상세/조합 페이지가 독립 진입점(F4/F5)이며 자체적으로 CTA 제공. §3, §2-7(SEO) |
| PRD-CTX-3 | 사용자는 **로그인·개인정보 제공을 원치 않음** | 개인화·저장은 전부 클라이언트(localStorage)로 처리, 서버 전송 없음. §2-1·§2-2 |
| PRD-CTX-4 | **1회성·짧은 세션**, 재방문 시 계정 복원 없음 | 상태 복원은 localStorage "최근 비교"에 한정, 크로스 디바이스 동기화 미제공. §2-2 |

## 6. 비타깃 (범위 밖)

- 프로파일러(가치관 진단)로 페르소나 추천을 기대하는 사용자 — 해당 기능 **영구 제외**(§3, §2-3).
- 계정·즐겨찾기·신고·소셜 피드를 원하는 사용자 — MVP 및 이후 범위에서 계정 기능 제외(§2-1, §3 제외 항목).
- 채용 공고 탐색·지원을 목적으로 하는 사용자 — loupit은 비교·정보 도구이며 채용 플랫폼이 아님(§1).
