# 데이터 요구사항

**목적**: loupit이 제공·유지하는 참조 데이터(회사·복지·유형·프리셋)의 범위, 구조, 신뢰도 표기, 정확성·갱신 정책을 확정한다.

**참조 브리프 섹션**: §2(핵심 결정), §4(유지 데이터 모델), §5(보존 실데이터), §6(읽기 전용 API), §8(빌드타임 생성).

**상위 추적성**: 본 항목은 PRD 상위 결정 §2.1(로그인/회원 제거), §2.7(SEO 정적 페이지), §5(큐레이션 데이터 보존)에 근거한다. 하위 문서(FRD/SPEC)는 D1~D6 ID를 인용한다.

---

## 개요

loupit의 데이터는 **읽기 전용 참조 데이터**만으로 구성된다. 사용자별 데이터·트랜잭션 데이터는 서버에 존재하지 않으며(§2.1), 사용자 입력·비교 기록은 브라우저 localStorage에만 남는다(§2.2). 서버 DB(MySQL 8.0 utf8mb4)는 큐레이션된 회사·복지 정보만 보관하고, `/api/v1/reference/all` 번들과 회사 상세 API로 노출된다(§6).

## D1. 유지 참조 테이블 (5종)

기존 스키마 명명규칙 준수(§4): 테이블 접두사 `T`, 전 컬럼 대문자, PK=INT AUTO_INCREMENT, 감사컬럼 4종(`INS_ID`/`INS_DTM`/`MOD_ID`/`MOD_DTM`) 전 테이블 공통, 금액 단위=만원, ENUM 금지(VARCHAR+COMMENT), 전 컬럼 한국어 COMMENT 필수.

| ID | 테이블 | 역할 | PK | 주요 컬럼 | 제약 |
|----|--------|------|----|-----------|------|
| D1.1 | `TCOMPANY_TYPE` | 기업유형 6종 마스터 | `COMP_TP_ID` | `COMP_TP_CD`, `COMP_TP_NM`, `GROWTH_RATE_VAL`(DECIMAL, 연평균 상승률), `GROWTH_LABEL_NM`, `STABILITY_SCORE_NO`(1~100) | `COMP_TP_CD` ∈ {large, startup, mid, foreign, public, freelance} |
| D1.2 | `TCOMPANY` | 회사 마스터 | `COMP_ID` | `COMP_ENG_NM`(영문식별자), `COMP_NM`(정식명), `COMP_TP_ID`(FK), `INDUSTRY_NM`, `LOGO_NM`, `WORK_STYLE_VAL`(JSON), `CAREERS_BENEFIT_URL` | `COMP_ENG_NM` unique, `COMP_NM` unique+FULLTEXT |
| D1.3 | `TCOMPANY_ALIAS` | 검색 별칭 | `ALIAS_ID` | `COMP_ID`(FK), `ALIAS_NM` | unique(`COMP_ID`,`ALIAS_NM`) |
| D1.4 | `TCOMPANY_BENEFIT` | 회사별 복지 | `BENEFIT_ID` | 하단 D3 참조(배지·출처·만료 포함) | unique(`COMP_ID`,`BENEFIT_CD`) |
| D1.5 | `TBENEFIT_PRESET` | 기업유형별 기본복지 | `PRESET_ID` | `COMP_TP_ID`(FK), `BENEFIT_CD`, `BENEFIT_NM`, `BENEFIT_AMT`, `BENEFIT_CTGR_CD`, `BADGE_CD`, `DEFAULT_CHECKED_YN`, `SORT_ORDER_NO` | 유형별 복지 폴백 소스 |

**D1.6 — WORK_STYLE_VAL(JSON) 키**: `remote`, `flex`, `unlimitedPTO`, `refreshLeave`, `overtime`. 회사 상세·비교 근무형태 계산에 사용.

**D1.7 — 복지 카테고리 코드(`BENEFIT_CTGR_CD`) 9종**: `compensation`, `flexibility`, `work_env`, `time_off`, `health`, `family`, `growth`, `leisure`, `perks`. `TCOMPANY_BENEFIT`·`TBENEFIT_PRESET` 공통 사용, 비교 리포트의 카테고리별 델타 집계 기준.

**D1.8 — 제거 테이블(생성 금지)**: `TMEMBER`, `TSOCIAL_ACCOUNT`, `TEMAIL_VERIFICATION`, `TPROFILER_RESULT`, `TCOMPARISON`, `TBENEFIT_REPORT`, `TCOMPANY_BENEFIT_BADGE_LOG`, `TCOMPARISON_FEED`, `TDAILY_STAT`, `TPOPULAR_CASE`, 및 프로파일러 계열 `TPROFILE`, `TPROFILE_JOB_FIT`, `TJOB_GROUP`, `TJOB`, `TPROFILER_QUESTION`, `TQUESTION_SCENARIO`. 신규 스키마·시드·API에서 참조 불가.

## D2. 보존할 실데이터 (큐레이션 재이식)

기존 `/home/ubuntu/job_change` 자산에서 아래 데이터만 재이식한다(§5). 코드는 새로 작성하되 데이터는 보존.

| ID | 데이터 | 규모 | 소스 | 폴백 |
|----|--------|------|------|------|
| D2.1 | 기업유형 | 6종 | 큐레이션 상수 | 없음 |
| D2.2 | 유형별 복지 프리셋 | 6유형 각각 | 큐레이션 상수 → `TBENEFIT_PRESET` | 회사 상세 복지의 폴백 소스 |
| D2.3 | 회사 마스터 | 200개 (KOSPI 100 + KOSDAQ 100) | 큐레이션. 각 {영문id, 정식명, 별칭[], 유형, 산업, 로고} | — |
| D2.4 | 회사별 상세 복지 | SQL 96개 파일 | `job_change/server/seed/benefit/sql/*.sql` (회사별 실제 복지, official/est 배지 포함) | 상세 복지 없는 나머지 104개 회사는 D2.2 유형 프리셋으로 폴백 |

**D2.5 — 폴백 규칙**: 회사 상세(`/api/v1/companies/{id}`)와 reference 번들은 `TCOMPANY_BENEFIT`에 회사별 실데이터가 있으면 그것을, 없으면 해당 회사 유형의 `TBENEFIT_PRESET`를 병합하여 반환한다. 프리셋 폴백 복지는 배지가 프리셋 기본값(주로 est)으로 표기된다.

**D2.6 — 검증 기준(정량)**: 재이식 완료 = `TCOMPANY` 200행 + `TCOMPANY_TYPE` 6행 + 96개 SQL 파일이 `TCOMPANY_BENEFIT`에 반영 + 200개 회사가 각각 상세 복지 또는 프리셋 폴백 중 하나로 비어있지 않은 복지 목록을 반환. 별칭은 회사당 1건 이상.

## D3. 배지·신뢰도 모델 (`TCOMPANY_BENEFIT`)

각 복지 항목은 데이터 신뢰도를 표기하는 배지·출처·검증/만료 메타데이터를 가진다(§4 `TCOMPANY_BENEFIT`).

| ID | 컬럼 | 의미 | 값/규칙 |
|----|------|------|---------|
| D3.1 | `BADGE_CD` | 신뢰도 배지 | `official`=공식 확인, `est`=추정 |
| D3.2 | `BADGE_SRC_CD` | 출처 유형 | `scrape_official`, `scrape_fallback`, `ai_parse`, `manual`, `user_report` |
| D3.3 | `BADGE_SRC_URL_CTNT` | 출처 URL | 공식 페이지 등 근거 링크(정성 텍스트) |
| D3.4 | `VERIFIED_DTM` | 검증 시점 | 마지막으로 사실 확인한 일시 |
| D3.5 | `EXPIRES_DTM` | 만료 시점 | 신뢰도 유효기간. 경과 시 "만료" 상태로 취급 |
| D3.6 | `BENEFIT_AMT` | 금액(만원) | 정성적 복지는 NULL |
| D3.7 | `QUAL_YN` / `QUAL_DESC_CTNT` | 정성 여부·설명 | 금액화 불가 복지의 서술 |
| D3.8 | `NOTE_CTNT` / `SORT_ORDER_NO` | 비고·정렬순서 | 표시 보조 |

**D3.9 — 신뢰도 → 불확실성 밴드 매핑**: 클라이언트 비교 계산은 배지·만료 상태에 따라 복지 금액에 불확실성 밴드를 적용한다(§7).

| 상태 | 밴드(±) |
|------|---------|
| `official` (미만료) | ±5% |
| `est` (미만료) | ±20% |
| 만료(`EXPIRES_DTM` 경과, 배지 무관) | ±15% |

**D3.10 — 표기 요구**: 회사 상세·비교 리포트 UI는 각 복지의 배지(official/est)와 만료 여부를 시각적으로 구분해 노출해야 한다. 프리셋 폴백 복지는 회사 공식 확인이 아님을 배지로 드러낸다(D2.5).

## D4. 데이터 정확성·면책·갱신 정책

| ID | 정책 | 내용 |
|----|------|------|
| D4.1 | 정확성 면책 | 복지·연봉·근무조건 데이터는 참고용이며 실제와 다를 수 있다. 면책조항 정책 페이지(F7)에 명시. 사용자 채용 의사결정의 최종 확인 책임은 사용자에게 있음을 고지. |
| D4.2 | 갱신 방식 | 데이터 갱신은 **빌드타임 재생성**으로만 이뤄진다. 시드/스크래핑 → DB 재시드 → 정적 회사·조합 페이지(200개+인기 조합) + `sitemap.xml`를 파이썬 생성 스크립트로 재빌드(§8). 런타임 쓰기 경로 없음(§2.4 쓰기 엔드포인트 없음). |
| D4.3 | 신선도 표기 | 복지 항목의 `VERIFIED_DTM`/`EXPIRES_DTM`을 근거로 신선도·만료를 표기(D3.4~D3.5). 만료 데이터는 불확실성 밴드 확대(D3.9)로 계산에 반영. |
| D4.4 | 캐시 정합성 | `/api/v1/reference/all`은 1시간 캐시(`Cache-Control: public, max-age=3600`, §6). 데이터 재시드 후 최대 캐시 TTL(1시간) 내 반영. 정적 페이지는 재빌드 시점에 갱신. |
| D4.5 | 출처 이력 | 배지 변경 로그 테이블(`TCOMPANY_BENEFIT_BADGE_LOG`)은 제거(D1.8). 출처·검증 이력은 행 자체의 `BADGE_SRC_*`/`VERIFIED_DTM` 최신값으로만 관리(이력 테이블 없음). |
| D4.6 | 사용자 신고 부재 | `user_report`가 `BADGE_SRC_CD` 값으로 존재하나(D3.2), MVP에는 **런타임 신고 수집 기능이 없다**(§3 신고 기능 제외). 해당 출처값은 수작업 큐레이션 시점에만 부여. |

## D5. 프로파일러 데이터 미이식 (명시)

| ID | 항목 | 결정 |
|----|------|------|
| D5.1 | 프로파일러 기능 | 가치관 진단 설문 → 페르소나 기능은 **영구 제외**(§2.3, §3). 사용자가 비교 우선순위를 직접 선택한다. |
| D5.2 | 프로파일러 데이터 | 프로파일러 관련 실데이터(질문·시나리오·페르소나·직무적합·직무그룹)는 **이식하지 않는다**(§5). |

| D5.4 | 번들 키 제외 | `/api/v1/reference/all` 응답 최상위에서 `profiles`/`job_groups`/`questions` 키를 제거(§6). 번들 키는 `company_types[]`, `benefit_presets{type_cd:[…]}`, `companies[{…, aliases[], benefits[]}]`만 포함. |

## D6. reference/all 번들 구조 (데이터 계약)

프론트가 부팅 시 로드하는 단일 소스(§6). 아래 최상위 키 외 필드는 추가하지 않는다.

| ID | 키 | 형태 | 내용 |
|----|----|------|------|
| D6.1 | `company_types` | 배열 | `TCOMPANY_TYPE` 6종 |
| D6.2 | `benefit_presets` | 객체 `{type_cd: […]}` | 유형별 `TBENEFIT_PRESET` 목록 |
| D6.3 | `companies` | 배열 `[{…, aliases[], benefits[]}]` | 회사 200개 + 별칭 + 복지(프리셋 폴백 병합, 배지 포함) 인라인 |

**D6.4 — 검색·상세 API 데이터**: `/api/v1/companies/search`는 `{comp_id, comp_nm, comp_tp_cd, industry_nm, logo_nm}` 최대 20건 반환. `/api/v1/companies/{id}`는 `{…, work_style_val, aliases[], benefits[]}`(프리셋 폴백 병합) 반환(§6).
