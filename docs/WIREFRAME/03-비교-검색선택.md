# 비교툴: 검색·회사 선택 화면 (WIREFRAME)

**문서 목적**: 비교 툴의 **진입 셸과 회사 검색·선택 화면군(`/compare`, `page_type=input`)**의 저충실도(lo-fi) 레이아웃을 확정한다. FLOW `03-비교-검색선택.md`가 소유한 화면 ID `SCR-20`~`SCR-24`의 구획(G1~G5·E1~E5·N1~N4·S1~S4·D1~D4)·상태·전이를 **모바일 우선 ASCII 와이어프레임**으로 그리고, 각 구획의 요소·콘텐츠·동작·근거 FR을 표로 명세한다. 픽셀 값·색·타이포는 규정하지 않는다(SPEC 소유). 검색·선택 뷰는 UX 보호를 위해 **무광고**다(브리프 §2-6).

**상위 추적**: WIREFRAME → FLOW → FRD → USECASE → PRD → 브리프. 상위 근거 = FLOW [03-비교-검색선택.md](../FLOW/03-비교-검색선택.md)(SCR-20~24·구획·FLOW-20~22), FRD [03-검색-회사선택.md](../FRD/03-검색-회사선택.md)(FR-10~FR-17). 전역 헤더/푸터·공통 폼/버튼·광고 배치 정본은 WIREFRAME [01-공통-레이아웃.md](01-공통-레이아웃.md)(1장 헤더·2장 푸터·5장 광고 슬롯 §5.1 배치표·8장 버튼·9장 폼)를 **이름으로 인용**하며 재정의하지 않는다. 데이터 계약은 FRD `02-데이터-계약.md`(FR-D3·D4·D6·D7·D8·D11)를 인용한다.

**범위 경계**: 본 문서는 **F1(검색·선택)의 진입 셸·검색·후보·선택·직접 입력 상태**의 저충실도 레이아웃만 소유한다. (1) 슬롯에 회사가 반영된 뒤의 조건 입력(연봉·복지 체크/금액·근무형태·통근·**우선순위**·희생요소)은 F2(WIREFRAME 04-비교입력)가 소유하고, 본 문서는 "다음 · 조건 입력" **진입 트리거**까지만 그린다. (2) 리포트 화면은 F3가 소유한다. (3) 회사 상세 CTA·인기 조합 프리필 링크의 **생성**은 각각 F4·F5가 소유하고, 본 문서는 진입점에서 지시자를 **소비**하는 슬롯 프리필 상태만 그린다(FR-16). (4) 프로파일러(가치관 진단 설문)·익명 서버 측 사용자 쓰기 UI는 제품 범위에서 영구 제외이므로 어떤 구획에도 없다; 로그인·회원·마이페이지·복지편집 UI는 복지 등록·수정 기여(SC14, 문서 09·13·SP-AUTH 소관) 한정 In-scope이나 이 익명 화면엔 등장하지 않는다(FR-01). 검색·선택 전 과정에서 사용자 데이터의 서버 전송·저장은 없고, 상태는 클라이언트 메모리·localStorage·URL 파라미터에만 존재한다(NFR16·NFR17·FR-07). 회사 검색·후보·선택의 대상은 **복지 데이터를 보유한 등록 회사 약 96개**다(200개 아님, 브리프 §5).

## [SCR-20] 비교툴 진입 화면(슬롯 a/b 셸) `/compare`

**목적**: 두 직장을 비교하기 위한 진입점. 현직(a, 비교 기준)·이직처(b, 비교 대상) 두 슬롯을 나란히 제공하고, 각 슬롯에서 회사 검색·선택(SCR-21→23) 또는 직접 입력(SCR-24)을 독립 수행하도록 셸을 구성한다. 부팅 시 참조 번들(`reference/all`, FR-02)이 준비된 상태를 전제로 하며, 두 슬롯이 구성되면 조건 입력(F2)으로 넘긴다. 입력 화면이므로 무광고다(브리프 §2-6).

### 광고 슬롯

- **없음(무광고 강제)**. `page_type=input`은 배치표(WIREFRAME 01 §5.1)에서 **자동광고 OFF · 수동 슬롯 없음 · 제휴 없음**이며, 검색·선택 뷰·셸 계열도 안전 기본값 OFF다. 입력 구역 내 수동 슬롯을 두지 않고 광고 컴포넌트 마운트 자체를 게이팅한다(FR-73·MON8, 브리프 §2-6). 절제된 슬롯은 비교 **결과 화면(F3)**에서만 허용된다.

### 반응형 메모

- 모바일: 슬롯 a→b 세로 스택(1열), 각 카드 풀폭. G5 CTA 풀폭(터치 44px+). 헤더 sticky.
- 데스크톱: 슬롯 a·b **좌우 2열 병렬**, 카드 폭 균등. G4 안내·G5 CTA는 2열 아래 중앙. 콘텐츠 max-width 중앙 정렬.

## [SCR-21] 슬롯 회사 검색·자동완성 후보 패널

**목적**: 특정 슬롯(a/b)에서 회사 정식명·별칭·영문·통칭으로 검색하여, 디바운스(300ms)된 서버 검색 결과를 자동완성 후보 리스트(최대 20건)로 좁혀 표시하고 선택 지점을 제공한다. UC-10(회사명)·UC-11(별칭·영문)·UC-12(자동완성·부분일치)의 검색·표시측을 담당한다.

### 모바일 와이어 (SCR-20 슬롯 A가 검색 상태일 때)


### 표시 데이터

- 후보 항목: `{comp_id, comp_nm, comp_tp_cd(6종), industry_nm|null, logo_nm|null}`(FR-D6). 별칭으로 매칭된 후보도 리스트에는 **정식명**으로 표시(FR-11·UC-11).
- 후보는 `comp_id` 기준 유일(중복 회사 미표기, FR-11 R3). 검색 대상은 등록 회사 약 96개(브리프 §5).
- 후보 텍스트(정식명·산업·유형 라벨)는 `textContent`/이스케이프로 삽입, `innerHTML` 직접 삽입 금지(NFR21). 로고에는 대체 텍스트(NFR15). 검색어·후보는 서버에 쓰기 전송하지 않는다(NFR16·NFR20).

| 0건 / 네트워크·5xx | SCR-22 무결과 / 오류 상태로 전이 |

- **없음(무광고 강제)**. 검색 상호작용 구역에 광고·제휴 슬롯을 배치하지 않는다(입력 화면 광고 게이팅 OFF, WIREFRAME 01 §5.1·브리프 §2-6).

- 모바일: 후보 패널이 검색 입력란 바로 아래 세로 확장, 반대 슬롯은 그 아래 대기. 후보 행 터치 타깃 44px+.
- 데스크톱: SCR-20의 2열 셸 유지 — 검색 중인 슬롯 카드 안에서 후보 패널이 아래로 드롭. 반대 슬롯 열은 독립 유지.

### 추적

## [SCR-22] 무결과·검색 오류 빈 상태

**목적**: 검색 결과가 0건이거나 검색 API가 실패했을 때, 후보 리스트 대신 상태에 맞는 빈 상태를 표시하고 다음 행동을 안내한다. **무결과(정상 0건)**와 **오류(네트워크·5xx)**를 문구·행동으로 구분해, 오류를 무결과로 오인해 직접 입력으로 넘기지 않게 한다. UC-13(검색결과 없음)·UC-91(검색 API 실패)을 담당한다.

## [SCR-23] 슬롯 회사 선택 반영 상태

**목적**: 후보 1건을 선택(또는 CTA/조합 프리필로 사전 지정)하면 그 회사를 지정 슬롯(a/b)에 반영하고, 유형·산업·복지 데이터를 비교 입력(F2)에 연동한 상태를 표현한다. 슬롯 교체·해제·동일 회사 처리의 시작점이다. UC-14(비교 대상 회사 선택)·UC-93(복지 자동 채움)을 담당한다.

- `matched[s]` = 정규화 회사 객체(`{comp_id, comp_nm, comp_tp_cd, industry_nm, logo_nm, work_style_val, aliases[], benefits[], …}`, FR-D8; 정규화 FR-D8.2 — 카테고리 폴백 `perks`·배지 폴백 `est`·불리언 강제).

| 동일 회사 지정 시도 | S4 안내, 반영 보류(상태 미변경) |

### 광고 슬롯

### 반응형 메모

### 추적

## [SCR-24] 직접 입력 모드(회사 미선택)

**목적**: 회사를 선택하지 않고 슬롯을 구성하는 경로. 검색 결과가 없거나(SCR-22) 사용자가 선택을 건너뛴 경우, 회사 연동 없이 조건만 직접 입력해 비교 입력(F2)으로 진입한다. 기업유형만 선택하면 해당 유형 프리셋으로 복지 초기값을 채운다. UC-15(회사 미선택 시 직접 입력 모드)를 담당한다.

- `matched[s]=null` 유지. `benS[s]` = 유형 프리셋 복사값(유형 선택 시) 또는 빈 배열/사용자 직접 추가값. 한 슬롯 회사 선택(SCR-23) + 다른 슬롯 직접 입력의 **혼합 구성** 허용(UC-15 1a). 서버 쓰기·전송 없음, 저장은 localStorage 한정이며 미지원·차단 시에도 비교 정상 동작(NFR16·NFR17·NFR25·FR-07·FR-E3).
