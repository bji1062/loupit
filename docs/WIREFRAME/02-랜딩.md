# 랜딩 화면 (WIREFRAME)

**문서 목적**: 서비스 홈인 **랜딩 화면(`/`, `page_type=landing`)**의 저충실도(lo-fi) 레이아웃을 확정한다. FLOW `02-랜딩.md`가 소유한 화면 ID `SCR-10`~`SCR-13`의 구획(G1~G10)·상태·광고 슬롯을 **모바일 우선 ASCII 와이어프레임**으로 그리고, 각 구획의 요소·콘텐츠·동작·근거 FR을 표로 명세한다. 픽셀 값·색·타이포는 규정하지 않는다(SPEC 소유).

**상위 추적**: WIREFRAME → FLOW → FRD → USECASE → PRD → 브리프. 상위 근거 = FLOW [02-랜딩.md](../FLOW/02-랜딩.md)(SCR-10~13·구획 G1~G10·FLOW-10·FLOW-11), 전역 헤더/푸터/동의 배너·광고 배치 정본은 FLOW [01-사이트맵과-네비.md](../FLOW/01-사이트맵과-네비.md) §4·§5(SCR-01 헤더/푸터, FR-73 배치 표)를 인용하며 재정의하지 않는다. FR 근거 = FRD.md FR 마스터표(FR-02·07·43·51·56·60·71~79·80·E5).

**범위 경계**: 본 문서는 **랜딩 화면 자체의 저충실도 레이아웃**만 소유한다. (1) 전이 목적지 화면(비교 툴 F1~F3 / 회사 상세 F4 / 인기 조합 F5 / 정책 F7)의 내부 레이아웃은 각 담당 WIREFRAME 문서가 소유하고 본 문서는 **진입 링크·프리필 계약**으로만 참조한다. (2) 광고/제휴 컴포넌트의 렌더·게이팅·표기·동의 로직은 F6(FR-7x)이 소유하고 본 문서는 랜딩에서의 **슬롯 위치·예약 높이·"광고" 표기**만 그린다. (3) 프로파일러(가치관 진단 설문)·익명 서버 측 사용자 쓰기 UI는 제품 범위에서 영구 제외이며, 로그인·회원·마이페이지·복지편집 UI는 복지 등록·수정 기여(SC14, 문서 09·13·SP-AUTH 소관) 한정 In-scope이나 이 익명 랜딩 화면엔 등장하지 않는다(FR-01). 랜딩 전 과정에서 사용자 데이터의 서버 전송·저장은 없고, 상태는 브라우저(localStorage/쿠키)·URL 파라미터에만 존재한다(FR-07).

### 광고 슬롯 위치 (`page_type=landing`, FR-73)

- **자동광고 ON** + **수동 슬롯 1개 = `content_bottom`**(G7) + **제휴 선택**(G8), 밀도 **중**.
- 위치: **콘텐츠(G5) 아래, 푸터(G9) 위**. 히어로(G2)·바로가기(G3·G4) **상단에는 수동 슬롯 없음**(비교 시작·탐색을 막지 않음).
- 예약 높이 고정으로 CLS ≤ 0.1(FR-74), 모든 슬롯에 "광고" 라벨(FR-76). 광고는 정적 본문 색인 이후 스크립트로 주입(비-JS 본문 가독 무손상, FR-E5).
