# 정책·고지 페이지·동의 배너 (WIREFRAME)

**문서 목적**: 수익화·규정 준수 대역의 화면군 — **정책·고지 페이지 4종(SCR-60)**, **광고·쿠키 개인화 동의 배너(SCR-61)**, **광고·제휴 컴포넌트(SCR-62)** — 의 저충실도(lo-fi) 레이아웃을 확정한다. FLOW `07-정책-광고-동의.md`가 소유한 SCR-60~62·구획(C0~C4·B1~B5·G1~G7)을 **모바일 우선 ASCII 와이어프레임**으로 1:1 저충실도화한다. SCR-60(정책 문서형 공통 셸)은 데스크톱 와이어도 그린다. **SCR-61 동의 배너와 SCR-62 광고 슬롯·제휴 카드의 시각 정본은 WIREFRAME `01-공통-레이아웃.md`(4·5·6장)가 이미 소유**하므로, 본 문서는 이를 **이름으로 인용**하고 재정의하지 않으며 정책·동의 흐름 맥락에서의 상태·전이·정책 연결만 그린다. 픽셀 값·색·타이포는 규정하지 않는다(SPEC 소유).

**상위 추적**: WIREFRAME → FLOW → FRD → USECASE → PRD → 브리프. 상위 근거 = FLOW [07-정책-광고-동의.md](../FLOW/07-정책-광고-동의.md)(SCR-60~62·구획, FLOW-60·61·62), FRD [10-정책페이지.md](../FRD/10-정책페이지.md)(FR-80~87). 전역 헤더(§01 1장)·전역 푸터(§01 2장)·동의 배너 SCR-61(§01 4장)·광고 슬롯 SCR-62(§01 5장)·제휴 카드 SCR-62(§01 6장)의 시각 정본은 WIREFRAME [01-공통-레이아웃.md](01-공통-레이아웃.md)를 인용하며 재정의하지 않는다. 정책 본문 **필수 항목**(개인정보처리방침 P1~P6·이용약관 T1~T4·면책조항 D-1~D-7·광고/제휴 고지 A-1~A-5)은 FRD 10(FR-81~85)이 소유하고, 본 문서는 그 항목을 화면의 **표시 데이터로 배치**할 뿐 법률 문안·연락처·수치 실값을 창작하지 않는다(플레이스홀더 규약).

**범위 경계**: 본 문서는 정책 페이지 화면·동의 배너·광고/제휴 컴포넌트의 **저충실도 레이아웃·구획 배치·상태·전이**만 소유한다.
1. **정책 문안 실값**(법률 문구·연락처·수치)은 운영자 확정 상수(A5)이며 본 와이어는 플레이스홀더로만 표시한다(FR-85 플레이스홀더 규약, 창작 금지).
2. **동의 배너 SCR-61·광고 슬롯/제휴 카드 SCR-62의 컴포넌트 원형**(CB-1~5·AD-1~3·AF-1~4)은 WIREFRAME 01(4·5·6장)이 소유한다. 본 문서는 이를 인용하고 정책·동의 흐름 맥락의 상태·전이만 추가한다(재정의 금지).
3. **광고 배치(위치·개수) 정본**은 FLOW 01 §5.1·FR-73(§01 5장 배치표)이 소유한다. 본 문서는 그 표를 어기지 않고 인용하며, 정책 페이지의 **광고 없음/최소**만 강제 명시한다.
4. **동의 상태의 저장·개인화/폴백 메커니즘**(localStorage/쿠키 키·개인화 신호)은 FRD 09(FR-78·79)가 소유한다. 본 문서는 배너 UI 상태와 전이로만 표현한다.
5. **로그인/회원/프로파일러/서버 측 사용자 쓰기 UI**는 어떤 구획·전이에도 등장하지 않는다(FR-01). 정책 열람·동의·제휴 클릭·정정요청 전 과정에서 사용자 데이터의 서버 전송·저장이 없고, 동의 상태는 브라우저(localStorage/쿠키)에만 존재한다(NFR16·FR-07).

## [SCR-60] 정책·고지 페이지 (4종 공통 셸 · 전역 접근 · 정정 경로)

**목적**: 개인정보처리방침·이용약관·데이터 정확성 면책조항·광고/제휴 고지 4종을 **각 고유 URL의 정적 페이지**로 게시하는 공통 문서 셸을 그린다. 4종은 동일 셸·`page_type=policy`를 공유하고 **본문 필수 항목만** 다르며, 전역 푸터·동의 배너·면책 링크로 도달 가능하다(FR-80·MON17). 공통 문서 레이아웃 = **제목(단일 `<h1>`) · 목차 · 본문(heading 위계) · 최종 수정일 · 정정요청 연락 경로**. 정책 문안은 FRD 10(FR-81~85)이 소유하고 본 화면은 그 항목을 표시·배치한다.

### 광고 슬롯 (`page_type=policy`)

- **자동광고 OFF/최소 · 수동 슬롯 0 · 제휴 없음**(FLOW 01 §5.1·FR-73 배치표 인용, MON7-6). 정책 페이지 자체에 광고·제휴 슬롯을 **두지 않는다**(▢ 표식으로 명시). 얇은 유틸리티 페이지로 클릭 유도·과도 광고 없이 AdSense 프로그램 정책에 정합(MON18).
- 광고 밀도 "없음"이므로 광고로 인한 레이아웃 이동이 없다(CLS는 폰트 로딩만 억제, NFR5).
- SCR-62(광고·제휴 컴포넌트)는 policy 게이팅으로 **마운트 자체가 되지 않는다**(FR-73·MON8). AdSense client id는 플레이스홀더 유지·산출물 시크릿 미포함(NFR22·MON5).

### 반응형 메모

- 모바일: 전 구획 1열 세로 스택. 목차→본문 앵커, 정정·관련 정책·▢ 표식·푸터 순. 터치 타깃 44px.
- 데스크톱: 목차 사이드바(sticky) + 본문 2단, 콘텐츠 max-width ~960px 중앙. 좁은 폭에서는 목차가 본문 위로 랩(단일 열 회귀).
- 본문 heading 위계·landmark는 모바일·데스크톱 동일(SEO·접근성 불변, NFR13).

## [SCR-61] 광고·쿠키 개인화 동의 배너 (정책·동의 흐름 맥락 · §01 4장 정본 인용)

**목적**: AdSense 쿠키 사용에 대한 첫 방문 고지·개인화 동의/거부를 받는 전역 오버레이 배너를, **정책 페이지로의 단일 진실 연결**과 **FLOW-60 상태 머신** 맥락에서 그린다. **배너 컴포넌트 원형(CB-1~CB-5·데스크톱 하단 바 포함)은 WIREFRAME 01 4장이 소유·정의**하므로 본 절은 이를 인용하고 재정의하지 않는다. 본 문서가 소유하는 정책 링크 대상(`/privacy`·`/ads` = SCR-60)의 도달성과, 상태별(첫 방문·재방문·저장 불가·미선택) 표시/전이만 추가한다. 상태는 **서버 저장 없이 브라우저(localStorage/쿠키)에만** 보관되고 사이트 이용을 막지 않는다(FR-78·79·NFR16).

### 광고 슬롯 · 반응형 메모

- **광고 슬롯**: 해당 없음. 배너 자체는 광고가 아니라 개인화를 게이팅하는 동의 UI이며 "광고" 라벨을 붙이지 않는다(§01 4장). 배너 결정에 따른 광고 종류는 SCR-62에서 적용되고, 폴백 상황에서도 슬롯 레이아웃은 예약 높이로 무손상(FR-74·79·MON6).
- **반응형**: 모바일=하단 시트(문안 여러 줄·버튼 2개 나란히), 데스크톱=하단 바 1행(문안 좌·버튼 우) — 원형은 §01 4장 인용. 본문 스크롤을 막지 않는다.
- **범위 비고**: 세분화된 개인화 설정(항목별 토글·해외 CMP) UI는 브리프 미확정으로 **정의하지 않는다**(2차 재검토, 창작 금지). 본 화면은 동의/거부 이분 선택만 그린다.

## [SCR-62] 광고·제휴 컴포넌트 ("광고" 표기 · 제휴 링크 클릭 · §01 5·6장 정본 인용)

**목적**: 콘텐츠 페이지 위에 마운트되는 광고·제휴 컴포넌트를, 본 대역이 소유하는 **"광고" 표기·제휴 링크 클릭 전이(FLOW-62)**와 **정책·입력 무광고 강제** 맥락에서 그린다. **슬롯 컨테이너·제휴 카드 원형(AD-1~3·AF-1~4)과 배치표는 WIREFRAME 01 5·6장이 소유**하므로 본 절은 이를 인용하고 재정의하지 않으며, "광고" 라벨 부착·제휴 링크 속성/외부 이동·게이팅 결과(policy/input 미마운트)만 추가한다.

### 맥락 와이어 — 모바일 (제휴 카드 + 클릭 전이 · §01 6장 AF-* 인용, 재정의 아님)

```
┌─ 📱 SCR-62 광고·제휴 컴포넌트 — "광고" 표기·제휴 클릭 (§01 5·6장 인용) ─┐
│  (호스트 콘텐츠 페이지 위 마운트 · policy·input = 미마운트/무광고 강제)   │
│                                                                      │
│  [광고]                                    ← "광고" 표기(§01 AD-1/AF-1, 은폐 금지)
│  ┌──────────────────────────────────────┐                            │
│  │ 이력서 첨삭 서비스                     │  ← label (affiliate.json · HTML 이스케이프)
│  │ 이직 준비 관련 서비스                  │     desc (선택 · 이스케이프)
│  │                       [ 바로가기 ↗ ]  │  ← 제휴 링크 클릭 → 외부(새 탭)
│  └──────────────────────────────────────┘     rel="sponsored nofollow noopener"
│                                                target="_blank"       │
└──────────────────────────────────────────────────────────────────────┘
  클릭 전이: 외부 제휴 대상 이동 · 서버 사용자 저장 0(NFR16)
  게이팅: url이 http/https 아니면 미렌더 · 활성 항목 없으면 컴포넌트 미노출
  policy·input: 컴포넌트 마운트 0(무광고 강제) — 정책 페이지엔 광고·제휴 없음
```

### 구획별 설명 (요소 원형 = §01 5·6장, 본 절은 표기·클릭 전이만 추가)

| G5 | "광고" 표기(§01 AD-1·AF-1) | "광고"(또는 "유료 광고") 라벨 | 모든 슬롯·제휴 영역 상시 표기, 인지 가능 색·크기, **은폐 금지** | FR-76·NFR19·MON11 |

### 광고 배치 (FLOW 01 §5.1·FR-73 정본 인용 · 재정의 아님)

| 호스트 화면 | `page_type` | 자동광고 | 수동 슬롯 | 제휴 |
| --- | --- | :---: | --- | :---: |
| SCR-01 랜딩 | `landing` | ON | 콘텐츠 하단 1 | 선택 |
| SCR-06 회사 상세 | `company` | ON | 본문 중간 1 + 하단 1 | O |
| SCR-07 인기 조합 | `combo` | ON | 본문 중간 1 + 하단 1 | O |
| SCR-04 비교 입력 | `input` | **OFF** | **없음** | **없음** |
| SCR-05 비교 리포트 | `result` | OFF | 리포트 하단 1(절제) | 선택 |
| **SCR-60 정책** | **`policy`** | **OFF/최소** | **없음** | **없음** |

- 본 표는 §01 5장 배치표의 인용이며 위치·개수 추가·변경을 금지한다. **정책 페이지(policy)·비교 입력(input)은 무광고 강제**로 컴포넌트가 마운트되지 않는다(FR-73·MON8).

### 반응형 메모 · 추적

- **반응형**: 모바일=세로 카드(라벨·설명·CTA 스택), 데스크톱=카드 폭 확대·다중 항목 2열. 슬롯은 예약 높이 반응형(§01 5·6장 인용).
- **SCR**: SCR-62(호스트 landing/company/combo/result 위 마운트, 원형=§01 5·6장). policy·input=미마운트.
- **FR**: FR-73(게이팅)·FR-74(CLS)·FR-75(`affiliate.json`)·FR-76("광고" 표기)·FR-77(링크 속성)·FR-86(정책 무광고).
- **UC**: UC-60(슬롯 노출·표기)·UC-61(제휴 클릭).
