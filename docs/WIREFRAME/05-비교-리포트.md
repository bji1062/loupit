# 비교 리포트 화면 (WIREFRAME)

**문서 목적**: 비교 툴 SPA(`/compare`) 안의 **비교 리포트(결과) 뷰(`page_type=result`)**의 저충실도(lo-fi) 레이아웃을 확정한다. FLOW `05-비교-리포트.md`가 소유한 화면 ID `SCR-40`~`SCR-43`의 구획(R0~R10·U1~U5·L1~L7·H1~H4)·상태·전이를 **모바일 우선 ASCII 와이어프레임**으로 그리고, 각 구획의 요소·콘텐츠·동작·근거 FR을 표로 명세한다. 판정 카드(vdCard)·총보상·시간조정 가치·워라밸/자율성·카테고리별 복지 델타·정성 복지·배지/불확실성 밴드로 구성되는 결과 화면과, 재계산·최근 비교 로컬 저장·공유·입력 수정 진입점을 다룬다. 지표 **값은 F3 계산 FR(FR-30~39, FRD 05)의 출력을 표시만** 하며 계산식을 재정의하지 않고, 표현·저장 규칙은 FRD 06(FR-40~45)을 인용한다. 픽셀 값·색·타이포는 규정하지 않는다(SPEC 소유). **결과 뷰는 리포트 하단 절제 광고 슬롯 정확히 1개**만 둔다(판정 상단·중간 삽입 금지, `page_type=result`, MON9).

**상위 추적**: WIREFRAME → FLOW → FRD → USECASE → PRD → 브리프. 상위 근거 = FLOW [05-비교-리포트.md](../FLOW/05-비교-리포트.md)(SCR-40~43·구획 R0~R10/U1~U5/L1~L7/H1~H4·FLOW-40·41·42·43), FRD [05-비교엔진.md](../FRD/05-비교엔진.md)(FR-30~39 계산 출력, 표시 대상), FRD [06-리포트-표현.md](../FRD/06-리포트-표현.md)(FR-40 구성·렌더·FR-41 배지·밴드 표시·FR-42 재계산·FR-43 로컬 저장·FR-44 폴백·FR-45 이스케이프). 전역 헤더/푸터·버튼·**배지(§7 BG-1~6)**·광고 슬롯의 정본은 WIREFRAME [01-공통-레이아웃.md](01-공통-레이아웃.md)(1장 헤더 HDR·2장 푸터 FTR·5장 광고 슬롯·6장 제휴·**7장 배지 BG**·8장 버튼 BTN)를 **이름으로 인용**하며 재정의하지 않는다. 형식·상세도 정합 = WIREFRAME [02-랜딩.md](02-랜딩.md)·[04-비교입력.md](04-비교입력.md). 브리프 §7(`compare`/`calc`·vdCard·밴드)·§4(만원 단위·`amt_source`·만료)·§2-2(localStorage)·§2-6(결과 화면 절제 광고)·§2-8(클라이언트 계산).

**범위 경계**: 본 문서는 **비교 리포트(결과) 뷰의 저충실도 레이아웃**만 소유한다. (1) 실효연봉·시간가치·워라밸 요인·카테고리 합산·vdCard 판정·밴드 계수의 **계산 로직**은 F3 계산 FR(FR-3x, FRD 05)이 소유하고 본 문서는 그 출력을 **표시 배치**로만 그린다. (2) 연봉·복지·근무형태·통근·우선순위·희생요소의 **입력 수집·검증·프리셋 폴백 UI**는 F2(WIREFRAME [04-비교입력.md](04-비교입력.md), SCR-30~34)가 소유하며, 본 문서는 입력 **변경→재계산 트리거**와 입력 화면 **복귀 진입점**만 그린다. (3) 하단 광고 슬롯의 **콘텐츠·게이팅**은 F6(01 §5·6장)이 소유하고 본 문서는 위치·개수·"광고" 표기·예약 높이만 그린다. (4) 회사 검색·선택은 F1(SCR-03)이 소유하며 "새 비교"는 그 화면 복귀 링크로만 다룬다. 프로파일러(가치관 진단 설문)·익명 서버 측 사용자 쓰기 UI는 제품 범위에서 영구 제외이므로 어떤 구획에도 없다; 로그인·회원·마이페이지·복지편집 UI는 복지 등록·수정 기여(SC14, 문서 09·13·SP-AUTH 소관) 한정 In-scope이나 이 익명 화면엔 등장하지 않는다(FR-01). 우선순위는 프로파일러 산출이 아니라 사용자 직접 선택이다(§2-3). 계산 입력·결과는 서버로 전송·저장되지 않으며(NFR16), 지속 저장은 브라우저 localStorage에만 발생한다(FR-07·FR-43). 회사 등록 범위는 복지 데이터 보유 ~96개다(브리프 §5).

## 화면 인덱스

| SCR-ID | 화면명 | `page_type` | 성격 | 이 문서에서 그리는 것 |
| --- | --- | --- | --- | --- |
| SCR-40 | 비교 리포트 결과 화면(판정·총보상·시간가치·워라밸·복지 델타·정성·배지/밴드) | `result` | SPA 뷰(SCR-05 상세) · 절제 슬롯 1 | 모바일 + 데스크톱 전체 레이아웃(R0~R10) |
| SCR-41 | 리포트 재계산·부분 미산출 상태 | `result` | SCR-40 위 갱신/부분 상태 | SCR-40 위 재계산·미산출 상태(U1~U5) |
| SCR-42 | 최근 비교 저장·목록 패널(로컬) + 저장 불가 폴백 | `result` | SCR-40 위 로컬 저장 패널/상태 | SCR-40 위 저장/목록/삭제/폴백(L1~L7) |
| SCR-43 | 리포트 공유(회사쌍 프리필 링크) | `result` | SCR-40 위 공유 액션/상태 | SCR-40 위 공유 링크 상태(H1~H4) |

> SCR-41·42·43은 SCR-40 정적 리포트 **위에 겹치는 상태/패널**이며 별도 라우트가 아니다. 판정 카드(vdCard)·총보상·시간가치·워라밸·카테고리 델타·정성·배지/밴드는 별도 화면이 아니라 **SCR-40의 구획(R0~R10)**이다. 네 상태 모두 `page_type=result`로 동일 광고 게이팅(**리포트 하단 절제 슬롯 1**)을 공유하고, 리렌더·패널 표시로 인한 레이아웃 이동을 막기 위해 광고·동적 영역은 고정 크기를 예약한다(CLS ≤ 0.1, FR-74·NFR5).

## [SCR-40] 비교 리포트 결과 화면 `/compare` (`page_type=result`)

**목적**: F2 입력을 받아 클라이언트 `compare()`가 산출한 a(현직)·b(이직처) 두 관점의 비교 리포트를 한 화면에 구성·표시한다. 판정 카드(R1)·총보상(R2)·시간조정 가치(R3)·워라밸/자율성(R4)·카테고리 델타 표(R5)·정성 복지(R6)를 두 관점 대비로 렌더하고, 최근 비교 저장(R7)·공유(R8)·입력 수정/새 비교(R9) 진입점과 리포트 하단 절제 광고 슬롯 1개(R10)를 둔다. 지표 값은 F3 계산 FR(FR-30~39)의 출력을 **표시**만 하며 재정의하지 않는다(표현·계산 분리, FR-40).

### 광고 슬롯

**리포트 본문 하단 수동 슬롯 정확히 1개**(R10, 절제). `page_type=result`는 **자동광고 OFF**이며 판정 카드(R1) 상단·중간 삽입은 금지한다(콘텐츠 위주 정책, §2-6·MON9). 슬롯 콘텐츠·게이팅은 F6(01 §5장) 소유이고 본 화면은 위치·개수·"광고" 표기(FR-76·NFR19)·고정 예약높이(FR-74, CLS ≤ 0.1)만 규정한다. 광고는 리포트 렌더 이후 클라이언트 스크립트로 주입되어 판정·조회 흐름을 방해하지 않는다. 제휴 컴포넌트는 선택(저밀도, 01 §6장).

### 반응형 메모

- 모바일: 전 구획 1열 세로 스택(R0→R1→…→R6→R7~R9→R10). R2~R4 카드는 카드 내부에서 a·b를 상하 또는 좌우 축약 대비, R5·R6 표는 가로 스크롤 없이 3열(카테고리·a·b·델타) 유지. 헤더 sticky.
- 데스크톱: 콘텐츠 max-width ~960px 중앙 정렬. R2·R3, R4, R5·R6은 2열 배치로 밀도 확대, R1·R0·R7~R9·R10은 full-width. 배지는 금액 옆 인라인 정렬(01 §7 반응형 메모).

## [SCR-42] 최근 비교 저장·목록 패널(로컬) + 저장 불가 폴백

**목적**: 조회한 비교를 "최근 비교"로 **브라우저 localStorage에만** 저장하고, 재방문 시 목록에서 불러와 리포트를 복원하며, 항목별·전체 삭제를 제공하는 패널이다. 서버 계정·서버 저장은 존재하지 않고 저장·불러오기·삭제 전 과정이 로그인 없이 기기 내에서만 동작한다. localStorage가 비활성·차단·용량 초과이면 저장·복원만 생략하고 비교·조회·재계산은 정상 동작하는 폴백 상태를 함께 표현한다(FR-43·44, UC-36).

## [SCR-43] 리포트 공유(회사쌍 프리필 링크)

**목적**: 조회 중인 비교를 다른 사람과 나누도록, **두 회사(a/b)를 프리필하는 비교 툴 링크**를 생성·제공한다. 공유 링크는 인기 조합 진입과 동일한 프리필 URL 계약(`/compare?a={식별자}&b={식별자}`, FR-62)을 재사용하며, 수신자는 그 링크로 비교 툴에 진입해 양사가 채워진 상태에서 **본인 조건을 직접 입력**한다(UC-52). 공유는 공개 참조 데이터인 **회사 식별자만** 담고 사용자의 연봉·복지 등 개인 입력값을 서버로 전송·저장하지 않는다(§2-8, NFR16). 정확한 공유 UI 기법(링크 복사·네이티브 공유)은 SPEC 확정 사항이며, 본 문서는 **회사쌍 프리필 링크**를 기본 계약으로 채택한다(창작 금지).
