# 비교 입력 화면 (WIREFRAME)

**문서 목적**: 비교 툴 SPA(`/compare`) 안의 **비교 입력 뷰(`page_type=input`)**의 저충실도(lo-fi) 레이아웃을 확정한다. FLOW `04-비교-입력.md`가 소유한 화면 ID `SCR-30`~`SCR-34`의 구획(C1~C8)·상태·전이를 **모바일 우선 ASCII 와이어프레임**으로 그리고, 각 구획의 요소·콘텐츠·동작·근거 FR을 표로 명세한다. 두 직장(현직 a / 이직처 b)을 병렬 입력하고 실시간 요약(`calc`)을 확인하다가 **비교 실행 또는 뒤로**로 갈라지는 F2 입력 뷰의 화면 뼈대만 다룬다. 픽셀 값·색·타이포는 규정하지 않는다(SPEC 소유). **이 뷰는 무광고 강제다**(`page_type=input`, 광고 컴포넌트 마운트 자체 게이팅, FR-73·MON8).

**상위 추적**: WIREFRAME → FLOW → FRD → USECASE → PRD → 브리프. 상위 근거 = FLOW [04-비교-입력.md](../FLOW/04-비교-입력.md)(SCR-30~34·구획 C1~C8·FLOW-30·31·32), FRD [04-비교입력.md](../FRD/04-비교입력.md)(FR-20·21·22·23·24·25, F2 입력 상태 변수 요약·의사코드). 전역 헤더/푸터·버튼·폼 컨트롤·배지·광고 게이팅의 정본은 WIREFRAME [01-공통-레이아웃.md](01-공통-레이아웃.md)(1장 헤더 HDR·2장 푸터 FTR·5장 광고 슬롯·6장 제휴·7장 배지 BG·8장 버튼 BTN·**9장 공통 폼 요소 FM-1~6**)를 **이름으로 인용**하며 재정의하지 않는다. 브리프 §7(`calc`/`compare`·`getSalRange`/`getOfferRange`·`getBenTotal`·`getWSHours`·`getOTPay`)·§4(만원 단위·데이터 모델)·§2-3(프로파일러 제외·직접 선택)·§2-6(입력 화면 광고 최소화).

**범위 경계**: 본 문서는 **비교 입력 뷰의 저충실도 레이아웃**만 소유한다. (1) 회사를 찾아 슬롯에 지정하는 검색·선택 화면(SCR-03)의 내부 레이아웃은 F1 WIREFRAME이 소유하고 본 문서는 **슬롯 교체·해제·직접 입력 진입 링크**로만 참조한다. (2) `calc` 실시간 요약과 `compare` 확정 리포트의 **계산식·리포트 표현**은 F3(SCR-05)가 소유하고 본 문서는 입력 뷰에 표시되는 **요약 패널(C7)·비교 실행 버튼(C8)**만 그린다(브리프 §7: 값 지정까지만 F2 담당). (3) 공통 폼 컨트롤·버튼·배지·광고 게이팅의 원형·정책은 WIREFRAME 01이 소유하고 본 문서는 인용만 한다. 프로파일러(가치관 진단 설문)·익명 서버 측 사용자 쓰기 UI는 제품 범위에서 영구 제외이므로 어떤 구획에도 없다; 로그인·회원·마이페이지·복지편집 UI는 복지 등록·수정 기여(SC14, 문서 09·13·SP-AUTH 소관) 한정 In-scope이나 이 익명 화면엔 등장하지 않는다(FR-01). 우선순위는 프로파일러 산출이 아니라 사용자 직접 선택이다(§2-3, FR-25). 입력값·최근 비교는 클라이언트 메모리·localStorage에만 존재하고 서버로 전송·저장되지 않는다(NFR16, FR-07·FR-E3).

| SCR-ID | 화면명 | `page_type` | 성격 | 이 문서에서 그리는 것 |
| --- | --- | --- | --- | --- |
| SCR-30 | 비교 입력 화면(a/b 병렬 5구획) | `input` | SPA 뷰 · 무광고 | 모바일 + 데스크톱 전체 레이아웃(C1~C8) |
| SCR-31 | 복지 프리셋 자동채움 상태(회사 지정 슬롯) | `input` | 슬롯 초기화 상태 | SCR-30 C3·C4 위 자동채움 상태 |
| SCR-32 | 직접 입력 모드 상태(회사 미지정 슬롯) | `input` | 슬롯 초기화 상태 | SCR-30 C1·C3 위 직접 입력 상태 |
| SCR-33 | 실시간 요약(`calc`) 패널 상태 | `input` | 향상(JS) 표시 상태 | SCR-30 C7 요약 패널 상세 |
| SCR-34 | 입력 검증·정정 안내 상태 | `input` | 오류·검증 상태 | SCR-30 입력 필드 위 검증 상태 |

> SCR-31·32·33·34는 SCR-30 위에 겹치는 **상태**이며 별도 라우트가 아니다. 슬롯 a·b는 각각 독립 초기화되므로 한 화면에서 슬롯 a=SCR-31(회사 지정)·슬롯 b=SCR-32(직접 입력)가 동시에 성립할 수 있다. 다섯 상태 모두 `page_type=input`으로 광고 컴포넌트를 마운트하지 않는다(무광고 강제, FR-73·MON8).

## [SCR-30] 비교 입력 화면 (a/b 병렬 5구획) `/compare` (`page_type=input`)

**목적**: 두 직장(현직 a / 이직처 b)의 연봉·복지·근무형태·통근·우선순위를 사용자가 **직접 입력**하고, 입력 즉시 대략 비교(`calc`)를 확인하며, 만족하면 **비교 실행**으로 확정 리포트(SCR-05)에 진입하는 F2 입력 뷰. 우선순위는 프로파일러 없이 사용자가 직접 고른다(§2-3). 입력 몰입을 보호하기 위해 광고를 마운트하지 않는다(§2-6·MON8).

## [SCR-33] 실시간 요약(`calc`) 패널 상태

**목적**: 입력을 바꿀 때마다 확정 리포트를 열지 않고도 **대략적인 비교 방향**을 즉시 확인하게 한다. 계산(`calc`)은 F3 소유이며(FR-30), 본 뷰는 입력 옆/아래 그 결과를 표시하는 **패널**(SCR-30 C7)만 규정한다. 확정 리포트(`compare`)는 비교 실행 시 SCR-05가 생성한다(브리프 §7: `calc`=실시간 요약, `compare`=확정 리포트).

### 모바일 와이어

