# 공통 레이아웃·네비·컴포넌트 (WIREFRAME)

**목적**: 모든 화면(SCR-01~09)이 재사용하는 **전역 레이아웃 골격·네비게이션·공통 컴포넌트**의 저충실도(lo-fi) 와이어프레임을 확정한다. 개별 화면 문서(랜딩·비교·회사상세·조합·정책)는 여기서 정의한 헤더·푸터·동의 배너·광고 슬롯·제휴 카드·배지·버튼·폼 블록을 **이름으로 인용**하고 재정의하지 않는다. 이 문서는 "공통 블록의 시각적 뼈대와 상태·반응형·광고 정책"만 소유한다.

**상위 추적**: WIREFRAME → FLOW → FRD → USECASE → PRD → 브리프. 상위 근거 = FLOW `01-사이트맵과-네비.md`(§4 전역 헤더/푸터/동의 배너, §5 광고·정책 전역 배치, SCR-01~09), FRD `09-광고-제휴.md`(FR-70~79: 광고 슬롯·제휴·"광고" 표기·CLS 예약·동의). 브리프 §2-5(수익화·"광고" 표기)·§2-6(광고 배치 정책)·§8(기술 스택: 바닐라 JS·Pretendard).

**범위 경계**: 전역 공통 블록만 그린다. 각 화면 고유 구획(검색 자동완성, 입력 5구획, 리포트 판정 카드, 회사 복지표 본문 등)은 해당 화면 WIREFRAME이 소유한다. 프로파일러·익명 서버 측 사용자 쓰기 UI는 어떤 공통 블록에도 등장하지 않는다; 로그인·회원·마이페이지·복지편집 UI는 복지 기여 SC14 한정(SP-AUTH)이라 이 익명 공통 블록엔 비등장(FR-01).

**다루는 화면·컴포넌트 ID**: 전역 골격 앵커 = **SCR-01**(랜딩, page_type=landing). 전역 컴포넌트 = **SCR-61**(광고·쿠키 동의 배너, 랜딩 인스턴스 SCR-13), **SCR-62**(광고·제휴 컴포넌트). 헤더·푸터는 전 화면(SCR-01~09) 공통 landmark.

## 0. 반응형·디자인 토큰 (전역 공통 규약)

모든 공통 블록에 적용되는 브레이크포인트·토큰. 개별 와이어는 이 규약을 전제한다.

| 항목 | 값·규칙 | 근거 |
| --- | --- | --- |
| 브레이크포인트 | 모바일 `< 768px`(1열 세로), 데스크톱 `≥ 768px`(가로·다열). **모바일 우선** | 브리프 §8 |
| 최대 콘텐츠 폭 | 본문 컨테이너 `max-width ≈ 960px`, 좌우 여백 16px(모바일)·24px(데스크톱) | 관례 |
| 폰트 | Pretendard, 본문 16px/1.6, 라벨 13px, "광고" 라벨 11~12px | 브리프 §8 |
| 광고 슬롯 예약 높이 | 삽입 전 `min-height` 확보로 CLS ≤ 0.1 (모바일 slot 100~280px, 반응형) | FR-74·NFR5 |
| 링크 | 모든 네비·정책 링크는 순수 `<a href>`(JS 없이 이동, 비-JS 색인) | FLOW §4·FR-E5 |
| 접근성 | landmark(`<header>`/`<nav>`/`<main>`/`<footer>`), 포커스 링, 라벨-입력 연결 | NFR |

**전역 페이지 골격(모바일)** — 모든 화면이 공유하는 세로 스택 순서:

```
┌──────────────────────────────┐
│  ▔▔  전역 헤더 (sticky)  ▔▔  │  ← 3장
├──────────────────────────────┤
│                              │
│         <main>               │
│   (화면별 고유 본문 구획)      │
│                              │
│   … 광고 슬롯은 page_type    │
│      게이팅으로만 삽입 …      │  ← 6·7장
│                              │
├──────────────────────────────┤
│  ▁▁  전역 푸터 (정책 4종) ▁▁ │  ← 4장
└──────────────────────────────┘
   ┌────────────────────────┐
   │ 동의 배너(첫 방문 오버레이)│    ← 5장 (하단 고정, main 위 겹침)
   └────────────────────────┘
```

## 1. 전역 헤더 (`<header>`/`<nav>`) — 전 화면 공통

**목적**: 로고(홈 복귀)와 비교 툴 진입("비교하기") 두 항목만 두는 최소 헤더. "비교하기"가 곧 회사 검색·비교의 진입점(검색 진입)이다.

**와이어 — 모바일**

```
┌──────────────────────────────┐
│ loupit            [ 비교하기 ]│
└──────────────────────────────┘
  ▲로고(→/)         ▲→/compare
```

**와이어 — 데스크톱**

```
┌───────────────────────────────────────────────────────────┐
│  loupit                                    [  비교하기  ]   │
└───────────────────────────────────────────────────────────┘
   ▲로고(→ /  홈 복귀)                        ▲ → /compare (SCR-02)
```

**구획별 설명**

| 구획ID | 요소 | 콘텐츠 | 동작 | 관련 FR |
| --- | --- | --- | --- | --- |
| HDR-1 | 로고 워드마크 | "loupit" 텍스트 | 클릭 → `/`(SCR-01) 홈 복귀 | 관례(FLOW §4.1) |
| HDR-2 | 비교하기 버튼(네비) | "비교하기" | 클릭 → `/compare`(SCR-02→검색 뷰 SCR-03). **검색 진입점** | FR-02(F1~F3 진입) |

- **검색 진입 정책**: 헤더에 상시 노출되는 전역 검색창은 두지 않는다. 검색은 `/compare` 진입 후 검색·선택 뷰(SCR-03)에서 수행한다(FLOW §4.1 최소 헤더). 회사 랭킹/리스트 메뉴는 MVP 미포함으로 넣지 않는다(창작 금지).
- 헤더는 sticky(스크롤 시 상단 고정) 권장하되, 광고·본문 색인에 영향 없음.

**상태 변형**

| 상태 | 표현 |
| --- | --- |
| 기본 | 로고 + "비교하기" 노출 |
| 현재가 `/compare` | "비교하기"에 active 표시(밑줄/강조) 선택적 |
| 비-JS | 두 항목 모두 순수 `<a>`로 정상 이동(FR-E5) |

**광고 슬롯**: 없음(헤더는 광고 영역 아님).

**반응형 메모**: 항목이 2개뿐이라 햄버거 메뉴 불필요. 모바일=로고 좌·버튼 우 1행, 데스크톱=동일 배치에 여백 확대.

**추적**: SCR-01~09 공통 / FLOW §4.1 / FR-02 / UC-63(전역 네비).

## 2. 전역 푸터 (`<footer>`) — 전 화면 공통

**목적**: AdSense 승인·규정 준수의 전제인 **정책 4종 전역 접근**과 데이터 정확성 면책 요약, 저작권을 제공한다. 4종 링크는 누락·404가 0이어야 한다.

**와이어 — 모바일 (세로 스택)**

```
┌──────────────────────────────┐
│ 데이터는 참고용이며 정확성을  │
│ 보장하지 않습니다. 자세히 →   │ ─→ /disclaimer (SCR-08)
│                              │
│ › 개인정보처리방침            │ ─→ /privacy
│ › 이용약관                    │ ─→ /terms
│ › 데이터 정확성 면책조항       │ ─→ /disclaimer
│ › 광고·제휴 고지              │ ─→ /ads
│                              │
│ © 2026 loupit                │
└──────────────────────────────┘
```

**와이어 — 데스크톱 (가로 정렬)**

```
┌───────────────────────────────────────────────────────────┐
│ 개인정보처리방침 · 이용약관 · 데이터 정확성 면책조항 · 광고·제휴 고지 │
│                                                           │
│ 데이터는 참고용이며 정확성을 보장하지 않습니다(면책조항).      │
│ © 2026 loupit                                             │
└───────────────────────────────────────────────────────────┘
```

**구획별 설명**

| 구획ID | 요소 | 콘텐츠 | 동작 | 관련 FR |
| --- | --- | --- | --- | --- |
| FTR-1 | 면책 요약 문구 | "데이터는 참고용…정확성 미보장" | 클릭 → `/disclaimer`(SCR-08) | FR-54·FR-83 |
| FTR-2 | 개인정보처리방침 링크 | 텍스트 링크 | → `/privacy`(SCR-08) | FR-80·FR-81 |
| FTR-3 | 이용약관 링크 | 텍스트 링크 | → `/terms`(SCR-08) | FR-80·FR-82 |
| FTR-4 | 면책조항 링크 | 텍스트 링크 | → `/disclaimer`(SCR-08) | FR-80·FR-83 |
| FTR-5 | 광고·제휴 고지 링크 | 텍스트 링크 | → `/ads`(SCR-08) | FR-80·FR-84 |
| FTR-6 | 저작권 | "© 2026 loupit" | 정적 | 관례 |

- 4종 링크는 **단일 진실**인 정책 페이지(SCR-08)로만 연결한다(중복 정의 금지). 동의 배너·회사 상세 면책 블록도 같은 페이지를 참조한다.
- 링크 404 = 생성물 검증 실패 항목(FR-80·FR-87).

**상태 변형**

| 상태 | 표현 |
| --- | --- |
| 기본 | 4종 링크 + 면책 요약 + 저작권 |
| 비-JS | 전부 순수 `<a>`로 이동 가능(FR-E5·NFR24) |

**광고 슬롯**: 없음(푸터는 광고 영역 아님).

**반응형 메모**: 모바일=세로 리스트(터치 타깃 44px 확보), 데스크톱=구분점(·) 가로 1행. 저작권·면책은 항상 하단.

**추적**: SCR-01~09 공통 / FLOW §4.2 / FR-80~84·87 / UC-63.

## 3. 전역 페이지 골격 (앵커: SCR-01 랜딩)

**목적**: 헤더·본문·푸터·동의 배너·광고 슬롯이 하나의 콘텐츠 페이지에서 어떻게 조립되는지 대표 예로 랜딩(SCR-01, page_type=landing)에 매핑. 다른 콘텐츠 페이지(회사상세·조합)도 같은 골격에 본문·광고 밀도만 다르다.

**와이어 — 모바일 (SCR-01 예시)**

```
┌──────────────────────────────┐
│ loupit            [ 비교하기 ]│ HDR (1장)
├──────────────────────────────┤
│ 로그인 없이 한국 회사 복지·   │
│ 연봉·근무조건을 비교하세요.   │  서비스 한 줄 소개
│      ┌────────────────────┐  │
│      │   비교 시작  →      │  │  Primary CTA → /compare
│      └────────────────────┘  │
│                              │
│ 대표 회사                     │
│ › 삼성전자  › SK하이닉스 …    │  내부 링크(→SCR-06)
│ 인기 조합                     │
│ › 삼성전자 vs SK하이닉스 …    │  내부 링크(→SCR-07)
│                              │
│ ┌──────────────────────────┐ │
│ │ 광고                      │ │  콘텐츠 하단 수동 슬롯 1
│ │   (AdSense) 예약높이 250px │ │  page_type=landing (6장)
│ └──────────────────────────┘ │
├──────────────────────────────┤
│ 정책 4종 · 면책 · © loupit    │ FTR (2장)
└──────────────────────────────┘
 ┌────────────────────────────┐
 │ 쿠키·광고 안내  [거부][동의] │  동의 배너(첫 방문, 5장)
 └────────────────────────────┘
```

**구획별 설명**(랜딩 고유 본문은 별도 랜딩 WIREFRAME 소유; 여기선 공통 배치만)

| 구획ID | 요소 | 콘텐츠 | 동작 | 관련 FR |
| --- | --- | --- | --- | --- |
| PG-1 | 전역 헤더 | HDR-1/2 | 1장 | FR-02 |
| PG-2 | 본문 `<main>` | 화면별 고유(여기선 소개+CTA+내부링크) | 화면 소유 | FR-71~73(진입) |
| PG-3 | 콘텐츠 하단 광고 | 수동 슬롯 1(landing) | 6장 컴포넌트 | FR-72·73 |
| PG-4 | 전역 푸터 | FTR-1~6 | 2장 | FR-80~84 |
| PG-5 | 동의 배너 | 첫 방문 오버레이 | 5장 컴포넌트 | FR-78·79 |

**광고 슬롯 위치**: landing = **자동광고 ON + 콘텐츠 하단 수동 1**(제휴 선택). 슬롯은 예약높이로 CLS 억제, "광고" 라벨 부착(§5.1 배치표·FR-73).

**반응형 메모**: 모바일 1열, 데스크톱은 본문 max-width 960px 가운데 정렬. 광고 슬롯은 본문 폭에 반응형.

**추적**: SCR-01 / FLOW §6 SCR-01·§5.1 / FR-71·72·73 / UC 진입 허브.

## 4. 광고·쿠키 동의 배너 (SCR-61 / 랜딩 인스턴스 SCR-13)

**목적**: 첫 방문 시 광고·쿠키 사용 목적과 제3자(Google) 고지, 정책 링크, 동의/거부 선택을 제공. 상태는 **브라우저(localStorage/쿠키)에만** 저장(서버 전송 없음). 이 배너 자체는 광고가 아니므로 "광고" 라벨을 붙이지 않는다.

**와이어 — 모바일 (하단 고정 시트)**

```
┌──────────────────────────────┐
│ 쿠키·광고 안내                │
│ loupit은 광고 게재를 위해     │
│ 쿠키를 사용하며 제3자(Google) │
│ 맞춤형 광고가 표시될 수 있습니다.│
│                              │
│ 개인정보처리방침 · 광고·제휴 고지│ ← 링크
│                              │
│ ┌──────────┐  ┌───────────┐  │
│ │   거부    │  │   동의     │  │
│ └──────────┘  └───────────┘  │
└──────────────────────────────┘
```

**와이어 — 데스크톱 (하단 바)**

```
┌───────────────────────────────────────────────────────────┐
│ 광고 게재를 위해 쿠키를 사용하며 제3자(Google) 맞춤형 광고가  │
│ 표시될 수 있습니다.  개인정보처리방침 · 광고·제휴 고지         │
│                                     [ 거부 ]   [  동의  ]    │
└───────────────────────────────────────────────────────────┘
```

**구획별 설명**

| 구획ID | 요소 | 콘텐츠 | 동작 | 관련 FR |
| --- | --- | --- | --- | --- |
| CB-1 | 고지 문안 | 쿠키·광고 목적·제3자(Google) 사실 | 정적 텍스트 | FR-78·MON13 |
| CB-2 | 개인정보처리방침 링크 | 텍스트 | → `/privacy`(SCR-08) | FR-78·FR-87 |
| CB-3 | 광고·제휴 고지 링크 | 텍스트 | → `/ads`(SCR-08) | FR-84·FR-87 |
| CB-4 | 거부 버튼 | Secondary | 비개인화/미노출 선택, 상태 로컬 저장 | FR-79 |
| CB-5 | 동의 버튼 | Primary | 개인화 광고 선택, 상태 로컬 저장 | FR-78·FR-79 |

- 선택 결과는 **광고 종류(개인화/비개인화/미노출)만 결정**하고 사이트 이용을 막지 않는다(FR-79). 링크 대상(SCR-08)은 반드시 도달 가능(404 금지, FR-87).

**상태 변형**

| 상태 | 표현 |
| --- | --- |
| 첫 방문(기본) | 배너 노출(하단 고정) |
| 재방문(저장됨) | **미노출**(저장 상태 적용) |
| localStorage/쿠키 불가 | 매 방문 노출, 기본 **비개인화/미노출 폴백**(FR-79·FR-E3) |
| 선택 완료 | 배너 닫힘, 이후 광고 종류에 반영 |

**광고 슬롯**: 해당 없음(배너는 동의 UI). 단, 미선택=미동의로 간주해 광고는 비개인화/미노출 폴백.

**반응형 메모**: 모바일=하단 시트(문안 여러 줄, 버튼 2개 나란히), 데스크톱=하단 바 1행(문안 좌·버튼 우). 본문 스크롤을 막지 않는다.

**추적**: SCR-61(전역)·SCR-13(랜딩) / FLOW §4.3·FLOW-60 / FR-78·79·87 / UC-62.

## 5. 광고 슬롯 컴포넌트 (SCR-62, 수동/자동)

**목적**: page_type 게이팅으로만 마운트되는 재사용 광고 슬롯. **"광고" 라벨 + 예약 높이(CLS 방지) + 미노출 폴백**을 표준화한다. 자동광고와 수동 슬롯 공통 컨테이너.

**와이어 — 모바일 (수동 슬롯, 노출 상태)**

```
┌──────────────────────────────┐
│ 광고                         │ ← "광고" 라벨(좌상단, 11~12px, 은폐 금지)
│ ┌──────────────────────────┐ │
│ │                          │ │
│ │      (AdSense 유닛)       │ │  예약높이 min-height 확보
│ │                          │ │  (CLS ≤ 0.1)
│ └──────────────────────────┘ │
└──────────────────────────────┘
```

**와이어 — 미노출 폴백(미승인·차단기·미동의·실패)**

```
┌──────────────────────────────┐
│ 광고                         │
│ ┌──────────────────────────┐ │  ← 예약 컨테이너 유지 또는
│ │        (빈 예약 영역)      │ │     초기 렌더 시 안정적 접힘.
│ └──────────────────────────┘ │     사후 레이아웃 이동 없음
└──────────────────────────────┘
```

**구획별 설명**

| 구획ID | 요소 | 콘텐츠 | 동작 | 관련 FR |
| --- | --- | --- | --- | --- |
| AD-1 | "광고" 라벨 | "광고"(또는 "유료 광고") | 상시 표기, 인지 가능 색·크기(은폐 금지) | FR-76·NFR19 |
| AD-2 | 슬롯 컨테이너 | 예약 높이(min-height) 박스 | 본문 색인 이후 클라이언트 스크립트로 광고 주입 | FR-71·72·74 |
| AD-3 | 자동광고 스니펫 | `AD_CLIENT` 참조(단일 설정) | page_type=ON일 때만 1회 삽입 | FR-70·71 |

**상태 변형**

| 상태 | 표현 |
| --- | --- |
| 노출(승인·동의) | 예약 높이 안에 광고 유닛 표시 |
| 미승인/플레이스홀더 | 빈 예약 컨테이너(레이아웃 무손상) |
| 차단기/네트워크 실패 | 안정적 접힘 또는 빈 예약(사후 이동 0) |
| 미동의 폴백 | 비개인화 또는 미노출, 예약 높이 유지(FR-79) |

**광고 슬롯 위치 정책(§5.1 배치표 정본 인용)**:

| page_type | 자동광고 | 수동 슬롯 위치·개수 | 제휴 |
| --- | :---: | --- | :---: |
| landing | ON | 콘텐츠 하단 1 | 선택 |
| company | ON | 본문 중간 1 + 하단 1 | O |
| combo | ON | 본문 중간 1 + 하단 1 | O |
| input | **OFF** | **없음(무광고 강제)** | **없음** |
| result | OFF | 리포트 하단 1(절제) | 선택 |
| policy | OFF/최소 | 없음 | 없음 |
| (검색·선택 뷰·셸·404) | OFF | 없음(안전 기본값) | 없음 |

**반응형 메모**: 슬롯은 반응형 유닛. 예약 높이는 모바일/데스크톱 각 규격에 맞춰 CSS `min-height`로 미리 확보(삽입 전후 이동 방지).

**추적**: SCR-62 / FLOW §5.1·FLOW-62 / FR-70·71·72·73·74·76 / UC-60.

## 6. 제휴 카드 컴포넌트 (SCR-62)

**목적**: `affiliate.json` 데이터에서 현재 page_type에 활성인 항목만 렌더하는 재사용 제휴 카드. "광고" 표기 + 광고성 링크 속성(`rel="sponsored nofollow"`·`noopener`) 적용. 링크는 코드 하드코딩 금지, 데이터에서만 온다.

**와이어 — 모바일**

```
┌──────────────────────────────┐
│ 광고                         │ ← "광고" 라벨(FR-76)
│ ┌──────────────────────────┐ │
│ │ 이력서 첨삭 서비스         │ │  label (이스케이프)
│ │ 이직 준비 관련 서비스       │ │  desc (선택, 이스케이프)
│ │                [ 바로가기 →]│ │  a rel="sponsored nofollow
│ └──────────────────────────┘ │      noopener" target=_blank
└──────────────────────────────┘
```

**구획별 설명**

| 구획ID | 요소 | 콘텐츠 | 동작 | 관련 FR |
| --- | --- | --- | --- | --- |
| AF-1 | "광고" 라벨 | "광고" | 상시 표기 | FR-76 |
| AF-2 | 라벨(label) | `items[].label`(한국어) | HTML 이스케이프 렌더 | FR-75 |
| AF-3 | 설명(desc) | `items[].desc`(선택) | 이스케이프, null이면 생략 | FR-75 |
| AF-4 | 바로가기 링크 | `items[].url`(http/https만) | `rel="sponsored nofollow noopener"`, `target=_blank` | FR-77 |

- 필터: `active===true && page_types.includes(page_type)`인 항목만. `input`·`policy`에는 어떤 항목도 렌더 금지(FR-75·FR-73). `url`이 http/https 스킴 아니면 렌더 안 함(NFR21).

**상태 변형**

| 상태 | 표현 |
| --- | --- |
| 활성 항목 있음 | 카드 렌더(항목당 1) |
| 해당 page_type 항목 없음/전부 비활성 | **컴포넌트 미노출** |
| input·policy | 항상 미노출(게이팅) |

**광고 슬롯 위치**: company·combo 본문 중간/하단, landing·result 하단(선택). input·policy·검색뷰=없음.

**반응형 메모**: 모바일=세로 카드(라벨·설명·CTA 스택), 데스크톱=카드 가로 폭 확대. 다중 항목은 세로 나열 또는 데스크톱 2열.

**추적**: SCR-62 / FLOW §5·FLOW-62 / FR-75·76·77 / UC-61.

## 7. 배지 컴포넌트 (공식/추정·금액 estimated·검증/만료)

**목적**: 복지 데이터의 신뢰 신호를 재사용 배지로 표준화. 회사 상세(SCR-51)·조합(SCR-54)·리포트(SCR-05)가 이 배지를 인용한다. 데이터 소스 = `TCOMPANY_BENEFIT`(BADGE_CD est/official, BENEFIT_AMT, VERIFIED_DTM, EXPIRES_DTM).

**와이어 — 배지 유형 카탈로그**

```
[공식]        official — 공식 출처 확인       (강조색·체크)

₩4,800만 (추정 ±20%)   estimated 금액 + 불확실성 밴드
₩5,000만 (공식 ±5%)    official 금액 + 밴드
검증 26.03.01          VERIFIED_DTM 표시
만료 25.12.31 ⚠        EXPIRES_DTM 지남 = 만료 경고 (±15%)
출처 ↗                 BADGE_SRC_URL_CTNT 링크(있으면)
```

**와이어 — 복지 항목 행에 적용(모바일)**

```
┌──────────────────────────────┐
│ 식대 지원            ₩20만/월  │
│ [공식]  출처 ↗  검증 26.03.01 │
├──────────────────────────────┤
│ 자기계발비      ₩100만/년(추정)│
│ [추정 ±20%]     만료 25.12.31⚠ │
├──────────────────────────────┤
│ 리프레시 휴가         정성적    │  ← BENEFIT_AMT NULL(QUAL_YN=Y)
│ [공식]  "연 1회 2주"           │     QUAL_DESC_CTNT 표시
└──────────────────────────────┘
```

**구획별 설명**

| 구획ID | 요소 | 콘텐츠 | 동작 | 관련 FR |
| --- | --- | --- | --- | --- |
| BG-1 | 공식/추정 배지 | `[공식]`/`[추정]`(BADGE_CD) | 시각 구분(색·아이콘), 무동작 | FR-54·FR-41 |

**상태 변형**

| 상태 | 표현 |
| --- | --- |

| 정성적(금액 NULL) | 금액 숨김, "정성적" + 서술 |
| 출처 URL 없음 | "출처 ↗" 생략 |

**광고 슬롯**: 없음(데이터 신뢰 표시 요소).

**반응형 메모**: 모바일=복지명·금액 1행, 배지·검증·만료·출처를 2행째에 랩. 데스크톱=1행에 배지 인라인 정렬.
