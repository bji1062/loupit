# 회사 상세·인기 조합 화면 (WIREFRAME)

**문서 목적**: 정적 콘텐츠 두 유형 — **회사 상세 페이지(`/company/{slug}`, `page_type=company`, 복지 데이터 보유 회사 ~96개)**와 **인기 비교 조합 페이지(`/vs/{a}-{b}`, `page_type=combo`)** — 의 저충실도(lo-fi) 레이아웃을 확정한다. FLOW `06-회사상세-인기조합.md`가 소유한 화면 ID `SCR-50`~`SCR-56`의 구획(G/B/C/V/M/W/L)·상태·광고 슬롯을 **모바일 우선 ASCII 와이어프레임**으로 그리고, 각 구획의 요소·콘텐츠·동작·근거 FR을 표로 명세한다. 픽셀 값·색·타이포는 규정하지 않는다(SPEC 소유).

**상위 추적**: WIREFRAME → FLOW → FRD → USECASE → PRD → 브리프. 상위 근거 = FLOW [06-회사상세-인기조합.md](../FLOW/06-회사상세-인기조합.md)(SCR-50~56·FLOW-50·51·52), FRD [07-회사페이지-정적생성.md](../FRD/07-회사페이지-정적생성.md)(FR-50~59), FRD [08-인기조합-정적생성.md](../FRD/08-인기조합-정적생성.md)(FR-60~65). 전역 공통 블록(헤더·푸터·동의 배너·광고 슬롯·제휴 카드·배지·버튼)은 WIREFRAME [01-공통-레이아웃.md](01-공통-레이아웃.md)의 §1 헤더(HDR-1/2)·§2 푸터(FTR-1~6)·§3 페이지 골격·§4 동의 배너(SCR-61)·§5 광고 슬롯(SCR-62·배치표 §5.1)·§6 제휴 카드(SCR-62)·§7 배지·§8 버튼(BTN-3)을 **이름으로 인용**하며 재정의하지 않는다. 형식·ASCII 스타일·상세도는 WIREFRAME [02-랜딩.md](02-랜딩.md)와 일치시킨다.

**범위 경계**: 본 문서는 **SCR-50~56 두 정적 페이지 유형의 저충실도 레이아웃**만 소유한다.
- (1) 전역 헤더·푸터·동의 배너·광고 슬롯·제휴 카드·배지·CTA 버튼의 정본 구성은 WIREFRAME 01이 소유하고 본 문서는 위치·상태·게이팅만 그린다(재정의 금지).
- (2) CTA 클릭 이후 진입 목적지(비교 툴 셸 SCR-02, 검색·슬롯 반영 SCR-20·SCR-23, 입력 SCR-04, 리포트 SCR-05)의 내부 레이아웃은 비교 계열 WIREFRAME이 소유하고 본 문서는 **CTA 발신 지점·프리필 파라미터 계약**까지만 그린다(FR-57·FR-62).
- (3) 미생성 slug·미배포 조합의 404 화면(SCR-09)은 FLOW 01 소유이며 본 문서는 상태 전이로만 인용한다.
- (4) 배지 파생·불확실성 밴드 계수는 §7 배지·FR-05/FR-D9(전역) 소유이며 본 문서는 표시(라벨·출처·검증/만료)만 그린다.
- **데이터 폴백 정합(2026-07-09 결정)**: `company`·`combo` 페이지는 등록 회사(모두 실복지 보유)만 대상이므로 **회사페이지용 프리셋 폴백이 없다**(브리프 §5, FR-53·FR-59, FR-61 R3). `TBENEFIT_PRESET`는 비교 툴 직접 입력 모드(등록 외 회사) 전용이다. 본 와이어는 이 결정을 따르며, 두 페이지의 복지·요약은 항상 `TCOMPANY_BENEFIT` 실데이터(amt_source=`real`)로 렌더한다.

로그인/회원/프로파일러/서버 측 사용자 쓰기 UI는 제품 범위에서 영구 제외이므로 어떤 구획·상태에도 없다(FR-01). 열람·CTA 진입 전 과정에서 사용자 데이터의 서버 전송·저장이 없고, 프리필 등 상태는 URL 파라미터·클라이언트에만 존재한다(NFR16). 본문(기업정보·근무형태·복지·요약·내부링크)은 JS 없이 초기 HTML에 완성되어 크롤러가 색인한다(NFR12·NFR24, FR-50·FR-60). 회사명·조합명은 레이아웃 예시일 뿐 특정 회사를 창작·확정하지 않으며 실제 목록은 빌드타임 큐레이션(96개)이 확정한다.

## [SCR-50] 회사 상세 페이지 `/company/{slug}` (정적, `page_type=company`)

| 미생성 slug·오타 | 정적 파일 부재 → **SCR-09 404 안내**(색인 제외·무크래시) | FR-59·FR-E6(UC-40 2a) |

### 광고 슬롯 위치 (`page_type=company`, §5.1 배치표)

- **자동광고 ON** + **수동 슬롯 2개(`content_mid` + `content_bottom`)** + **제휴 O**, 밀도 **상**(§5.1 정본).
- 위치: `content_mid`는 CTA(G5) 아래·내부링크(G6) 위(섹션 사이), `content_bottom`은 내부링크(G6) 아래·푸터(G8) 위. **복지표(SCR-51) 구획 내부에는 슬롯 없음**(가독 보호, FLOW 06 SCR-51). CTA는 콘텐츠 요소로 "광고" 표기 대상 아님.
- 모든 슬롯 예약 높이 고정(CLS ≤ 0.1, FR-74), "광고" 라벨 부착(FR-76). 광고 미실행(크롤러·차단기·미동의)에도 본문 표시·색인 유지(FR-E5·NFR24).

### 반응형 메모

- 모바일: 전 구획 1열 세로 스택. 복지표는 항목명·금액 1행, 배지·검증/만료·출처를 2행째 랩(§7). 헤더 sticky.
- 데스크톱: 콘텐츠 max-width ~960px 중앙. G2 로고·기업정보 가로 배치, G3 근무형태 인라인, 복지표 다열 `<table>`, CTA 중앙. 광고는 본문 폭 반응형.

### 관련 SCR·FR·UC 추적

- **SCR**: SCR-50(본), 상위 SCR-06. 구획 SCR-51·52. 전이 SCR-02·SCR-23(프리필 소비)·SCR-53·SCR-08·SCR-09. 공통 HDR/FTR/SCR-61·SCR-62·§7.

## [SCR-52] "이 회사로 비교하기" CTA 프리필 진입점 (SCR-50 구획 → SPA)

## [SCR-55] 양사 프리필 "비교 툴에서 열기" 진입점 (SCR-53 구획 → SPA)

**목적**: 조합의 두 회사를 슬롯 a(현직)·b(이직처)에 **동시에** 사전 지정한 상태로 비교 툴에 진입시키는 양사 프리필 진입점. 회사 상세 단일 슬롯 CTA(SCR-52)의 양사 확장이며, 회사 지시는 경로 파싱이 아니라 명시적 파라미터로 전달한다(FR-62 R2, UC-52). 버튼 위계는 §8 BTN-3 인용.

### 모바일 와이어

```
┌─ 📱 SCR-55 양사 프리필 "비교 툴에서 열기" (SCR-53 구획 → SPA) ┐
│
│    ┌────────────────────────────────┐
│    │  ▶ 비교 툴에서 열기              │   W1 순수 <a href> (BTN-3)
│    └────────────────────────────────┘
│
│   href = /compare?a={식별자}&b={식별자}            W2 양사 지시자·슬롯 계약
│    · a = 정규 첫 회사 → 슬롯 a(현직)
│    · b = 둘째 회사    → 슬롯 b(이직처)
│    · 식별자 = COMP_ID 또는 COMP_ENG_NM/회사명(명시적 파라미터)
│    · 상태는 URL·클라이언트만(서버 전송·저장 없음)
│
│   전이(FLOW-51):
│    클릭 → SCR-02 셸 부팅 · reference/all 로드(FR-02)

│    입력·재계산 → SCR-05 리포트(vdCard) · 클라이언트 계산만
```

**광고 정책 준수**: `company`·`combo`는 콘텐츠 페이지로 **자동광고 ON + 본문 중간(`content_mid`) 1 + 하단(`content_bottom`) 1 + 제휴 O**(밀도 상, §5.1 정본). 복지표(SCR-51)·요약(SCR-54)·내부링크(SCR-56) 구획 내부에는 광고를 두지 않고 섹션 사이·본문 하단에만 배치한다. 모든 슬롯 "광고" 표기·예약 높이 고정(CLS ≤ 0.1)·비-JS 본문 색인 무손상(FR-58·65·74·76·E5). CTA는 콘텐츠 요소로 광고가 아니며, 진입 후 비교 입력(`input`)은 무광고 강제.

**데이터·범위 정합**: 등록 회사 ~96개(200 아님)는 모두 실복지를 보유하므로 두 페이지의 복지·요약은 실데이터(amt_source=`real`)로만 렌더하고 **프리셋 폴백이 없다**(브리프 §5, FR-53·59·61 R3, 2026-07-09 결정). 정적 생성(JS 없이 본문 색인), 로그인/프로파일러/서버 쓰기 UI는 어떤 구획·상태에도 없다(FR-01). 회사명·조합명은 레이아웃 예시이며 실제 목록은 빌드타임 큐레이션이 확정한다.
