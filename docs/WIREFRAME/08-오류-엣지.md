# 오류·엣지 화면 (WIREFRAME)

**문서 목적**: 정상 경로가 아닌 **비정상·엣지 상황**의 저충실도(lo-fi) 레이아웃을 확정한다. FLOW `08-오류-엣지-플로우.md`가 소유한 화면·상태 ID `SCR-90`~`SCR-95`의 구획(X/Y/Z/W/V/U 대역)·상태·복구 경로를 **모바일 우선 ASCII 와이어프레임**으로 그리고, 각 구획의 요소·콘텐츠·복구 동작·근거 FR을 표로 명세한다. 다루는 상황은 (1) 참조 번들 `reference/all` 로드 실패의 재시도 UX, (2) 검색 API 실패(오류≠무결과 구분·번들 폴백), (3) localStorage 사용 불가 비침투 안내, (4) 기업유형 프리셋 직접 입력 기본복지 표시(등록 96개 밖 회사, 프리셋은 폴백이 아니라 직접입력 템플릿), (5) 404 페이지(없는 회사 slug·잘못된 URL), (6) 비-JS·크롤러 정적 본문 폴백(점진적 향상)이다. 픽셀 값·색·타이포·최종 카피는 규정하지 않는다(SPEC 소유).

**상위 추적**: WIREFRAME → FLOW → FRD → USECASE → PRD → 브리프. 상위 근거 = FLOW [08-오류-엣지-플로우.md](../FLOW/08-오류-엣지-플로우.md)(SCR-90~95·구획 X1~X4/Y1~Y4/Z1~Z3/W1~W5/V1~V3/U1~U4·FLOW-90~96), FRD [12-오류-엣지.md](../FRD/12-오류-엣지.md)(FR-E1~FR-E7). 전역 헤더/푸터·광고 슬롯·제휴 카드·배지·버튼·폼 등 공통 블록은 WIREFRAME [01-공통-레이아웃.md](01-공통-레이아웃.md)(HDR-1/2·FTR-1~6·AD-1/2/3·AF-1~4·BG-1~6·BTN-1~4·FM-1~6, §5.1 광고 배치표)를 **이름으로 인용**하고 재정의하지 않는다. 형식·상세도 정합 근거 = WIREFRAME [02-랜딩.md](02-랜딩.md)(특히 SCR-11 로딩·SCR-12 참조 번들 실패·SCR-13 동의 오버레이의 상태 표현). 브리프 §2-2(localStorage)·§2-4(서버 무쓰기)·§2-6(광고 배치)·§2-8(클라이언트 계산)·§4/§5(데이터 모델·프리셋 직접입력 모드).

**범위 경계**: 본 문서는 **오류·엣지 상태/렌더 모드의 저충실도 레이아웃**만 소유한다. SCR-9x는 특정 라우트 하나가 아니라 여러 기능 화면 위에 겹치는 **상태·렌더 모드**이며(02의 SCR-11/12/13이 SCR-10 위 상태인 것과 동형), 호스트 화면(비교 툴 셸 SCR-02·검색 뷰 SCR-03·입력 SCR-04·리포트 SCR-05·회사상세 SCR-06·조합 SCR-07·정책 SCR-08·404 안내 SCR-09·F1 무결과 빈 상태 SCR-22)과 공통 블록을 **인용**하되 재정의하지 않는다. 정상(해피패스) 본문 레이아웃은 각 담당 WIREFRAME이 소유한다. 프로파일러(가치관 진단 설문)·익명 서버 측 사용자 쓰기 UI는 제품 범위에서 영구 제외이므로 어떤 오류·엣지 화면·상태에도 등장하지 않는다; 로그인·회원·마이페이지·복지편집 UI는 복지 등록·수정 기여(SC14, 문서 09·13·SP-AUTH 소관) 한정 In-scope이나 이 익명 화면엔 등장하지 않는다(FR-01). 어떤 상태에서도 사용자 입력·상태의 서버 전송·저장은 없다(NFR16·NFR20).

**핵심 불변식(전 화면 적용)**: **무크래시(NFR26)** — 어떤 실패에도 미처리 예외·흰 화면을 남기지 않는다. **오류 ≠ 무결과** — 실패(네트워크·타임아웃·5xx·파싱)와 정상 빈 결과(검색 0건·저장분 없음)는 서로 다른 문구·복구 행동으로 구분한다. **렌더 안전(NFR21)** — 회사명·검색어·복지 등 삽입 문자열은 `textContent`/이스케이프로 삽입한다. **한정 재시도** — 무한 자동 재시도·무한 스피너 금지, "다시 시도"는 트리거당 재요청 1회.

## [SCR-90] 참조 번들 로드 실패 오류 상태

**목적**: 비교 툴 셸(SCR-02)이 부팅 시 1회 로드하는 참조 번들(REF)의 로드가 실패하거나 손상 응답이 도착했을 때, 앱을 크래시시키지 않고 오류 상태로 전이하여 **재시도 수단**을 제공하고, 참조 확보 전까지 검색(F1)·비교 입력(F2)·비교 계산(F3)을 비활성한다. FR-E1(로드 실패·재시도)·FR-02(부팅 로드)·FR-D1(번들 계약)의 실패 경로를 화면으로 확정한다. UC-90을 담당한다. (랜딩에서의 동종 실패는 정적 본문 위 오류 배너인 SCR-12가 별도로 소유하며, 본 상태는 **비교 툴 셸 내부**의 오류 상태다.)

## [SCR-91] 검색 실패·번들 폴백 매칭 상태

**목적**: 회사 검색(`GET /api/v1/companies/search?q=`)이 실패했을 때, 앱을 크래시시키지 않고 **실패를 무결과와 구분**해 안내하며, 참조 번들(REF)이 이미 로드되어 있으면 `REF.companies[]`의 이름·별칭 클라이언트 매칭으로 검색을 계속하게 한다. 검색 무결과·오류 빈 상태의 화면 UX 자체는 F1의 SCR-22가 소유하고, 본 상태는 그 위에 실패의 **횡단 처리**(오류/무결과 구분·번들 폴백·무크래시)를 그린다. UC-91/FR-E2를 담당한다.

## [SCR-92] localStorage 불가 비침투 안내 상태

**목적**: 브라우저 localStorage가 사용 불가(사생활/시크릿 모드·저장소 차단·용량 초과 `QuotaExceededError`)할 때, 저장·복원 실패를 예외로 흡수하여 **비교 기능은 정상 동작**시키고 "최근 비교"만 저장·복원되지 않게 한다. 차단성 오류 화면을 강제하지 않고 **비침투적으로만** 안내한다. FR-E3(localStorage 불가 흡수)·FR-07(로컬 지속성)의 실패 경로를 확정한다. UC-92를 담당한다.

## [SCR-93] 기업유형 프리셋 직접입력 기본복지 표시 상태

**목적**: 비교 툴 **직접 입력 모드**에서 사용자가 등록 회사(복지 실데이터 보유 ~96개)에 없는 회사를 직접 입력하고 기업유형(`comp_tp_cd`)을 고르면, 대응하는 기본 복지 프리셋(TBENEFIT_PRESET)으로 초기 복지 **템플릿**을 채워 비교 입력이 정상 구성되게 한다(UC-15/21). 프리셋은 회사페이지 폴백이 아니라 **직접 입력 템플릿**이다 — 등록 회사는 모두 실복지를 보유하므로 회사 상세·조합·`companies/{id}` 응답에는 프리셋을 병합하지 않으며, 미존재 `comp_id`는 404로 처리한다(SCR-95). FR-E4·FR-06·FR-D3이 규약을 소유하고, 본 상태는 **직접 입력 시 프리셋 채움과 사용자 편집**을 그린다. UC-93을 담당한다.

- 모바일: 유형 칩(FM-5 세그먼트 유사) 랩 → 프리셋 복지 목록 1열 세로(체크+금액+배지 2행 랩, 공통 배지 §7 반응형 규약).

### 추적

## [SCR-94] 비-JS·크롤러 정적 본문 폴백 상태

**목적**: JavaScript를 실행하지 않는 방문자(JS 비활성 브라우저 A2)·검색 크롤러(A3)가 정적 콘텐츠 페이지(회사 상세 F4·인기 조합 F5·정책 F7)에 접근했을 때, 초기 HTML만으로 **본문이 가독·색인**되게 하고, 인터랙션(비교 CTA)·광고 슬롯은 JS 향상 요소로 분리한다(점진적 향상). 정적 본문 구성 자체는 FR-5x·FR-6x가 소유하고, 본 상태는 그 **비-JS 렌더 모드와 향상 경계**를 그린다. UC-94/FR-E5를 담당한다.

## [SCR-95] 404·not-found 오류 상태

**목적**: 존재하지 않는 회사 slug·미배포 조합·잘못된 URL 요청, 그리고 `companies/{id}` API의 미존재 식별자 요청을 **404/not-found**로 일관 처리하여 색인에서 제외하고, 앱을 크래시시키지 않으며, 유효 페이지로의 이동 경로를 안내한다. 중복 경로(조합 역순 등)는 canonical로 정규화한다. 정적 404 페이지는 사이트맵의 SCR-09로 렌더되며, 본 상태는 그 404·not-found를 유발하는 처리·흡수를 소유한다. UC-95/FR-E6을 담당한다.
