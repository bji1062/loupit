## C. 반드시 알아야 할 함정

### C-1. 테스트는 `run_tests.sh`로만 (기존 규칙, 여전히 유효)
서빙 스키마 `LOUPIT`을 테스트에도 재사용한다. 맨 `pytest server/tests`는 conftest 가드가
`[C-1 안전장치]`로 차단한다(정상 동작). 백엔드 검증은 **`bash infra/deploy/run_tests.sh`** 또는
release 게이트 경유로만. generator·web 테스트는 DB 무관이라 직접 실행 안전:
```bash
python3 -m pytest -q generator/tests/     # DB 무접촉
node --test 'web/**/*.test.js'            # ⚠ node v24는 `node --test web/`를 모듈로 오해 — glob 필수
```

### C-2. 배포 = `RELEASE_CONFIRM=1 bash infra/deploy/release.sh`
게이트가 최선두라 실패 시 서빙 상태가 바뀌지 않는다. TCOMPARE_LOG는 백업→재시드→재주입으로 보존된다.

### C-3. ⚠️ **nginx conf는 리포→`/etc` 자동 배포가 없다**
`release.sh`는 `reload`만 하고 **conf를 복사하지 않는다**. 프로덕션 conf를 바꿨다면 수동 배치 필요:
```bash
sudo cp /etc/nginx/sites-available/loupit.conf /root/nginx-loupit.conf.bak-$(date +%Y%m%d-%H%M%S)
sudo cp infra/nginx/loupit.conf /etc/nginx/sites-available/loupit.conf
sudo nginx -t && sudo systemctl reload nginx
```
- **순서 주의**: 작업트리가 곧 라이브 문서루트라, 새 경로를 참조하는 HTML을 먼저 커밋하면
  conf 적용 전까지 그 경로가 전부 404가 된다. **conf 먼저 → 참조 변경 나중.**
- `nginx -t`는 `alias`/`root` 대상 디렉터리 존재를 검사하지 않는다(오타는 런타임 404).
  스모크 SM-15a~e가 이걸 잡는 유일한 선이다.
- **beta conf는 리포본을 복사하지 말 것** — 리포는 `beta.jobcho.wiki` 인증서를 참조하는데 미발급이라
  nginx 기동이 실패한다. 라이브 beta는 `beta.loupit.co` 변형에 필요한 블록만 제자리 패치해 둔 상태.

### C-4. 자산은 `/assets/v2/` 세대 + no-cache
- **`web/assets`에 `.gz` 사전압축 금지**(불변식). gzip_static이 원본과 신선도를 비교하지 않아
  no-cache 재검증을 조용히 무력화한다. v2 location은 `gzip_static off`.
- 새 자산 참조는 반드시 `/assets/v2/…`. 구 `/assets/` 경로는 하위호환용 no-cache 서빙만 유지.

### C-5. 백업은 살아 있다
`loupit-backup.timer` enabled+active(매일 03:00), `/var/backups/loupit/`에 축적 중,
6테이블 전량 + TCOMPARE_LOG 데이터 포함, `Dump completed` 트레일러 정상. 다만 §B-4의 갭 참고.

### C-6. 서버 접촉 작업은 이 호스트에서만
nginx·systemd·MySQL·라이브 검증은 `158.180.79.39`에서. `server/.env`를 root로 편집하면 소유권이
바뀌어 API 두 개가 재시작 시 죽는다 — 편집 후 `sudo chown ubuntu:ubuntu server/.env && chmod 600`.

### C-7. 브라우저 검증은 playwright 직접 호출
gstack browse는 root 환경에서 기동 실패한다. 대신:
```js
import { chromium } from '/root/.claude/skills/gstack/node_modules/playwright/index.mjs';
const browser = await chromium.launch({ chromiumSandbox: false });
```

## D. 이 세션에서 배운 것 (재발 방지)

1. **"테스트 green"이 "동작함"을 뜻하지 않는다.** 감사 #12를 완료로 표시했지만 `web/`만 고치고
   generator 정적 경로를 안 건드려, 실제로는 102개 페이지에서 광고·동의가 미실행이었다.
   같은 기능이 두 경로(SPA 셸 / 정적 생성)에 있으면 **양쪽을 다 확인**할 것.
2. **그래프 속성은 그래프로 검증해야 한다.** 내부링크 1차 구현은 "회사당 링크 ≥1"만 테스트해
   통과했지만, 큰 업종군이 폐쇄 싱크가 되어 크롤러가 갇히는 구조였다. 연결성이 유지된 유일한
   이유는 한 회사의 수동 메타 한 줄이었다. → 도달성 회귀 테스트 GC-26 신설.
3. **가드가 사각지대를 가질 수 있다.** GC-20(죽은 링크 404 가드)이 정작 새로 추가한 `/vs/` 링크를
   검사하지 못했다(테스트가 프로덕션과 다른 배선으로 호출). **테스트 배선 == 프로덕션 배선**.
4. **스모크는 상태코드를 봐야 한다.** SM-9가 gzip만 검사해 404 폴백(그것도 gzip)을 통과시키고 있었다.
5. **문서-코드 드리프트는 조용히 쌓인다.** SPEC/TASK 여러 곳이 이미 폐기된 계약을 현행처럼 기술 중이었다.
   계약을 바꾸면 SPEC 본문·검증표·TASK 리프를 **같은 커밋에서** 함께 고칠 것.
