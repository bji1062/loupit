# 광고와 정책 (USECASE)

**목적**: 콘텐츠 페이지에서의 광고 노출·조회, 제휴 링크 클릭("광고" 표기 인지), 광고·쿠키 개인화 동의/거부, 정책·고지 페이지(개인정보처리방침·이용약관·면책·광고/제휴 고지) 열람이라는 수익화·규정 준수 상호작용을 F6(광고·제휴 슬롯)·F7(정책 페이지)에 근거하여 표준 유즈케이스로 정의한다.

**상위 추적**: 본 문서는 USECASE→PRD 체인에 속하며, 각 UC는 PRD 기능 요구사항 F6·F7과 수익화 요구사항 MON*(PRD 06-수익화.md, MON1~MON19)을 인용한다. 사용자 기준점은 타깃 사용자 PRD-SEG-1·PRD-SEG-2(비교 툴 사용자, A1)·PRD-SEG-3(검색 유입 열람자, A2)이며, 광고·제휴 공급 주체는 A4, 정책 페이지 게시·배포 주체는 A5다(01-액터와-개요). 배치·동의·표기 근거는 브리프 §2-5(수익화·"광고" 표기)·§2-6(배치 정책)·§2-7(SEO=승인 콘텐츠), §8(AdSense 스니펫·플레이스홀더 client id·affiliate.json)이다.

**범위 경계**: 이 문서는 F6(광고·제휴 노출·클릭·동의)와 F7(정책 페이지 열람)만 다룬다. 광고가 배치되는 콘텐츠 페이지 자체의 열람(회사 상세 F4=UC-4x, 인기 조합 F5)과 비교 입력·리포트(F2/F3=UC-2x·UC-3x)는 각 담당 문서에서 다루며, 본 문서는 그 페이지들 위에서 일어나는 광고·제휴·동의·정책 상호작용에 한정한다. 01문서 마스터 인덱스의 예정 항목 UC-M1(광고 슬롯 렌더링)·UC-M2(제휴 링크 렌더/클릭)·UC-L1(정책 페이지 열람)을 본 문서가 UC-60~UC-63으로 상세화한다(도메인 문자 대신 담당 문서 번호 기반 UC-6x 대역 사용, 02~05 문서 관례와 정합). 프로파일러와 익명 경로 서버 사용자저장은 제품 범위에서 영구 제외이며, 로그인·회원은 이 익명 광고·정책 흐름에 등장하지 않고(로그인·재직 인증은 복지 등록·수정 기여 SC14 한정 — 09-참여.md), 동의 상태를 포함한 모든 사용자 상태는 브라우저(localStorage/쿠키)에만 존재한다(브리프 §2-1·§2-2, PRD 04-범위 SC10·SC14).

## [UC-60] 콘텐츠 페이지 광고 슬롯 노출·조회("광고" 표기)

**액터**: 노출 소비 — A1 익명 방문자(비교 결과 화면, PRD-SEG-1·2)·A2 검색엔진 유입 사용자(랜딩·회사 상세·인기 조합, PRD-SEG-3). 공급 — A4 광고·제휴 시스템(AdSense). 설정 — A5 빌드/운영자(MON5 주입).

**사전조건**
- 공통 사전조건 1~3.
- 자동광고 스니펫은 페이지 유형별 조건부로만 로드되고(MON3), 수동 슬롯은 지정 위치에 배치된다(MON4).
- 광고 슬롯 컨테이너는 예약 높이/폴백을 가진다(MON6, CLS 방지).

**기본 흐름**
1. 사용자가 콘텐츠 페이지에 진입한다: 랜딩(MON7-1) / 회사 상세(MON7-2) / 인기 비교 조합(MON7-3), 또는 비교 결과 화면(MON7-5)을 연다.
2. 페이지 유형 판별로 광고 컴포넌트 마운트가 게이팅된다(MON8): 콘텐츠 페이지는 자동광고 ON + 수동 슬롯(본문 중간·하단), 비교 결과 화면은 리포트 하단 절제 슬롯 1개만(MON9).
3. A4(AdSense)가 마운트된 슬롯에 광고를 공급·렌더한다.
4. 각 광고 영역에 "광고" 표기가 사용자 인지 가능하게 부착되어 콘텐츠와 명확히 구분된다(MON11, MON18).
5. 사용자가 콘텐츠를 열람하며 구분된 광고를 조회(노출)한다. 이 노출이 SEO 콘텐츠 기반 수익의 실현 지점이다(MON2).

**대안·예외 흐름**

| 분기 | 조건 | 처리 |
|------|------|------|
| 1a | 사용자가 비교 입력 화면(MON7-4)에 진입 | 자동광고 스니펫·수동 슬롯·제휴 컴포넌트를 렌더링하지 않는다(MON8). 입력 화면은 광고 노출 없음(UX 보호, 결정 6) |
| 2a | 사용자가 정책 페이지(MON7-6)에 진입 | 자동광고 OFF/최소, 수동 슬롯 없음. 얇은 유틸리티 페이지로 AdSense 정책 정합성 유지(UC-63) |
| 3a | 광고 로드 실패·미승인·광고 차단기 | 슬롯 컨테이너의 예약 높이/폴백으로 레이아웃 무손상(MON6). 승인 전에는 플레이스홀더 유지(MON5·MON18) |
| 3b | 회사 상세/조합 페이지 본문 색인 시점 | 광고는 클라이언트 스크립트로 주입되므로 정적 본문 색인에 영향 없음(UC-44와 정합, §2-7) |
| 4a | 개인화 동의 상태에 따른 광고 종류 | 동의 시 개인화, 미동의/거부 시 비개인화 또는 미노출로 폴백(UC-62/MON15). 어느 경우든 본 슬롯 노출·레이아웃 규칙은 동일 적용 |

**사후조건**
- 콘텐츠 페이지 및 비교 결과 화면 하단에 "광고" 표기된 슬롯이 노출된다. 비교 입력 화면은 무광고를 유지한다.
- 공통 사후조건 제약(서버 사용자저장 없음). 개인화 여부는 UC-62의 클라이언트 동의 상태를 따른다.

**추적**: F6 / A1·A2(노출)·A4(공급)·A5(주입) / MON2·MON3·MON4·MON6·MON7·MON8·MON9·MON11·MON18 / 결정 5·6, §2-5·§2-6 / 연계 UC-62(동의·폴백)·UC-44(본문 색인 무영향)·UC-63(정책 페이지 무광고).

## [UC-61] 제휴 링크 노출·클릭("광고" 표기 인지)

**액터**: 클릭 — A1 익명 방문자·A2 검색엔진 유입 사용자. 제휴 대상 — A4 광고·제휴 시스템(`affiliate.json` 기반).

**사전조건**
- 공통 사전조건 1~3.
- `affiliate.json`에 현재 페이지 유형에 해당하는 제휴 항목(라벨·링크·대상 페이지 유형 등)이 존재한다(MON10).
- 제휴가 배치 대상인 페이지 유형이다: 회사 상세(MON7-2)·인기 조합(MON7-3)의 "관련 서비스", 랜딩·비교 결과 화면은 선택(MON7 표).

**기본 흐름**
1. 콘텐츠 페이지에서 제휴 컴포넌트가 `affiliate.json` 데이터로 렌더된다(MON10, 개별 제휴 링크 코드 하드코딩 없음).
2. 제휴 영역에 "광고"(또는 "유료 광고") 라벨이 눈에 띄는 색·크기로 명확히 표기된다(숨김 금지, MON11).
3. 사용자가 해당 링크가 광고임을 인지한 상태에서 제휴 링크를 클릭한다.
4. 링크에 광고성 링크 속성 `rel="sponsored nofollow"`와 외부 이동 `target`/`rel="noopener"`가 적용되어 외부 대상으로 이동한다(MON12).
5. 외부 이동은 loupit 서버에 사용자 데이터를 저장하지 않는다.

**대안·예외 흐름**

| 분기 | 조건 | 처리 |
|------|------|------|
| 1a | `affiliate.json`에 해당 페이지 유형 항목 없음 | 제휴 컴포넌트를 노출하지 않는다. 노출·미노출·링크 교체가 데이터만으로 제어되며 코드 수정이 필요 없다(MON10, F6 수용기준 4) |
| 1b | 비교 입력 화면(MON7-4) | 제휴 컴포넌트를 렌더링하지 않는다(MON8) |
| 2a | "광고" 표기를 눈에 띄지 않게 숨기려는 배치 | 금지 — 표시광고 명확성 요건 위반(MON11 주의). 라벨은 인지 가능한 색·크기로 유지 |
| 3a | 사용자가 클릭하지 않고 이탈 | 노출만으로 표기 요건(MON11)은 충족. 서버 저장 없음 |

**사후조건**
- 제휴 링크가 "광고" 표기와 함께 노출되며, 클릭 시 광고성 속성이 적용된 외부 이동이 일어난다.
- 공통 사후조건 제약(서버 사용자저장 없음).

**추적**: F6 / A1·A2(클릭)·A4(제휴) / MON10·MON11·MON12 / 결정 5·7(SEO 링크 속성), §2-5, §8(affiliate.json) / 연계 UC-63(광고·제휴 고지 페이지와 정합).

## [UC-62] 광고·쿠키 개인화 동의/거부

**액터**: A1 익명 방문자·A2 검색엔진 유입 사용자.

**사전조건**
- 공통 사전조건 1~3.
- AdSense가 쿠키를 사용하므로 첫 방문 시 광고/쿠키 고지·동의 UI를 제공할 수 있는 상태다(MON13).
- 동의 상태는 서버 저장 없이 브라우저(localStorage/쿠키)에만 보관 가능한 환경이다(결정 1·2).

**기본 흐름**
1. 첫 방문 시 광고/쿠키 사용 고지와 동의 UI(동의 배너)가 표시된다(MON13).
2. 동의 배너는 개인정보처리방침·광고/제휴 고지(F7 정책 페이지)로 링크되어, 사용자가 쿠키·광고 목적·제3자(Google) 사용 사실을 확인할 수 있다(MON14, 동의↔정책 단일 진실 연결 → UC-63).
3. 사용자가 개인화 광고에 대해 동의 또는 거부를 선택한다.
4. 선택된 동의 상태가 브라우저(localStorage/쿠키)에만 저장된다(서버 전송·저장 없음, 결정 1·2).
5. 동의 시 개인화 광고가, 거부/미동의 시 비개인화 광고 또는 광고 미노출이 적용된다(MON15). 어느 선택에서도 사이트 이용은 가능하다.
6. 폴백(비개인화/미노출) 상황에서도 슬롯 레이아웃은 예약 높이/폴백으로 무손상 유지된다(MON6 → UC-60).

**대안·예외 흐름**

| 분기 | 조건 | 처리 |
|------|------|------|
| 3a | 사용자가 선택 없이 열람 계속 | 미동의 상태로 간주해 비개인화/미노출로 폴백(MON15). 사이트 이용은 정상 |
| 4a | 재방문(동의 상태 기저장) | 브라우저에서 저장된 동의 상태를 읽어 배너 재노출 없이 적용. 서버 계정 복원이 아니며 기기 내에 한정(PRD-CTX-4) |
| 4b | localStorage/쿠키 비활성·차단 | 동의 상태를 보관하지 못해 매 방문 고지. 기본은 비개인화/미노출 폴백(MON15) |
| 1a | 해외(EEA/UK 등) 유입 CMP 요건 | MVP는 한국 대상이며 CMP 필요 여부는 브리프 미확정 → 2차 재검토(창작 금지, 06-수익화.md §5 비고) |

**사후조건**
- 광고 개인화 동의/거부 상태가 기기 내(localStorage/쿠키)에만 저장되고, 그에 맞춰 광고 개인화 또는 폴백이 적용된다.
- 공통 사후조건 제약(서버 저장·크로스 디바이스 동기화 없음).

## [UC-63] 정책·고지 페이지 열람(개인정보처리방침/이용약관/면책/광고·제휴 고지)

**액터**: 열람 — A1 익명 방문자·A2 검색엔진 유입 사용자. 게시·배포 — A5 빌드/운영자(MON17, UC-A4). 색인 — A3 검색 크롤러/봇.

**사전조건**
- 공통 사전조건 1~3.
- 정책 페이지 4종이 정적으로 게시되어 전역 내비게이션(푸터 등)에서 접근 가능하다(MON17, UC-A4 배포 완료).

**기본 흐름**
1. 사용자가 전역 푸터의 정책 링크, 또는 동의 배너의 정책 링크(UC-62, MON14)로 정책 페이지에 진입한다.
2. 요청한 정책 페이지 유형에 따라 해당 정적 페이지가 반환된다(4종: 개인정보처리방침 / 이용약관 / 면책조항 / 광고·제휴 고지).
3. 사용자가 정책 내용을 열람한다.
4. 정책 페이지는 얇은 유틸리티 페이지로 자동광고 OFF/최소·수동 슬롯 없음이며(MON7-6), 클릭 유도·과도 광고 없이 AdSense 프로그램 정책에 정합한다(MON18).

**대안·예외 흐름 — 페이지 유형별 열람 내용/정합**

| 페이지 유형 | 열람 내용 | 정합 근거 |
|-------------|-----------|-----------|
| 개인정보처리방침 | 사용자별 서버 저장 없음·입력 데이터는 브라우저 localStorage에만 저장됨을 명시. 광고/분석 쿠키·제3자(Google) 사용 사실 명시 | §2-1·§2-2, F7 수용기준 2, MON13·MON14 |
| 이용약관 | 서비스 이용 조건. loupit은 비교·정보 도구이며 채용 플랫폼이 아님 | §1, PRD 03-타깃-사용자 §6(비타깃) |
| 면책조항(데이터 정확성) | 비교 데이터·복지 정보의 정확성 한계 명시(추정·만료 배지, 불확실성 밴드) | F7 수용기준 4, §7(밴드), UC-42(배지)·F3 |
| 광고·제휴 고지 | "광고" 표기 원칙, 제휴 관계 고지, 공정위 표시광고 규정 정합 | MON11, F6, §2-5 |

| 분기 | 조건 | 처리 |
|------|------|------|
| 1a | 동의 배너 링크로 진입(UC-62) | 동의↔정책 단일 진실 연결(MON14). 배너와 정책 페이지가 동일 사실(쿠키·제3자·광고 목적)을 참조 |
| 2a | 크롤러(A3)의 정책 페이지 요청 | 정적 HTML이라 JS 없이 색인 가능. 필수 정책 게시 사실이 AdSense 승인 심사에서 확인됨(MON17·MON18) |
| 2b | 특정 정책 페이지 미게시·404 | AdSense 정책 위반 위험(MON17 미충족) — 게시 누락은 UC-A4 배포 단계에서 방지 대상(리스크 R1) |

**사후조건**
- 4종 정책·고지 내용이 사용자에게 열람 가능하고, 동의 배너·광고/제휴 표기와 단일 진실로 연결된다.
- 공통 사후조건 제약(열람 과정 서버 사용자저장 없음).
