## 위임 케이스 (다른 SP 스위트에 실구현/검증 — 본 도메인 이중 구현 없음)
- **SP-DS(SP-DSN) 위임**: `styles.css`의 `.ad-label`·`.ad-slot`·`.ad-slot--content-mid`(`min-height:280px`)·`.ad-slot--content-bottom`/`--report-bottom`(`min-height:250px`)·`.affiliate-card`·`.affiliate-cta` 실 색·배경·테두리·여백·라벨 폰트 크기(SP-DS-3 광고 라벨 색). 본 도메인은 클래스명 부여·`data-reserve-*` 방출·`min-height` 하한 규격·라벨 11px 하한 **계약만** 소유(색·polish 재정의 금지, T-08.5.1·T-08.8.1).
- **SP-FE 위임**: `dom.js`의 `escapeHtml`·`safeUrl`·`el`(SP-FE-9, `ads.js`가 import·소비 — `filterAffiliate`/`buildAffiliateAttrs`의 `safeUrl`, 렌더의 `el`/`textContent`)·`store.js` `store`(SP-FE-10, 동의 봉투 저장). 안전 유틸 회귀는 UT-ESC-1~3(SP-FE, Tier0). `mountAds`/`initConsentBanner` **마운트 지점**(`bindGlobalUI`·report 뷰 전환)은 SP-FE-5·3이 호출. 본 도메인은 함수 제공·소비만, 유틸·호출 배선 재구현 금지.
- **SP-GEN 위임**: 정적 회사/조합/정책 페이지(F4·F5·F7) 본문·SEO·`<body data-page-type>`·`data-ad-position` 컨테이너·`data-affiliate-host`·`#consent-banner` 공통 파셜·인라인 마운트 스크립트(SP-ADS-9.2) **방출**. 방출 정확도·개수·비-JS 본문은 GC-*(SP-GEN). 본 도메인은 소비 셀렉터 계약(SP-ADS-9.1)만 소유(T-08.9.1).
- **SP-POL 위임**: 정책 페이지 4종(개인정보처리방침 `/privacy`·광고·제휴 고지 `/ads`) 본문·게시. 본 도메인은 동의 배너가 링크하는 연결(CS-3)·`policy` 무광고 게이팅만.
- **SP-ADS-10 릴리스 게이트(SP-ARCH-9·SP-TEST 종합)**: UT-ADS-* 전부 green + MB-ADS-2·3·4·6·8 통과. **AdSense 승인 게이트(코드 게이트 아님, M8)**: 승인 후 실 client id 발급 → `adsConfig.AD_CLIENT`·`AD_SLOT.*` 치환(코드 로직 무변경 C-5) → 재검증. Tier-0 회귀 게이트는 UT-ADS-GATE-1(T-08.3.1)만(SPEC.md §5).
